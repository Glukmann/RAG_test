## Пример использования TRepForm

Создадим файл шаблона `test.tpl` следующего содержания:

```
Test Report
--------------------------------------------
|--<!--id:r-->################--|--<!--Name:c-->#################--|
--------------------------------------------
```

Пример RSL-программы, использующей формирование отчёта при помощи файла шаблона `test.tpl`:

```rsl
import prnfrm;

ob = TRepForm ("test");
ob.value("id") = "123";
ob.value("name") = "ааа";
ob.Write;
```

## Методы и свойства класса TRepForm

### Свойства

| Свойство | Описание |
|----------|----------|
| `Value(name \| id)` | Чтение и запись значения в поле шаблона с именем `name` или индексом `id` |

### Методы

| Метод | Описание |
|-------|----------|
| `Write` | Выводит в стандартный выходной поток шаблон формы с заполненными полями |

## Стандартный класс TPattFieldR

Экземпляры класса `TPattFieldR` создаются автоматически для каждого поля, заданного в файле шаблона. Каждый объект содержит информацию для одного поля. Доступ к экземплярам осуществляется посредством свойства `Field` класса `TRepForm`.

### Свойства класса TPattFieldR

| Свойство | Описание |
|----------|----------|
| `Value` | Чтение и запись значения в связанное с объектом поле |
| `Name` | Имя поля (только для чтения) |
| `Attr` | Атрибуты выравнивания поля. Возможные значения: `1` — по левому краю, `2` — по правому краю, `3` — по центру |

## Поддержка интерактивного режима

В состав языка RSL входят процедуры, позволяющие организовать интерактивный режим работы, значительно облегчающий процесс общения с компьютером.

Пользователь имеет возможность:

- создавать меню для выбора варианта из заранее заданного списка;
- производить запрос на ввод данных любого типа;
- организовывать скроллинг по файлам;
- разрабатывать специальные диалоговые окна для ввода и корректировки данных.

Процедуры ввода данных описаны в разделе встроенных процедур. В данном разделе приведено описание пользовательских меню и диалоговых окон.

## Меню

Если разрабатываемая программа имеет несколько вариантов выполнения, то самый простой способ организовать ветвление — дать пользователю возможность выбора варианта из меню.

В Object RSL предусмотрена возможность создавать как вертикальное, так и горизонтальное меню.

### Вертикальное меню

Для создания вертикального меню необходимо создать массив строк с вариантами выбора и вызвать процедуру `Menu`.

```rsl
array m;
m(0) = "Первый вариант решения";
m(1) = "Другой вариант";
m(2) = "Еще один вариант";

вариант = Menu (m, "Какой вариант желаете выбрать?");

if (вариант == 0)
    println ("Выбран первый вариант");
elif (вариант == 1)
    println ("Выбран другой вариант");
elif (вариант == 2)
    println ("Выбран еще один вариант");
end;
```

Процедура `Menu` возвращает индекс выбранного элемента меню.

### Горизонтальное меню

Горизонтальное меню создаётся и сохраняется в специальной библиотеке. Для создания, редактирования и тестирования горизонтальных меню предусмотрены специальные команды в режиме редактирования ресурсов.

Описание горизонтального меню начинается с ключевого слова `MENU`, после которого указывается имя меню и комментарий. Далее в блоках `POPUP`—`END` описывается ниспадающее меню. Ключевое слово `END` заканчивает описание.

Имя и комментарий для ниспадающего меню указываются после ключевого слова `POPUP`. Внутри блока `POPUP`—`END` описываются пункты меню `ITEM` и разделители `DELIM`.

Для каждого пункта необходимо задать:

| Элемент | Описание |
|---------|----------|
| `имя` | Название пункта меню на экране |
| `идентификатор` | Целочисленный код, используемый для обработки выбранного пункта |
| `комментарий` | Текст, отображаемый в статус-строке при выборе пункта |

**Важно:** целочисленный идентификатор не должен быть равен `256`, так как данный код используется для обработки нажатия клавиши `ESC`.

### Пример горизонтального меню

```rsl
MENU Demo, "Демонстрационное меню"
    POPUP "~К~оманды", "Команды демонстрационного меню"
        ITEM "~П~ечать", 102, "Печать отчета"
        DELIM
        ITEM "~В~ыход", 101, "Окончание работы"
    END
END
```

Для вызова горизонтального меню из RSL-программы используется процедура `RunMenu`, которая подробно описана в разделе встроенных процедур.

## Диалоговые окна

Если необходимо предоставить пользователю возможность просматривать и корректировать сложную совокупность данных, то лучше всего для этого воспользоваться диалоговыми окнами.

Форма диалогового окна разрабатывается в специальном редакторе диалоговых окон и сохраняется в файле-библиотеке.

Для языка RSL диалоговое окно выглядит как структура. Чтение значений из полей этой структуры позволяет получать значения, введённые пользователем в поля диалоговой панели. Запись в поля структуры новых значений позволяет определить содержимое полей диалоговой панели.

### Определение диалога

```rsl
Record dlg (demo) dialog;
```

Идентификатор `dlg` связан с объектом типа `RECORD`, имеющим название `demo` и задающим структуру диалогового окна из библиотеки диалогов.

### Запуск диалога

```rsl
RunDialog (dlg)
```

### Альтернативная библиотека диалогов

Имеется возможность одновременной работы с диалоговыми панелями, хранящимися в разных файлах библиотек ресурсов:

```rsl
Record dlg (demo, "mydir\\mylib.lbr") dialog;
```

### Использование класса TRecHandler

Идентификатор диалога может также являться объектом класса `TRecHandler` или его наследником. Для создания такого идентификатора необходимо создать экземпляр класса `TRecHandler`:

- первый параметр — имя диалоговой панели из библиотеки диалогов;
- второй параметр — имя LBR-файла с диалоговыми панелями (если не задан — используется библиотека по умолчанию);
- третий параметр — `TRUE`.

Все сопутствующие процедуры (`RunDialog`, `UpdateFields` и т.п.) при этом могут принимать ссылку на объект типа `TRecHandler` или его наследника.

```rsl
dlg = TRecHandler ("Test", NULL, TRUE);
RunDialog (dlg);
```

### Функциональные клавиши

| Клавиша | Действие |
|---------|----------|
| `[Esc]` | Завершение работы с диалоговым окном без сохранения введённых значений |
| `[F9]` | Завершение работы с диалоговым окном и сохранение введённых значений |
| `[Enter]` | Завершение редактирования текущего поля и перевод фокуса ввода в следующее поле. Если нажата в последнем поле — эквивалентно `[F9]` |

## Процедура обработки сообщений диалога

Если необходимо специальным образом обрабатывать нажатия клавиш в диалоговой панели, проверять корректность введённой информации и т.д., пользователь может написать специальную процедуру обработки сообщений. Эта процедура вызывается после действий пользователя с диалоговым окном.

Процедура `RunDialog` генерирует сообщения только в том случае, если пользователем определена специальная процедура для их обработки.

### Подключение процедуры обработки

```rsl
RunDialog (dlg, "EvMacro")
/* или */
RunDialog (dlg, @EvMacro)
```

### Сигнатура процедуры обработки

```rsl
macro EvMacro (dlg, cmd, id, key)
```

| Параметр | Описание |
|----------|----------|
| `dlg` | Ссылка на объект типа `RECORD` со структурой диалогового окна |
| `cmd` | Код сообщения (каждый код имеет предопределённую константу) |
| `id` | Номер текущего поля диалога (`V_INTEGER`), который может быть использован для определения имени поля при помощи процедуры `FldName` |
| `key` | Код нажатой клавиши. Появляется, если `cmd` равен `DLG_KEY` или `DLG_MOUSE` |

### Список обрабатываемых сообщений

| Сообщение | Описание |
|-----------|----------|
| `DLG_INIT` | Посылается перед отображением диалога на экране. Можно произвести инициализацию и установить фокус ввода в нужное поле при помощи `SetFocus`. Необходимо вернуть `CM_IGNORE` |
| `DLG_KEY` | Пользователь нажал клавишу на клавиатуре |
| `DLG_BUTTON` | Пользователь щёлкнул мышью на экранной кнопке |
| `DLG_REMFOCUS` | Посылается перед переносом фокуса из текущего поля |
| `DLG_SETFOCUS` | Посылается непосредственно перед установкой фокуса ввода в текущее поле |
| `DLG_SAVE` | Перед запросом на выход из диалога с сохранением изменений. Если вернуть `CM_CANCEL` — выход не произойдёт |
| `DLG_DESTROY` | Посылается непосредственно перед удалением диалогового окна с экрана |
| `DLG_MOUSE` | Пользователь использует мышь |

### Значения параметра key для мыши

| Значение | Действие |
|----------|----------|
| `0` | Нажатие левой кнопки мыши |
| `2` | Отпускание левой кнопки мыши |
| `7` | Двойной щелчок левой кнопкой мыши |

### Возвращаемые значения процедуры EvMacro

При обработке сообщений `DLG_KEY` и `DLG_BUTTON` процедура `EvMacro` может вернуть одно из следующих значений:

| Значение | Действие |
|----------|----------|
| `CM_SAVE` | Завершить работу с диалогом и сохранить введённые данные |
| `CM_CANCEL` | Завершить работу с диалогом, не сохраняя данные |
| `CM_IGNORE` | Игнорировать нажатие клавиши или кнопки (по умолчанию для кнопки) |
| `CM_DEFAULT` | Выполнить действие по умолчанию |

При обработке сообщений `DLG_REMFOCUS` и `DLG_SAVE` может возвращаться:

| Значение | Действие |
|----------|----------|
| `CM_CANCEL` | Не перемещать фокус из текущего поля или не закрывать диалог |
| `CM_DEFAULT` | Разрешить перемещение фокуса или закрытие диалога с сохранением |

Если процедура `EvMacro` не обрабатывает сообщение, она должна вернуть `CM_DEFAULT`.

### Обновление полей и установка фокуса

Если при обработке сообщений полям структуры диалога присваивается новое значение, для отображения изменений на экране необходимо вызвать процедуру `UpdateFields`.

Для установки фокуса ввода в нужное поле применяется процедура `SetFocus`. Эта процедура посылает сообщение `DLG_REMFOCUS` в процедуру обработки сообщений от диалоговой панели.

### Пример процедуры обработки сообщений

```rsl
macro EvenMacro (dlg, cmd, id, key)
    /* Установим фокус ввода на поле Dat */
    if (cmd == DLG_INIT)
        SetFocus (dlg, "Dat");
        return CM_IGNORE;
    /* Нажата клавиша F3 */
    elif (cmd == DLG_KEY)
        if (key == 317)
            dlg.Int = 500;
            dlg.Lab = "NewLabel";
            dlg.Str = "Это новая строка";
            UpdateFields (dlg);
        /* Нажата клавиша Enter */
        elif ((key == 13) and (id == 6))
            SetFocus (dlg, 0);
            return CM_IGNORE;
        end;
    /* Нажата экранная кнопка */
    elif (cmd == DLG_BUTTON)
        /* Выйти с сохранением */
        if (trim (fldname (dlg, id)) == "~O~k")
            return CM_SAVE;
        /* Выйти без сохранения */
        elif (trim (fldname (dlg, id)) == "~C~ancel")
            return CM_CANCEL;
        end;
    end;
    /* Проверка правильности ввода данных */
    if (cmd == DLG_REMFOCUS)
        if ((id == 1) and (dlg.Int < 100))
            MsgBox ("Значение должно быть больше 100");
            return CM_CANCEL;
        end;
    elif (cmd == DLG_SETFOCUS)
        message ("Focus installed");
    elif (cmd == DLG_SAVE)
        if (dlg.Int < 100)
            SetFocus (dlg, 1);
            MsgBox ("Значение должно быть |>100");
            return CM_CANCEL;
        end;
    elif (cmd == DLG_DESTROY)
        message ("Dialog is destroyed");
    end;
    return CM_DEFAULT;
end
```

### Пример работы с диалогом в ООП-стиле

```rsl
const K_F3 = 317, K_F4 = 318;

macro MyDlg (dlg, cmd, id, key)
    return dlg.MsgHandler (cmd, id, key)
end;

class (TRecHandler) TestDialogBase (resName)
    InitTRecHandler (resName, NULL, true);
    
    macro onF3
        MsgBox ("Default F3 handler");
    end;
    
    macro onF4
        MsgBox ("Default F4 handler")
    end;
    
    macro MsgHandler (cmd, id, key)
        if (cmd == DLG_KEY)
            if (key == K_F3) onF3;
            elif (key == K_F4) onF4;
            end;
        end;
    end;
    
    macro Run
        RunDialog (this, @MyDlg);
    end;
end;

class (TestDialogBase) TestDialog (par)
    InitTestDialogBase ("Test");
    var prop = par;
    rec.f1 = "Test string";
    rec.f2 = "Other Test string";
    
    macro onF3
        rec.f1 = prop;
        rec.f2 = "22222222222";
        UpdateFields (this, 1);
        MsgBox ("New F3 handler");
        prop = "New prop Value"
    end;
    
    macro MsgHandler (cmd, id, key)
        trace ("Cmd = ", cmd);
        return MsgHandler (cmd, id, key);
    end;
end;

dlg = TestDialog ("Value for f1 field");
dlg.Run
```

## Скроллинг

С помощью языка RSL пользователь может организовать просмотр на экране Btrieve-файла с возможностью "прокручивания" его вперёд и назад.

Для решения этой задачи служит макромодуль `RslScr`, в котором определены специальные процедуры и переменные. Основным компонентом модуля является специальная процедура обработки сообщений от диалоговой панели `ScrollMes`. Эта процедура обрабатывает нажатия клавиш `[PgUp]`, `[PgDown]` и клавиш управления курсором для выполнения прокрутки записей из файла.

### Инициализация скроллинга

Для корректной работы процедуре `ScrollMes` необходима инициализирующая информация:

- файл, по которому производится скроллинг;
- соответствие полей диалоговой панели полям в файле.

Эта информация задаётся при помощи процедуры `SetScroll`:

```rsl
SetScroll (FILE, DlgName, FileName, ...)
```

| Параметр | Описание |
|----------|----------|
| `FILE` | Идентификатор файла, по которому будет производиться прокрутка |
| `DlgName` | Имя первой колонки в диалоговой панели |
| `FileName` | Имя поля в файле `FILE`, данные из которого выводятся в колонку `DlgName`. Если имя поля в файле совпадает с именем колонки, можно указать `NULL` |

Пары `DlgName-FileName` должны быть указаны для каждой колонки в скроллинге. Процедура `SetScroll` должна вызываться перед вызовом `RunDialog`.

### Структура колонок в диалоговой панели

Область прокрутки в диалоговой панели должна быть создана как совокупность колонок. Каждая колонка образуется из набора редактируемых полей. Названия полей в колонке должны быть одинаковыми за исключением окончания. Окончанием в названии поля должен быть номер строки в колонке. Первая строка имеет номер `0`.

Например:

- `ColName0` — имя первой строки в колонке;
- `ColName1` — имя второй строки в колонке.

Именем колонки подразумевается имя поля без окончания. В данном случае именем колонки будет `ColName`. Именно это имя передаётся процедуре `SetScroll`.

### Порядок создания скроллинга

1. С помощью редактора ресурсов создать диалоговую панель.
2. Используя директиву `import`, импортировать в RSL-программу модуль `RslScr`.
3. Непосредственно перед вызовом процедуры `RunDialog` вызвать `SetScroll` для задания соответствия имён полей в файле и диалоговой панели.
4. Вызвать процедуру `RunDialog` и задать процедуру обработки сообщений.
5. В процедуре обработки сообщений вызывать предопределённую процедуру `ScrollMes` для всех необработанных сообщений.

### Пример вызова SetScroll

Область прокрутки в диалоговой панели состоит из двух колонок с названиями `Blnc` и `NameB`:

```rsl
SetScroll (ff,
    "Blnc", "Balance",
    "NameB", "Name_Part")
```

После вызова `SetScroll` запуск диалоговой панели с областью прокрутки производится обращением к процедуре `RunDialog`. Если не указывается пользовательская процедура обработки сообщений, необходимо указать имя стандартной процедуры `ScrollMes`. Если создаётся пользовательская процедура обработки сообщений, то в ней необходимо для всех необработанных сообщений вызывать `ScrollMes`.

Стандартная процедура `ScrollMes` обрабатывает нажатие клавиши `[Enter]`. При нажатии на данную клавишу диалоговая панель закрывается, выбранная запись в области прокрутки делается текущей записью в файле и процедура `RunDialog` возвращает `TRUE`.

### Рекурсивный вызов скроллинга

Пользователю необходимо быть внимательным при отображении диалога с областью прокрутки из процедуры обработки сообщений от диалоговой панели, также содержащей область прокрутки. Так как макромодуль `RslScr` содержит ряд глобальных переменных, рекурсивный вызов может уничтожить данные для текущей диалоговой панели. Чтобы избежать этого, нужно сохранять значения глобальных переменных.

```rsl
var SaveApos, SaveNrec, SaveNfields, SaveScrff, InUse = FALSE;
array saveDlgFields;
array SaveFileFields;

SaveApos = Apos (0);
SaveNrec = Nrec;
SaveNfields = Nfields;
SaveScrff = Scrff;
SaveDlgFields = DlgFields;
SaveFileFields = FileFields;

SetScroll (Scrff,
    "Blnc", "Balance",
    "NameB", "Name_Part"
);
```

## Пример скроллинга с отбором записей

Данный пример демонстрирует использование области скроллинга в диалоговой панели. Также в нём демонстрируется техника замены процедур, которая позволяет организовать отбор записей для показа в области скроллинга.

Так как код из макромодуля `rslscr.mac` для навигации по файлу использует только процедуры `next`, `prev`, `rewind`, то, подменяя эти процедуры на специальные, обеспечивается навигация только по заданному подмножеству записей.

В данном примере по нажатию клавиши `[F5]` производится запрос максимального и минимального номера счёта для отображения в области скроллинга. Предполагается, что в редакторе диалоговых панелей созданы основная панель `BLNC`, содержащая колонки с именами `Blnc`, `NameB` и `Num`, и вспомогательная панель `RBdlg` для задания интервала счетов.

В примере также показано, как можно использовать процедуру `UserFill`, которая заменяется на `MyFill`. Процедура `MyFill` заполняет колонку `Num`, используя первые два символа из колонки `Blnc`. Колонки `Blnc` и `NameB` заполняются данными из файла `balance.dbt`.

```rsl
/*
    Пример использования скроллинга по файлу с
    отбором записей из заданного интервала
*/
import rslscr;            /* Поддержка скроллинга */
file ff (balance) key 0;  /* Рабочий файл */

/* Отображать на экране будем только заданный интервал счетов */
var fromB = "",    /* Минимальное значение */
    toB = "";      /* Максимальное значение */

/* ---------------------------------------------------
   Процедуры MyNext, MyPrev, MyRewind будут
   использоваться вместо стандартных next, prev, rewind
   для обеспечения отбора записей из заданного интервала
-----------------------------------------------------*/
var first = TRUE;

macro MyNext (ff)
    /* Проверка на превышение максимального значения */
    macro Test
        if (toB and (ff.Balance > toB)) return FALSE end;
        return TRUE
    end;
    if (first and fromB)
        first = FALSE;
        /* Установка минимального значения для первой записи */
        ff.Balance = fromB;
        if (getGE (ff)) return Test end
    else
        if (next (ff)) return Test end
    end;
    return FALSE
end;

macro MyPrev (ff)
    /* Проверка достижения минимального значения */
    macro Test
        if (fromB and (ff.Balance < fromB)) return FALSE; end;
        return TRUE;
    end;
    if (first and toB)
        first = FALSE;
        /* Установка максимального значения для последней записи */
        ff.Balance = toB;
        if (getLE (ff)) return Test end
    else
        if (prev (ff)) return Test end
    end;
    return FALSE
end;

macro MyRewind (ff)
    rewind (ff);
    first = TRUE
end;

macro MyFill (dlg)
    var str, i = 0;
    while (i < Nrec)
        str = dlg (FldIndex (dlg, "Blnc" + i));
        dlg (FldIndex (dlg, "Num" + i)) = substr (str, 1, 2);
        i = i + 1
    end
end;
```
