## Разработка печатных форм

в doc-формате


в doc-формате
В ИБС предусмотрена подготовка печатных форм средствами офисного пакета. С этой
целью используется механизм печати по шаблонам, выполняющий импорт данных из ИБС в
печатные формы, подготовленные в doc-формате.
В состав механизма входят следующие макрофайлы, размещенные в каталоге MAC\CB на
терминале пользователя:
- dlwreps.mac – передаёт данные на терминал пользователя и выбирает режим работы
механизма печати по шаблону;
- dlwrept.mac – макрос обработки содержит собственно механизм печати по шаблону, с
помощью которого создаётся и заполняется печатная форма;
- dlreptl.mac – используется для определения путей к шаблонам печатных форм при
функционировании ИБС RS-Bank V.6 в двухуровневой архитектуре "Клиент-сервер";
- dlreptls.mac – содержит вспомогательные функции для определения имен txt-файла
данных и rtf-файла с печатной формой, а также путей к ним.
Шаблон печатной формы размещен в dot-файле.
Сбор данных для заполнения печатной формы выполняет макрос формирования данных.
Заполненная печатная форма помещается в файл с расширением ".rtf".
Имена макросов, осуществляющих выгрузку данных в текстовый файл; имена файлов с
шаблонами; описание закладок в шаблонах; имена файлов с заполненными печатными
формами приведены в Руководствах к подсистемам, в которых реализовано использование
механизма печати по шаблону.

Процесс генерации документа с использованием механизма по шаблону выполняется в два
этапа:
- Формирование текстового файла с данными.
На сервере приложений в каталоге TxtFile макрос формирования данных создает
текстовый файл и помещает в него данные и дополнительную
 служебную
информацию. Данные будут в последствии использованы при заполнении печатной
формы.
Имя созданного файла передается в функцию DLWREPS_PrintReportFromTagFile,
расположенную в макросе dlwreps.mac, которая передает его на терминал пользователя.
После чего на терминале запускается макрос обработки dlwrept.mac.
- Заполнение печатной формы.
Макрос обработки создает на основе dot-шаблона
 rtf-файл c печатной формой, в
которой замещает содержимое закладок данными из переданного текстового файла.
Алгоритм работы макроса обработки на терминале является универсальным для любых
шаблонов приложений офисного пакета, т.к. не зависит ни от вида конкретного шаблона,
ни от состава закладок шаблона.
После замещения данных макрос обработки запускает приложение офисного пакета и
открывает созданный файл с документом.

Текстовый файл данных состоит из набора строк.
В первой строке содержится имя файла-шаблона приложения dot-формата.
Во второй строке содержится имя rtf-файла печатной формы.
Далее следует произвольное число строк вида:
~Метка
Значение
Где:
- Метка – идентификатор закладки в шаблоне dot-формата.
Меткой считается текст, размещенный между тильдой (символ "~") и маркером абзаца
(маркер устанавливается по клавише [Enter]).
- Значение – строковые данные, которыми замещается текст в месте, обозначенном
указанной закладкой.
Значением, соответствующим метке, считается текст до следующей тильды, который
может содержать неограниченное число строк.

## Обрамление таблицы

Ячейки таблицы в шаблоне можно обрамлять рамками. Такой прием часто используется для
визуального выделения вводимой информации.
Однако в приложении офисного пакета есть особенность – при добавлении, например,
нижнего подчеркивания в ячейке на строке 1 автоматически добавляется верхнее
подчеркивание на строке 2, что в последствии приводит к нежелательным результатам в
виде двойных жирных линий.
Избежать подобных случаев можно с помощью добавления двух пустых строк после строки
с подчеркиванием с последующим их удалением в файле с данными.
Пример шаблона с таблицей, где ячейки таблицы имеют нижнее обрамление

## Вставка изображений

Подготовка печатных форм с использованием технологии Apache POI позволяет добавлять
изображения на страницу с документом.
Такая функциональность часто требуется при добавлении факсимиле или оттиска печати.
Для добавления изображения используется запись, вида:
~InsertImage
<Метка>#Путь к файлу с изображением#ширина#высота
Файл с изображением на момент формирования отчета должен уже присутствовать в
доступной директории. Например, для подписи можно воспользоваться стандартным
механизмом выгрузки подписи из справочника субъектов.
Для масштабирования изображения предусмотрены два необязательных параметра:
#ширина#высота. Если эти параметры не указаны, то изображение будет выведено "как
есть". Если указать ширину и высоту изображения, то оно с масштабируется согласно
заданным параметрам.
Пример.
Пример вывода изображения из файла SIGN_325.jpg, который находится в
директории D:\RS_BANK\WorkFile\ сервера приложений, в метку Sign с
масштабированием в 154 точки по ширине и 142 точки по высоте:
~InsertImage
Sign#D:\RS_BANK\WorkFile\SIGN_325.jpg#154#142
