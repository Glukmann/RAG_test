# RSL Forms

Введение
Стандартный RSL-модуль RsbFormsInter предназначен для обеспечения возможности
работы с визуальными компонентами. 
Визуальные компоненты в рамках стандартного RSL-модуля RsbFormsInter делятся на
следующие группы:
· Элементы управления:
· текстовое поле;
· элемент управления типа CheckBox;
· элемент управления типа "Кнопка";
· элемент управления типа RadioButton;
· элемент управления типа ComboBox, представляющий собой выпадающий список.
· Окна:
· панель – диалоговое окно с произвольно расположенными на нем элементами
управления;
· окно с закладками – диалоговое окно, содержащее несколько закладок, каждая из
которых является панелью.
· Текстовые метки.
Все визуальные компоненты объединены в иерархию классов
.
В стандартном RSL-модуле RsbFormsInter используется событийная модель. Для
представления событий вводится иерархия событий. Каждое событие имеет свой источник.
Источник события представляет собой экземпляр класса, являющегося наследником
базового класса TRsbEventSource
. Все визуальные компоненты, за исключением
текстовых меток, являются источниками событий. С источниками событий можно
ассоциировать обработчики событий.
При ссылке на информацию, расположенную в настоящем руководстве, указывается номер
страницы (в скругленном прямоугольнике

раздела, содержащего требуемую информацию.

Способ создания визуального компонента типа "Окно" определяется его видом:

Особенности работы с событиями
Событие – это экземпляр базового класса TRsbEvent
 или его классов-наследников.
Каждое событие имеет свой источник. Источник события представляет собой экземпляр
класса, являющегося наследником базового класса TRsbEventSource
. С источниками
событий можно ассоциировать обработчики событий.
В качестве обработчика события могут выступать:
· ссылка на глобальную функцию;
· объект типа R2M.
Обработчики событий ассоциируются с источником и типом события, причем с одной парой

тип 
события> 
может 
быть 
ассоциировано 
произвольное 
количество
обработчиков.
Обработка событий
Процесс обработки событий осуществляется в строгой последовательности, определяемой
порядком генерации событий. Если при обработке события происходит генерация другого
события, то новое событие ставится в очередь и обрабатывается после завершения
обработки родительского события.
Технология работы с RSL-модулем RsbFormsInter
Очередь 
событий 
представляет 
собой 
FIFO-очередь 
без 
приоритетов, 
которая
предназначена для хранения объектов событий, которые не могут быть обработаны в
момент генерации. Очередь событий существует в единственном экземпляре.
При обработке событие передается в обработчики, ассоциированные с соответствующим
источником и типом события, в порядке, обратном их генерации. Если один из обработчиков
события возвращает значение FALSE, то его обработка завершается и событие считается
обработанным.
Источники событий могут образовывать иерархии, основанные на отношении агрегации
("часть-целое"). После обработки события обработчиками его источника событие
передается обработчикам источника, расположенного в иерархии агрегации выше источника
данного события.
Пример.
Если 
событие 
"Нажатие 
клавиши" 
обрабатывалось 
обработчиком
визуального компонента типа "Элемент управления", то после обработки
оно передается в обработчик визуального компонента "Экранная форма".
Любой обработчик в цепочке может завершить обработку события.

resName – наименование загружаемого ресурса.
lbrName – наименование библиотеки ресурсов. Если параметр не указан или
значение параметра равно NULL, то меню загружается из пользовательской
библиотеки ресурсов, наименование которой задано в настройке банка
BANK_INI\ОБЩИЕ ПАРАМЕТРЫ\ФАЙЛЫ\LBRNAME.

Виды событий
В стандартный RSL-модуль RsbFormsInter
 входят следующие константы, определяющие
виды событий:
RSB_EV_WINDOW_CLOSING = 0 – идентификатор события, которое генерируется при
инициировании закрытия окна.
RSB_EV_KEY_PRESSED =1 – идентификатор события, которое генерируется при нажатии
клавиши клавиатуры.
RSB_EV_MOUSE =2 – идентификатор события, которое генерируется при нажатии
клавиши мыши.
RSB_EV_ITEM_SELECTED = 3 – идентификатор события, которое генерируется при
выборе значения в выпадающем списке (для элемента управления типа
ComboBox).
RSB_EV_CONTROL_CHECKED = 4 – идентификатор события, которое генерируется при
установке или снятии отметки в элементе управления типа CheckBox или
RadioButton.
RSB_EV_BUTTON_CLICKED = 5 – идентификатор события, которое генерируется при
нажатии на кнопку.
RSB_EV_ENTER_WINDOW = 6 – идентификатор события, которое генерируется при
входе в окно формы.
RSB_EV_QUIT_WINDOW = 7 – идентификатор события, которое генерируется при выходе
из окна. Выход из окна генерируется после закрытия окна.
RSB_EV_SET_FOCUS = 8 – идентификатор события, которое генерируется при установке
фокуса ввода в элемент управления.
RSB_EV_REMOVE_FOCUS = 9 – идентификатор события, которое генерируется при
снятии фокуса ввода с элемента управления.
RSB_EV_MIN_USER = 100 – минимальный идентификатор пользовательского события.
Типы полей экранных форм
В стандартный RSL-модуль RsbFormsInter
 входят следующие константы, определяющие
типы полей экранных форм:
FBS – кнопка с тенью.
FBT – кнопка.
FCL – элемент управления типа CheckBox или RadioButton.
FET – редактируемое поле.
FSP – вложенная панель.
FVT – нередактируемое поле.
FVW – многострочное нередактируемое поле.
FWR – многострочное редактируемое поле.
