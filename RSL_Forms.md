# Введение

Стандартный RSL-модуль `RsbFormsInter` предназначен для обеспечения возможности работы с визуальными компонентами.

Визуальные компоненты в рамках стандартного RSL-модуля `RsbFormsInter` делятся на следующие группы:

- **Элементы управления:**
  - текстовое поле;
  - элемент управления типа `CheckBox`;
  - элемент управления типа "Кнопка";
  - элемент управления типа `RadioButton`;
  - элемент управления типа `ComboBox`, представляющий собой выпадающий список.
- **Окна:**
  - панель – диалоговое окно с произвольно расположенными на нём элементами управления;
  - окно с закладками – диалоговое окно, содержащее несколько закладок, каждая из которых является панелью.
- **Текстовые метки.**

Все визуальные компоненты объединены в иерархию классов. В стандартном RSL-модуле `RsbFormsInter` используется событийная модель. Для представления событий вводится иерархия событий. Каждое событие имеет свой источник. Источник события представляет собой экземпляр класса, являющегося наследником базового класса `TRsbEventSource`. Все визуальные компоненты, за исключением текстовых меток, являются источниками событий. С источниками событий можно ассоциировать обработчики событий.

> **Примечание.** При ссылке на информацию, расположенную в настоящем руководстве, указывается номер страницы (в скругленном прямоугольнике), соответствующий нужному разделу, или гиперссылка. Ссылка на раздел другого руководства даётся с указанием названия главы и раздела, содержащего требуемую информацию.

# Технология работы с RSL-модулем RsbFormsInter

В разделе представлено описание особенностей работы со следующими элементами:

- визуальными компонентами типа "Окно";
- событиями.

## Особенности работы с визуальными компонентами типа "Окно"

При работе с визуальными компонентами типа "Окно" следует учитывать:

- способ создания визуального компонента;
- фазу жизненного цикла визуального компонента.

## Создание визуальных компонентов типа "Окно"

Способ создания визуального компонента типа "Окно" определяется его видом:

- визуальный компонент "Окно с закладками" создаётся только динамически из программного кода;
- визуальный компонент "Панель" создаётся либо динамически из программного кода, либо с помощью загрузки ресурса из библиотеки ресурсов.

## Фазы жизненного цикла визуальных компонентов типа "Окно"

Жизненный цикл динамически создаваемого визуального компонента типа "Окно" включает 2 фазы:

- **Фаза построения** – продолжается от момента создания визуального компонента до начала его использования. Под "использованием" визуального компонента подразумевается выполнение следующих действий: отображение его на экране и т. д. В фазе построения визуальные компоненты типа "Окно" доступны для изменения: на визуальный компонент "Панель" можно добавлять элементы управления, а для визуального компонента "Окно с закладками" – новые закладки.
- **Фаза использования** – продолжается с момента начала использования визуального компонента до его удаления. В этой фазе визуальные компоненты типа "Окно" недоступны для изменения.

Способ установки/определения фазы жизненного цикла визуального компонента типа "Окно" зависит от его вида:

- Для визуального компонента "Панель" жизненный цикл определяется способом его создания:
  - При динамическом создании визуальный компонент "Панель" находится на фазе построения. При вызове методов `run()` и `use()` визуальный компонент переходит в фазу использования.
  - Если визуальный компонент "Панель" создаётся с помощью загрузки ресурса из библиотеки ресурсов, то его фаза жизненного цикла определяется как "Фаза использования".
- В отличие от визуального компонента "Панель" визуальный компонент "Окно с закладками" всегда при создании находится на фазе построения. При вызове методов `run()` и `use()` визуальный компонент переходит в фазу использования.

> **Внимание!** Перейти из фазы использования в фазу построения невозможно.

## Особенности работы с событиями

Событие – это экземпляр базового класса `TRsbEvent` или его классов-наследников.

Каждое событие имеет свой источник. Источник события представляет собой экземпляр класса, являющегося наследником базового класса `TRsbEventSource`. С источниками событий можно ассоциировать обработчики событий.

В качестве обработчика события могут выступать:

- ссылка на глобальную функцию;
- объект типа `R2M`.

Обработчики событий ассоциируются с источником и типом события, причём с одной парой `<источник, тип события>` может быть ассоциировано произвольное количество обработчиков.

## Обработка событий

Процесс обработки событий осуществляется в строгой последовательности, определяемой порядком генерации событий. Если при обработке события происходит генерация другого события, то новое событие ставится в очередь и обрабатывается после завершения обработки родительского события.

Очередь событий представляет собой FIFO-очередь без приоритетов, которая предназначена для хранения объектов событий, которые не могут быть обработаны в момент генерации. Очередь событий существует в единственном экземпляре.

При обработке событие передаётся в обработчики, ассоциированные с соответствующим источником и типом события, в порядке, обратном их генерации. Если один из обработчиков события возвращает значение `FALSE`, то его обработка завершается и событие считается обработанным.

Источники событий могут образовывать иерархии, основанные на отношении агрегации ("часть-целое"). После обработки события обработчиками его источника событие передаётся обработчикам источника, расположенного в иерархии агрегации выше источника данного события.

**Пример.**

Если событие "Нажатие клавиши" обрабатывалось обработчиком визуального компонента типа "Элемент управления", то после обработки оно передаётся в обработчик визуального компонента "Экранная форма". Любой обработчик в цепочке может завершить обработку события.

# Классы

В стандартный RSL-модуль `RsbFormsInter` входят следующие группы классов:

- Классы визуальных компонентов.
- Классы механизма событий.
- Классы меню.

## Классы визуальных компонентов

К классам визуальных компонентов относятся следующие классы:

- `TRsbVisualComponent` – базовый класс визуального компонента.
- `TRsbActiveVisualComponent` – класс является наследником классов `TRsbVisualComponent` и `TRsbEventSource`.
- Классы окон:
  - `TRsbWindow` – базовый класс для классов `TRsbForm`, `TRsbFormContainer`.
  - `TRsbForm` – базовый класс для класса `TRsbPanel`.
  - `TRsbPanel` – класс визуального компонента "Панель".
  - `TRsbFormContainer` – базовый класс для класса `TRsbTabbedWindow`. Класс объекта "Контейнер форм".
  - `TRsbTabbedWindow` – класс визуального компонента "Окно с закладками".
- Классы элементов управления:
  - `TRsbControl` – базовый класс для классов `TRsbEditField`, `TRsbComboBox`, `TRsbCheckBox`, `TRsbRadioButton`, `TRsbPushButton`.
  - `TRsbEditField` – класс элемента управления типа "Текстовое поле".
  - `TRsbComboBox` – базовый класс элемента управления типа `ComboBox`, представляющего собой выпадающий список.
  - `TRsbCheckBox` – базовый класс элемента управления типа `CheckBox`.
  - `TRsbRadioButton` – класс элемента управления типа `RadioButton`.
  - `TRsbPushButton` – класс элемента управления "Кнопка".
- `TRsbLabel` – класс объекта "Текстовая метка".

Метод позволяет получить позицию визуального компонента в знакоместах.

#### Методы

##### `addControl(control:Object)`

Метод позволяет добавить элемент управления на форму.

| Параметр | Тип | Описание |
|---|---|---|
| `control` | `Object` | Объект класса `TRsbControl`, представляющий собой элемент управления. |

**Пример.**

Добавить на визуальный компонент `panel` элемент управления `m_EditFld`:

```rsl
import RsbFormsInter;
private const FT_STRING = 7;
class (TRsbPanel) MyRsbPanel1
  InitTRsbPanel();
  setSize(25,6);
  setPosition(2,2);
  var m_sEditFld:String = "RsbFormsInter";
  var m_EditField:TRsbEditField = TRsbEditField(FT_STRING);
  m_EditField.bindValue(this, "m_sEditFld", 20);
  m_EditField.setPosition(3,2);
  m_EditField.setSize(15,1);
  addControl(m_EditField);
end;
var panel = MyRsbPanel1;
panel.run;
```

В результате выполнения макроса на экране отображается визуальный компонент "Панель" с расположенным на нём элементом управления типа "Текстовое поле", содержащим текст `RsbFormsInter`.

## Классы механизма событий

К классам механизма событий относятся следующие классы:

- `TRsbEvent` – базовый класс объекта "Событие", для которого не существует специализированных классов-наследников. Класс является базовым для классов `TRsbKeyPressedEvent`, `TRsbMouseEvent`, `TRsbWindowClosingEvent`.

- `TRsbMouseEvent` – класс события, определяемого константой `RSB_EV_MOUSE`.
- `TRsbWindowClosingEvent` – класс события, определяемого константой `RSB_EV_WINDOW_CLOSING`.
- `TRsbEventQueue` – базовый класс объекта "Очередь событий".
- `TRsbEventSource` – базовый класс источника событий.

IMPORT RsbFormsInter;

IMPORT RsbFormsInter;
class (TRsbPanel) MyRsbPanel

  end;

# Константы

В стандартный RSL-модуль `RsbFormsInter` входят следующие виды констант:

- Виды событий.
- Типы полей экранных форм.

## Виды событий

В стандартный RSL-модуль `RsbFormsInter` входят следующие константы, определяющие виды событий:

| Константа | Значение | Описание |
|---|---|---|
| `RSB_EV_WINDOW_CLOSING` | `0` | Идентификатор события, которое генерируется при инициировании закрытия окна. |
| `RSB_EV_KEY_PRESSED` | `1` | Идентификатор события, которое генерируется при нажатии клавиши клавиатуры. |
| `RSB_EV_MOUSE` | `2` | Идентификатор события, которое генерируется при нажатии клавиши мыши. |
| `RSB_EV_ITEM_SELECTED` | `3` | Идентификатор события, которое генерируется при выборе значения в выпадающем списке (для элемента управления типа `ComboBox`). |
| `RSB_EV_CONTROL_CHECKED` | `4` | Идентификатор события, которое генерируется при установке или снятии отметки в элементе управления типа `CheckBox` или `RadioButton`. |
| `RSB_EV_BUTTON_CLICKED` | `5` | Идентификатор события, которое генерируется при нажатии на кнопку. |
| `RSB_EV_ENTER_WINDOW` | `6` | Идентификатор события, которое генерируется при входе в окно формы. |
| `RSB_EV_QUIT_WINDOW` | `7` | Идентификатор события, которое генерируется при выходе из окна. Выход из окна генерируется после закрытия окна. |
| `RSB_EV_SET_FOCUS` | `8` | Идентификатор события, которое генерируется при установке фокуса ввода в элемент управления. |
| `RSB_EV_REMOVE_FOCUS` | `9` | Идентификатор события, которое генерируется при снятии фокуса ввода с элемента управления. |
| `RSB_EV_MIN_USER` | `100` | Минимальный идентификатор пользовательского события. |

## Типы полей экранных форм

В стандартный RSL-модуль `RsbFormsInter` входят следующие константы, определяющие типы полей экранных форм:

| Константа | Описание |
|---|---|
| `FBS` | Кнопка с тенью. |
| `FBT` | Кнопка. |
| `FCL` | Элемент управления типа `CheckBox` или `RadioButton`. |
| `FET` | Редактируемое поле. |
| `FSP` | Вложенная панель. |
| `FVT` | Нередактируемое поле. |
| `FVW` | Многострочное нередактируемое поле. |
| `FWR` | Многострочное редактируемое поле. |
