## Управление каталогами

### RemoveDir

```rsl
RemoveDir (name: string): bool
```

Процедура удаляет каталог, имя которого задаётся параметром `name`. Каталог может быть удалён как на сервере, так и на терминале.

Возвращает `TRUE` при успешном удалении и `FALSE` в противном случае.

```rsl
if (not RemoveDir ("$rslDir"))
    println ("Error remove dir")
end
```

### GetCurDir

```rsl
GetCurDir ([isRemote: bool]): string
```

Процедура возвращает название текущего каталога на сервере. Если задан параметр `isRemote` и его значение равно `TRUE`, то возвращается название текущего каталога на терминале.

```rsl
/* Печатаем название текущего каталога на терминале */
println (GetCurDir (true))
```

## Классы и объекты

В Object RSL предусмотрено несколько вспомогательных процедур, предназначенных для поддержки работы с классами и объектами.

### GenClassName

```rsl
GenClassName (obj): string
```

Процедура возвращает строку с именем класса для объекта `obj`. Если параметр `obj` не является ссылкой на объект, то возвращается значение типа `V_UNDEF`.

### GenAttach

```rsl
GenAttach (id, methodName, macroName | macroAddr)
```

Процедура позволяет заменить метод `methodName` объекта, заданного идентификатором `id`, глобальной макропроцедурой `macroName` или ссылкой на неё `macroAddr`.

Если заменяемый метод вызывается из заменяющей процедуры, то рекурсивный вызов не происходит, а вызывается метод предыдущего уровня иерархии.

### GenObject

```rsl
GenObject (className [, parm1, parm2, ...])
```

Данная процедура позволяет создавать объекты, имена классов которых задаются при помощи параметров. Первый параметр — строка, задающая имя класса, к которому относятся объекты. Далее следуют необязательные параметры, передаваемые конструктору (инициализатору) объекта.

Процедура возвращает ссылку на созданный объект. Если объект заданного класса не может быть создан, произойдёт ошибка времени выполнения.

Доступ к свойствам объектов осуществляется аналогично доступу к полям файлов:

```rsl
Ob = GenObject ("MyClass");
Ob.prop1 = "New value"
```

Вызов методов осуществляется следующим образом:

```rsl
Ob.method.1 (par1, par2)
```

### GenRun

```rsl
GenRun (ob, methodName [, par1, par2, ...])
```

Процедура вызывает метод с именем `methodName` для объекта `ob` и передаёт ему параметры `par1`, `par2` и т.д. Процедура возвращает значение, возвращаемое вызываемым методом.

```rsl
Val = GenRun (Ob, "Method2", 10)
```

### GenSetProp / GenGetProp

```rsl
GenSetProp (ob, propName, val)
GenGetProp (ob, propName)
```

Процедуры устанавливают и возвращают значение свойства с именем `propName` объекта `ob`.

### IsEqClass

```rsl
IsEqClass (className, obj): bool
```

Процедура проверяет принадлежность объекта `obj` классу объектов с именем `className`. Возвращает `TRUE`, если объект принадлежит классу, и `FALSE` — в противном случае.

```rsl
/* Определяем класс AA */
Class AA
End;

/* Наследуем класс BB от класса AA */
Class (AA) BB
End;

if (IsEqClass ("AA", BB))
    println ("Объект класса BB является и объектом класса АА")
end
```

### GenPropID

```rsl
GenPropID (obj, propName): integer
```

Процедура возвращает индекс свойства с именем `propName` для объекта `obj`. Используя полученный индекс, можно получить доступ к свойству при помощи выражения:

```rsl
A = obj (n);
/* или */
A = obj [n];
```

## Обработка меню

### Menu

```rsl
Menu (array [, prompt] [, head] [, x] [, y] [, n])
```

Процедура формирует на экране меню для выбора варианта. Текст каждой строки меню необходимо определить в элементах массива `array`, который является первым параметром и имеет тип `V_ARRAY`.

| Параметр | Описание |
|----------|----------|
| `prompt` | Сообщение, выводимое в нижнюю строку экрана |
| `head` | Текст, выводимый в заголовок окна. По умолчанию — `"Меню"` |
| `x`, `y` | Координаты левого верхнего угла окна меню. По умолчанию — центр экрана |
| `n` | Номер текущего активного пункта меню |

Процедура возвращает номер элемента массива `ARRAY`, соответствующего выбранному варианту.

**Пример 1 — меню в заданной позиции с заголовком и подсказкой по умолчанию:**

```rsl
array a;
a (0) = "Первый пункт";
Выбор = Menu (a, NULL, NULL, 4, 6)
```

**Пример 2 — меню с заголовком и подсказкой по центру экрана, активен третий пункт:**

```rsl
Array mn;
mn = "Первый пункт";
mn = "Второй пункт";
mn = "Третий пункт";
Menu (mn, "Подсказка", "Заголовок", NULL, NULL, 2)
```

### RunMenu

```rsl
RunMenu (menuName, procName | procAddr [, lbrName])
```

Процедура активизирует выполнение меню, созданного в специальной библиотеке ресурсов. Имя меню в библиотеке ресурсов определяется параметром `menuName`.

| Параметр | Описание |
|----------|----------|
| `procName` / `procAddr` | Имя процедуры обработки команд меню или ссылка на неё |
| `lbrName` | Имя альтернативной библиотеки ресурсов |

Процедура обработки вызывается автоматически, когда пользователь выбирает пункт меню. В качестве параметра она получает целочисленный идентификатор выбранного пункта `id`, определённый при создании меню. Если она возвращает `TRUE`, то это же значение возвращает и процедура `RunMenu`, и обработка меню завершается.

При нажатии клавиши `[Esc]` RSL вызывает процедуру обработки команд меню с идентификатором, равным `256`. Если она вернёт `NULL` или не вернёт никакого значения, обработка меню завершится, и `RunMenu` возвратит значение `FALSE`. Если завершение обработки меню по клавише ESC нежелательно, для идентификатора `256` или для всех необрабатываемых идентификаторов нужно возвращать значение `FALSE`.

```rsl
/* Предположим, в библиотеке создано меню Demo:
   - выход с идентификатором 101
   - печать отчёта с идентификатором 102 */

macro HandleMenu (id)
    if (id == 101) /* Завершить работу меню */
        return TRUE
    elif (id == 102) /* Напечатать отчёт */
        RunReport
    end;
    return FALSE
    /* Возвращать FALSE для необрабатываемых идентификаторов */
end;

RunMenu ("Demo", @HandleMenu)
```

## Обработка диалоговых окон

### RunDialog

```rsl
RunDialog (id [, macName]): bool
```

Процедура отображает на экране диалоговое окно, заданное идентификатором `id`. Поиск по заданному идентификатору ведётся в библиотеке диалогов, созданной специальным редактором диалогов.

Второй необязательный параметр `macName` задаёт имя макроса для обработки сообщений диалогового окна.

Процедура возвращает `TRUE`, если пользователь сохранил данные в диалоге, и `FALSE` в противном случае.

```rsl
record id (name) dialog;
RunDialog (id)
```

Диалог в примере имеет два поля: `Name` и `Address`.

### UpdateFields

```rsl
UpdateFields (id [, N])
```

Процедура обновляет содержимое поля с номером `N` в диалоговом окне, заданном идентификатором `id`. Если номер поля не указан, обновляются все поля диалоговой панели.

### SetFocus

```rsl
SetFocus (id, N)
```

Процедура устанавливает фокус ввода для диалогового окна, заданного идентификатором `id`, в поле с номером `N`.

### MsgBox

```rsl
MsgBox (mes)
```

Процедура выводит на экран диалоговое окно с сообщением, заданным параметром `mes`. В качестве параметра могут быть переданы данные любого типа.

Для того чтобы сообщение занимало несколько строк, необходимо разделить его текст на части при помощи вертикальной черты `|`. Удаление сообщения с экрана производится при помощи клавиши `[Esc]`.

```rsl
MsgBox ("Это первая строка|Это вторая строка")
```

### DisableFields / EnableFields

```rsl
DisableFields (dlg [, fldNum])
EnableFields (dlg [, fldNum])
```

Процедуры делают поля диалоговой панели `dlg` нередактируемыми или отменяют запрет на редактирование. Если задан параметр `fldNum`, действие применяется только к полю с заданным номером.

### SetTimer

```rsl
SetTimer (dlg [, timeOut [, install]])
```

Процедура устанавливает таймер для диалоговой панели `dlg`. После установки таймера обработчику сообщений данной панели через одинаковые промежутки времени, заданные в миллисекундах параметром `timeOut`, будет посылаться сообщение `DLG_TIMER`. По умолчанию данный параметр принимается равным `3000` (3 секунды).

Чтобы отключить ранее установленный таймер, необходимо вызвать процедуру `SetTimer`, передав ей в качестве параметра `install` значение `FALSE`. В этом случае параметр `timeOut` игнорируется.

При успешном выполнении процедура возвращает `TRUE`, в противном случае — `FALSE`.

```rsl
record tm (timer) dialog;

macro MesProc (dlg, cmd, id)
    if (cmd == DLG_INIT)
        SetTimer (dlg, 1000) /* Устанавливаем таймер на 1 сек. */
    elif (cmd == DLG_TIMER)
        /* Увеличиваем счетчик и обновляем поля на экране */
        dlg.Counter = dlg.Counter + 1;
        UpdateFields (dlg)
    end;
    return CM_DEFAULT
end;

RunDialog (tm, @MesProc)
```

### MsgBoxEx

```rsl
MsgBoxEx (val: variant [, flags: integer, defInd: integer, title: string, statLn: string]): integer
```

Процедура выводит диалоговое окно с сообщением, задаваемым параметром `val`. Параметр `val` может иметь любой тип, он автоматически преобразуется в строку.

Параметр `flags` представляет собой сумму констант, определяющих набор кнопок:

| Константа | Описание |
|-----------|----------|
| `MB_OK` | Кнопка `"ОК"` |
| `MB_YES` | Кнопка `"Да"` |
| `MB_NO` | Кнопка `"Нет"` |
| `MB_CANCEL` | Кнопка `"Отмена"` |
| `MB_ERROR` | Цвет фона для сообщений об ошибках |

Если параметр равен `0`, в диалоговом окне не будет ни одной кнопки. Если параметр не задан, по умолчанию принимается `MB_OK`.

Параметр `defInd` определяет кнопку, выбранную по умолчанию:

| Значение | Кнопка |
|----------|--------|
| `IND_OK` | `"ОК"` |
| `IND_YES` | `"Да"` |
| `IND_NO` | `"Нет"` |
| `IND_CANCEL` | `"Отмена"` |

Процедура возвращает код выбранной кнопки. В случае ошибки выполнения процедура возвращает код `IND_ERROR`.

## Обработка скроллинга

Данный раздел содержит описание переменных и макропроцедур обработки скроллинга, определённых в макромодуле `RslScr`.

### Переменные

| Переменная | Описание |
|------------|----------|
| `LastUsed` | Индекс последней заполненной строки в скроллинге |
| `Nrec` | Количество строк в скроллинге |
| `Nfields` | Количество колонок в скроллинге |
| `Scrff` | Ссылка на скроллируемый файл |
| `Apos` | Массив физических адресов записей для каждой строки скроллинга |
| `DlgFields` | Массив имён колонок скроллинга |
| `FileFields` | Массив номеров полей в файле для каждой колонки скроллинга |

### Макропроцедуры

#### SetScroll

```rsl
SetScroll (FILE, { DlgName, FileName } …)
```

Данная процедура устанавливает соответствие имён полей в файле и в диалоговой панели. Эта процедура должна вызываться перед вызовом `RunDialog`.

| Параметр | Описание |
|----------|----------|
| `FILE` | Идентификатор файла, по которому будет производиться скроллирование |
| `DlgName` | Имя колонки в диалоговой панели |
| `FileName` | Имя поля в файле, данные из которого выводятся в колонку. Если имя совпадает с именем колонки, можно указать `NULL` |

Пары `DlgName – FileName` должны быть указаны для каждой колонки в скроллинге.

#### FindRow / FindCol

```rsl
FindRow (dlg, id): integer
FindCol (dlg, id): integer
```

Процедуры возвращают номер строки/колонки скроллинга в диалоге `dlg` для поля с номером `id`. Если поле не принадлежит области скроллинга, возвращается `-1`.

#### FillDown / FillUp

```rsl
FillDown (dlg, pos)
FillUp (dlg, pos)
```

Процедуры заполняют область скроллинга в диалоге `dlg` данными из файла. При `FillDown` первая строка скроллинга получает данные из записи файла с физическим адресом `pos`. При `FillUp` данные получает последняя строка скроллинга.

#### SetDlgFields

```rsl
SetDlgFields (dlg, row)
```

Процедура заполняет строку скроллинга с номером `row` данными из файла для диалога `dlg`.

#### ScrollMes

```rsl
ScrollMes (dlg, cmd, id, key)
```

Стандартная процедура для обработки сообщений диалога с областью скроллинга. Эта процедура должна вызываться для всех необработанных программой пользователя сообщений.

#### UserFill

```rsl
UserFill (dlg)
```

Эта процедура вызывается каждый раз перед перерисовкой области скроллинга. Оригинальная процедура не выполняет никаких действий. Она может быть заменена при помощи процедуры `ReplaceMacro` на процедуру, выполняющую нужные пользователю действия.

## Другие процедуры

### ExecMacro

```rsl
ExecMacro (string, .....)
```

Данная процедура вызывает для выполнения процедуру, имя которой содержится в первом параметре `string`. Это может быть символьная константа или переменная типа `V_STRING`. После первого параметра указываются значения, которые должны быть переданы вызываемой процедуре в качестве фактических параметров.

Если процедура с именем `string` существует, процедура `ExecMacro` возвращает значение `TRUE`, в противном случае — `FALSE`.

Процедуру можно вызвать и по ссылке на неё. В этом случае в качестве параметра `ExecMacro` принимает ссылку на макропроцедуру.

```rsl
macro MyMacro (parm)
    println ("Вызвана процедура MyMacro с параметром", parm)
end;

ExecMacro (@MyMacro, "Это параметр для MyMacro")
```

### ExecMacro2

```rsl
ExecMacro2 (string, .....)
```

Процедура эквивалентна процедуре `ExecMacro`, но возвращает значение, которое возвращает запускаемая процедура.

### ExecMacroFile

```rsl
ExecMacroFile (Module [, ProcName [, Parm1, Parm2, ...]])
```

Данная процедура производит компиляцию и выполнение макромодуля, заданного строкой `Module`. Строка `ProcName` задаёт имя макропроцедуры внутри модуля `Module`, которой передаётся управление. Список параметров `Parm1`, `Parm2` и т.д. передаётся процедуре `ProcName`.

Возвращаемое значение:
- Значение, возвращаемое процедурой `ProcName`.
- Если `ProcName` не задана — значение типа `V_UNDEF`.

**Пример:**

Модуль `add2.mac`:

```rsl
println ("This is add.mac");

macro MainProc
    println ("This is MainProc");
    i = 0;
    while (getParm (i, a))
        println (a);
        i = i + 1
    end;
    return "RetFrom MainProc"
end
```

Запуск из модуля `main.mac`:

```rsl
println ("This is main.mac");
println (ExecMacroFile ("add", "MainProc", 10, 20, "par"))
```

Результат выполнения:

```
This is main.mac
This is add.mac
This is MainProc
par
RetFrom MainProc
```

### ReplaceMacro

```rsl
ReplaceMacro (string1, [string2])
```

Данная процедура позволяет заменить процедуру с именем `string1` на глобальную процедуру с именем `string2`. После этого любой вызов процедуры `string1` будет перенаправляться в процедуру с именем `string2`.

Заменить можно не только макропроцедуру, но и любую стандартную процедуру, такую как `println`, `open`, `next`, `prev` и т.д.

Если заменяемая процедура вызывается из заменяющей, то перенаправление вызова не происходит: из заменяющей процедуры вызывается процедура предыдущего уровня замены. Одна процедура может быть заменена несколько раз.

Если параметр `string2` не задан, это приводит к снятию замены и восстановлению оригинальной процедуры. Если процедура с именем `string1` была заменена несколько раз, то для восстановления оригинальной процедуры необходимо столько же раз вызвать `ReplaceMacro` без параметра `string2`.

```rsl
/* Пример замены макропроцедур */
macro Test
    println ("Это Test")
end;

macro ReplTest
    println ("Это ReplTest");
    Test
end;

macro ReplReplTest
    println ("Это ReplReplTest");
    Test
end;

/* Выполняем первую замену процедуры Test */
ReplaceMacro ("Test", "ReplTest");
println ("Вызываем Test после выполнения первой замены...");
Test;

/* Выполняем вторую замену процедуры Test */
ReplaceMacro ("Test", "ReplReplTest");
println ("Вызываем Test после выполнения второй замены...");
Test;

println ("Отменяем одну замену...");
ReplaceMacro ("Test");
Test;

println ("Отменяем вторую замену...");
ReplaceMacro ("Test");
Test
```

Результат работы:

```
Вызываем Test после выполнения первой замены...
Это ReplTest
Это Test
Вызываем Test после выполнения второй замены...
Это ReplReplTest
Это ReplTest
Это Test
Отменяем одну замену...
Это ReplTest
Это Test
Отменяем вторую замену...
Это Test
```

### ExecExp

```rsl
ExecExp (string)
```

Процедура вычисляет выражение, заданное параметром `string`, и возвращает результат вычисления.

```rsl
Val = ExecExp ("10 + 20*5")
```

### Индикатор выполнения

#### InitProgress

```rsl
InitProgress (maxRecord [, msg, head])
```

Процедура инициализирует и выводит на экран индикатор выполнения циклического процесса. В первом параметре типа `V_INTEGER` необходимо указать максимальное значение, которое может принимать счётчик отображаемого цикла. Каждый раз при переходе к следующему значению счётчика закрашенная полоска индикатора увеличит свою длину.

Остальные параметры не являются обязательными и имеют тип `V_STRING`:

| Параметр | Описание |
|----------|----------|
| `msg` | Сообщение, выводимое в нижнюю строку экрана |
| `head` | Текст заголовка индикатора выполнения |

```rsl
InitProgress (1000);
var i = 0;
while (i < 1000)
    i = i + 1;
    UseProgress (i)
end;
RemProgress
```

#### UseProgress

```rsl
UseProgress (record)
```

Процедура приводит изображение на экране в соответствие текущему значению индикатора выполнения циклического процесса. Параметр `record` задаёт текущее значение счётчика цикла.

#### RemProgress

```rsl
RemProgress
```

Данная процедура удаляет с экрана окно индикатора выполнения, выведенное процедурой `InitProgress`.

### GetEnv

```rsl
GetEnv (string): string
```

Процедура возвращает значение переменной среды с именем, заданным в виде символьной строки.

```rsl
println (getenv ("PATH"))
```

Эту же процедуру можно выполнить проще:

```rsl
println (PATH)
```

так как если имя переменной RSL совпадает с именем переменной среды, то переменная RSL инициализируется значением переменной среды.

### RunError

```rsl

```

Процедура вызывает аварийное завершение выполнения RSL-программы с генерацией ошибки времени выполнения. Кроме этого, производится откат всех изменений, внесённых в файлы в рамках транзакции как на RSL, так и в вызывающем модуле на C или C++.

В качестве параметра процедуры используется символьная строка `ermes`, содержащая текст сообщения об ошибке.

### Exit

```rsl
Exit ([integer] [, string])
```

Данная процедура прекращает выполнение макропрограммы. Если эта процедура вызвана в рамках транзакции, то она завершает выполнение макросов, вызванных для выполнения транзакции, и актуализирует все изменения, внесённые в файлы в рамках транзакции как на RSL, так и в вызывающем модуле на C или C++. Процедура `ProcessTrn` при этом возвращает значение `TRUE`. Вся RSL-программа при этом не завершается.

| Параметр | Описание |
|----------|----------|
| `integer` | Режим просмотра результата: `1` — не показывать окно просмотра; `2` — сразу распечатать на принтере |
| `string` | Сообщение, которое выводится в выходной поток |

### IsStandAlone

```rsl
IsStandAlone: bool
```

Программный комплекс RS-Bank может работать в двух- либо трёхуровневой архитектуре "клиент-сервер". Процедура `IsStandAlone` позволяет определить, в каком именно режиме функционирует приложение.

Возвращаемые значения:

| Значение | Режим |
|----------|-------|
| `TRUE` | Двухуровневая архитектура |
| `FALSE` | Трёхуровневая архитектура |

```rsl
if (IsStandAlone)
    println ("StandAlone")
else
    println ("Communication")
end
```

### TestEvent

```rsl
TestEvent ([pause])
```

Процедура позволяет узнать, была ли нажата клавиша на клавиатуре или кнопка "мыши", и возвращает:

| Событие | Возвращаемое значение |
|---------|----------------------|
| Нажатие произведено | Код нажатой клавиши |
| Нажатий не было | `0` |

Необязательный параметр `pause` используется для 32-разрядных программ и определяет время в миллисекундах, по прошествии которого процедура вернёт ноль, если нажатий не произошло. Если до истечения заданного интервала времени будет произведено движение мышью, процедура выйдет из цикла ожидания раньше времени и вернёт ноль.

Если в качестве данного параметра указать значение `-1`, процедура будет дожидаться нажатия кнопки "мыши" или любой клавиши клавиатуры. По умолчанию параметр `pause` принимается равным нулю.

```rsl
File ff (account);
Const ESC = 27;
Var Done = FALSE;

while (not Done && next (ff))
    println (ff.account);
    if (TestEvent == Esc)
        Done = TRUE
    end
end
```

## Средство разработки расширений для языка RSL (DLM SDK)

Язык RSL является средством настройки программных комплексов, разрабатываемых компанией R-Style Softlab, под конкретные требования пользователей. Он позволяет создавать макропрограммы, которые формируют нестандартные отчёты, проводят анализ или диагностику базы данных, выполняют другие функции. Встречаются ситуации, когда для решения поставленных задач стандартных средств языка не хватает. Для устранения данной проблемы был создан инструмент DLM SDK.

DLM SDK предназначен для разработки на языке C/C++ динамически загружаемых модулей. Использование данного инструмента позволяет дополнить RSL новыми эффективными процедурами и даже классами объектов, используя для этого всю мощь компилируемых языков С/С++.

В состав дистрибутивного комплекта DLM SDK входят библиотеки и заголовочные файлы для разработки расширений.
