# Reporting RSLprc

Введение
Настоящее Руководство содержит описание переменных, классов, процедур и констант
языка интерпретатора RSL, которые используются при создании макромодулей АС RS-
Reporting V.6, входящей в ИБС RS-Bank V.6, и при написании пользователем собственных
макропрограмм.
Информация об общесистемных интерфейсах RSL содержится в следующих Руководствах:
· Руководство программиста "Интерфейсы языка RSL для взаимодействия с ИБС RS-Bank
V.6. Часть 1" (файл Books\Tools\CoreRSLprc_1.pdf) – содержит описание общесистемных
спецпеременных 
и 
модулей 
BalanceInter,

cryptdlm.d32, CTInter.
· Руководство программиста "Интерфейсы языка RSL для взаимодействия с ИБС RS-Bank
V.6. Часть 2" (файл Books\Tools\CoreRSLprc_2.pdf) – содержит описание модулей
CurrInter, FIInter, GateInter, InsCarryDoc, OprInter, PcRateInter, PTInter, RsbDataSet,
RsbObjFactory, RsSysLog, SFInter.
· Руководство программиста "Интерфейсы языка RSL для взаимодействия с ИБС RS-Bank

Определение периода работы
Настоящие спецпеременные первоначально имеют в макрофайле значения, заданные в
режиме "Задание рабочего диапазона дат". Изменения значений этих переменных,
сделанные при работе макрофайла, не отразятся на значениях периода в самой программе
и будут потеряны при запуске других макрофайлов. Сделать изменения глобальными можно
при помощи процедуры ИнфоЗадатьПодсистемеГлоб
.
ДатаОтчета – дата окончания текущего отчетного периода.
ПредДатаОтчета – дата начала текущего отчётного периода.
ДатаОтчетаПП – дата окончания предыдущего отчетного периода.
ПредДатаОтчетаПП – дата начала предыдущего отчётного периода.
ВидПериода – возможные значения переменной: год, квартал, месяц, день, период.
Накопительный – переменная, которая характеризует период как накопительный или
наоборот. Для накопительного отчёта дата начала периода всегда равна 1 января
указанного года.
Пример.

 elif ( ВидПериода == 2 ) /* Квартал */
   НазваниеВидаПериода = string( (m–1)/3 + 1, "–Й
КВАРТАЛ");
 elif ( ВидПериода == 3 ) /* Месяц */

     string( "период с ", ПредДатаОтчета, " по ",
ДатаОтчета);

 return НазваниеВидаПериода;
END;

Спецпеременные
НомерПодразделения – номер подразделения, которое было установлено в качестве

Определение имени отчетной формы,
имени и вида переменной
Спецпеременные, описанные в данном разделе, определяют имя отчетной формы, имя и
вид переменной, в контексте которых вызвана макропрограмма. Эти переменные
используются в макропрограммах, выполняемых в качестве шагов операций.
{Название отчета} – название отчетной формы, в контексте которой выполняется
макропрограмма.
{Имя переменной} – имя переменной, в контексте которой вызывается макропрограмма.
Переменная используется при рекурсивном расчете или в случае, когда операция
была запущена из списка переменных. В последнем случае переменная
инициализируется именем той переменной, которая в этот момент была активной
в справочнике форм.
{Вид переменной} – вид переменной. Эта переменная инициализируется видом
переменных, в контексте которого выполняется макропрограмма.

## Класс: `RcbApplication`

```rsl
RcbApplication()
```

Класс "Приложение", представляет собой подсистему "Регламентируемая отчетность".
Через объект класса осуществляется доступ ко всем глобализмам подсистемы. К таким,
например, относятся: параметры подсистемы, текущий отчет, с которым идет работа,
фабрика объектов и т.д.

**Примечание:**

В системе присутствует только один объект указанного класса, который
создается при входе в подсистему и разрушается при выходе из нее.

**Свойства:**

currentReport – текущий отчет. Если работа производится в контексте отчета, указанное
свойство содержит объект класса RcbReport
, являющийся текущим отчетом.
В противном случае свойство принимает значение Null. Свойство доступно
только для чтения и имеет тип Object.

**Пример:**

import RcbCoreInter;
var currentReport = RcbApplication().currentReport;
if (currentReport == NULL)
   MsgBox("Работа вне контекста отчета");
else
   MsgBox(currentReport.form.name);
end;
objectFactory – свойство создает объекты подсистемы. Свойство имеет тип Object,
представляет собой объект класса RcbObjectFactory
 и доступно только для
чтения.

**Пример:**

import RcbCoreInter;
var 
form 
=
RcbApplication().objectFactory.CreateForm("Балансовые
счета");
parameters – параметры приложения. Свойство имеет тип Object, представляет собой
объект класса RcbApplicationParameters
 и доступно только для чтения.
summator – сумматор, выполняющий сведение счетов. Свойство имеет тип Object и
представляет собой объект класса RcbSummator
.
transactionManager – менеджер транзакции; позволяет сохранить изменения в БД или
"откатить" их. Свойство имеет тип Object, представляет собой объект класса
RcbTransactionManager
 и доступно только для чтения.

**Пример:**

import RcbCoreInter;
var currentReport = RcbApplication().currentReport;
currentReport.attributeValue("Бн10201РуП").Reset();
RcbApplication().transactionManager.Commit();
Exit(0);
OnError(err);
RcbApplication().transactionManager.Rollback();
Exit(1);

## Класс: `RcbAttribute`

```rsl
RcbAttribute()
```

Объект класса представляет собой один атрибут формы. Атрибутом может являться,
например, какое-либо поле отчета, ячейка таблицы и т.д. Кроме того, атрибутом может
быть и какой-нибудь скрытый (невидимый в печатной форме) и вспомогательный
показатель формы.

**Примечание:**

Объект данного класса нельзя явно создавать в RSL.

**Свойства:**

activePeriod – период, в течение которого атрибут входит в состав формы. Свойство
имеет тип Object и представляет собой объект класса RcbPeriod
.
form – форма, которой принадлежит атрибут. Свойство имеет тип Object и представляет
собой объект класса RcbForm
.

**Пример:**

import RcbCoreInter;
var
