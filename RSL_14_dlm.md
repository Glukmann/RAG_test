## Общее описание

DLM SDK (Dynamic Loadable Module SDK) — инструмент для создания двоичных DLM-модулей, которые загружаются в память динамически во время выполнения RSL-программы.

В состав DLM SDK входит:

- библиотеки;
- заголовочные файлы;
- документация;
- примеры использования.

В данном разделе представлено описание наиболее простых примеров создания DLM-модулей. Полная документация с примерами приведена в дистрибутиве.

---

## Создание и использование DLM-модулей

Двоичные DLM-модули, созданные при помощи DLM SDK, загружаются в память динамически во время выполнения RSL-программы.

### Поддерживаемые платформы

| Платформа | Описание |
|-----------|----------|
| `DOS Real Mode` | Реальный режим DOS |
| `DPMI16` | Защищённый режим DOS 16-разрядный |
| `DPMI32` | Защищённый режим DOS 32-разрядный |
| `WIN32` | Windows 32-разрядная |

### Требования к компиляторам

| Платформа | Компилятор |
|-----------|------------|
| DOS Real Mode | Borland C/C++ 3.0 или выше |
| DPMI16 | Borland C/C++ 4.02 или выше + Power Pack for DOS |
| DPMI32 / WIN32 | Borland C/C++ 4.0 или выше, либо Microsoft Visual C++ 5.0 |

### Типы модулей

| Платформа | Тип модуля |
|-----------|------------|
| DOS Real Mode | Специального вида EXE-файл |
| DPMI16 | 16-разрядная DLL |
| DPMI32 / WIN32 | 32-разрядная DLL |

Для каждой платформы, на которых может работать интерпретатор RSL, требуется своя версия DLM-модуля.

### Соглашение об именах файлов

| Платформа | Расширение |
|-----------|------------|
| DOS Real Mode | `.dlm` |
| DPMI16 | `.d16` |
| DPMI32 / WIN32 | `.d32` |

Имя файла должно удовлетворять требованиям операционной системы (желательно не более восьми символов для совместимости с реальным режимом DOS).

### Подключение DLM-модуля к RSL-программе

Для того чтобы DLM-модуль стал доступен RSL-программе, имя модуля без расширения необходимо указать в директиве `IMPORT`:

```rsl
import demo    /* Подключение файла, содержащего код DLM-модуля */
```

## Пример простого DLM-модуля

Рассмотрим DLM-модуль, который реализует для языка RSL процедуру с именем `RslHello`. Данная процедура выводит в стандартный выходной поток текстовую строку `"Вызвана процедура RslHello"`.

```c
#include "rsl\rsldll.h"

static void DLMAPIC RslHello (void)
{
    print ("Вызвана процедура RslHello\r\n");
}

void DLMAPI EXP AddModuleObjects (void)
{
    AddStdProc (V_UNDEF, "RslHello", RslHello, 0);
}
```

### Разбор примера

1. К программе подключается файл `rsldll.h`, содержащий описание интерфейса с интерпретатором RSL.
2. Вводится определение процедуры `RslHello`, которая будет вызываться из RSL-программы.

**Важно:** все процедуры, вызываемые из макропрограмм на языке RSL, должны иметь прототип, как приведённая `RslHello`: не принимать параметров, не возвращать значений и иметь тип вызова `DLMAPIC`.

Для передачи параметров и возвращения значений предусмотрен специальный механизм, который подробно рассматривается в разделе «Передача параметров и возврат значений».

### Регистрация процедуры

После создания процедуры для RSL её необходимо зарегистрировать при помощи стандартной функции `AddStdProc`:

```c
AddStdProc (type, name, func, reserved)
```

| Параметр | Описание |
|----------|----------|
| `type` | Тип RSL-переменной, возвращаемой процедурой |
| `name` | Имя процедуры, используемое для вызова из RSL |
| `func` | Указатель на функцию, реализующую алгоритм процедуры |
| `reserved` | Зарезервированный параметр, всегда должен быть равен `0` |

Функция `AddStdProc` вызывается для каждой регистрируемой процедуры из функции `AddModuleObjects`, которую необходимо определять аналогично примеру. Интерпретатор RSL вызывает функцию `AddModuleObjects` автоматически при загрузке DLM-модуля.

### Компиляция модуля

Предположим, что исходный текст DLM-модуля находится в файле `demo.c`. Для компиляции модуля для платформы DPMI32 выполните команду:

```bash
bcc32 -WXD -Ih -edemo.32 demo.c lib\rsldlm32.lib dpmi32x.lib
```

В результате компиляции будет получен модуль `demo.d32`.

### Использование модуля в RSL

```rsl
import demo;
RslHello
```

## Передача параметров и возврат значений

Для доступа к переданным в макропроцедуру параметрам применяется функция `GetParm`:

```c
bool GetParm (int index, VALUE **value)
```

| Аргумент | Описание |
|----------|----------|
| `index` | Номер параметра, значение которого необходимо получить |
| `value` | Возвращает ссылку на значение RSL-переменной |

Если запрашиваемый параметр не существует, функция возвращает значение `FALSE`. В этом случае второй аргумент инициализируется ссылкой на временное значение неопределённого типа `V_UNDEF`.

Переменная RSL описывается структурой `VALUE`. В поле `v_type` этой структуры хранится тип переменной, а в поле `value` — собственно значение переменной для конкретного типа данных.

### Возврат значений

Для возврата значений в RSL-программу служит функция `ReturnVal`:

```c
void ReturnVal (int type, void *value)
```

| Параметр | Описание |
|----------|----------|
| `type` | Тип возвращаемого значения |
| `value` | Указатель на возвращаемое значение |

### Пример: процедура MultiStrLen

Разработаем процедуру `MultiStrLen`, которая принимает в качестве параметров произвольное количество строк и возвращает их суммарную длину:

```c
#include "rsl\rsldll.h"
#include <string.h>

static void DLMAPIC MultiStrLen (void)
{
    VALUE *v;
    int i = 0;
    long len = 0;

    for (i = 0; GetParm (i, &v); ++i)
        if (v->v_type == V_STRING)
            len += strlen (v->value.string);

    ReturnVal (V_INTEGER, &len);
}

void DLMAPI EXP AddModuleObjects (void)
{
    AddStdProc (V_INTEGER, "MultiStrLen", MultiStrLen, 0);
}
```

### Изменение фактических параметров

Все параметры в RSL-процедуры передаются по значению. Однако, используя стандартную функцию `PutParm`, можно изменить значение фактического параметра в программе, вызвавшей данную процедуру.

```c
static void DLMAPIC RetDemo (void)
{
    long val = 200;
    PutParm (0, V_INTEGER, &val);
}
```

Функция `PutParm` отличается от функции `ReturnVal` дополнительным первым аргументом, который задаёт номер устанавливаемого параметра.

## Инициализация и деинициализация модуля

Имеется возможность инициализации и деинициализации DLM-модуля. Для этого служат две специальные функции:

| Функция | Вызов | Назначение |
|---------|-------|------------|
| `InitExec` | Автоматически перед началом выполнения RSL-программы | Инициализация модуля |
| `DoneExec` | Автоматически после выполнения RSL-программы | Освобождение занятых ресурсов |

```c
void DLMAPI EXP InitExec (void)
{
    /* Выполняем инициализацию */
}

void DLMAPI EXP DoneExec (void)
{
    /* Выполняем освобождение занятых ресурсов */
}
```

## Создание классов объектов

Помимо процедур, вызываемых из RSL-программ, при помощи инструмента DLM SDK можно создавать целые классы объектов со свойствами и методами. Объекты классов в RSL создаются и уничтожаются с помощью механизма поставщиков объектов.

В DLM-модуль можно включить код, реализующий поставщик объектов требуемого типа, и, используя функцию `AddModuleObjects`, зарегистрировать этого поставщика. Детальная информация по созданию классов приведена в полной документации DLM SDK.
