## Объектные типы данных

| Тип | Описание |
|-----|----------|
| `RECORD` | Хранит информацию о структуре файла или диалоговой панели. В отличие от `FILE`, не имеет явной связи с файлом (связь задаётся процедурами RSL или неявно в вызывающем модуле на C/C++) |
| `DBFFILE` | Хранит информацию о файле формата DBF (структуру и текущие значения полей) |
| `TXTFILE` | Хранит информацию о текстовом файле (структуру и текущие значения полей) |
| `ARRAY` | Вектор переменных, который можно индексировать |
| `CLASS` | Совокупность свойств и методов; служит для создания экземпляра класса — объекта |

Все объекты RSL, кроме переменных, требуют явного определения их имени. Имена переменных по умолчанию считаются объявленными в момент их первого появления в тексте программы. Однако допускается явное объявление переменных при помощи `VAR`.

## Скалярные типы данных

Под типом понимается совокупность информации, определяющей данное: сколько памяти нужно выделить, какой вид имеет его представление, какие операции над ним определены.

В Object RSL поддерживаются следующие скалярные типы данных:

| Тип | Описание |
|-----|----------|
| `Variant` | Переменная может принимать значения любых типов. Используется по умолчанию |
| `Integer` | Четырёхбайтовое целое число |
| `Money` | Восьмибайтовое число для денежных величин |
| `Double` | Восьмибайтовое число с плавающей точкой |
| `DoubleL` | Десятибайтовое число с плавающей точкой |
| `String` | Строка символов |
| `Bool` | Логическая величина (`TRUE` / `FALSE`) |
| `Date` | Дата |
| `Time` | Время |
| `Dttm` | Дата/время |
| `Object` | Ссылка на объект |
| `ProcRef` | Ссылка на макропроцедуру |

Переменная получает или изменяет тип автоматически в момент присваивания ей значения, то есть когда она присутствует в левой части операции присваивания.

## Коды типов данных (ValType)

Тип значения переменной можно определить с помощью процедуры `ValType`, которая возвращает код типа данных:

| Код | Тип данных | Описание |
|-----|------------|----------|
| `V_INTEGER` | Integer | Целые числа со знаком длиной 4 байта |
| `V_STRING` | String | Символьные строки |
| `V_DOUBLE` | Double | Числа с плавающей точкой длиной 8 байт |
| `V_DOUBLEL` | DoubleL | Числа с плавающей точкой длиной 10 байт. При задании констант указывается `L`, например `1234.567L` |
| `V_BOOL` | Bool | Логические переменные |
| `V_MONEY` | Money | Денежные суммы. Внутреннее представление — `V_DOUBLE` |
| `V_DATE` | Date | Даты в формате ДД.ММ.ГГГГ |
| `V_TIME` | Time | Время в формате ЧЧ:ММ:СС |
| `V_DTTM` | Dttm | Дата и время совместно |
| `V_UNDEF` | — | Неопределённое значение |
| `V_FILE` | File | Ссылка на файл базы данных |
| `V_STRUC` | Record | Ссылка на структуру из словаря базы данных |
| `V_ARRAY` | Array | Ссылка на массив |
| `V_TXTFILE` | TxtFile | Ссылка на текстовый файл |
| `V_DBFFILE` | DbfFile | Ссылка на файл формата DBF |
| `V_GENOBJ` | Object | Ссылка на объект класса |
| `V_PROC` | ProcRef | Ссылка на процедуру |

Если переменной не присвоено никакого значения, в том числе при явном определении её имени, она имеет неопределённый тип (`V_UNDEF`).

При использовании структуры, описанной в словаре, все её поля получают те типы, которые были заданы в словаре.

## Явное декларирование типов

Кроме автоматического определения типа, в Object RSL имеется возможность явно декларировать типы:

- переменных,
- параметров макропроцедур и методов,
- возвращаемых значений макропроцедур и методов.

Декларируемый тип переменной или формального параметра указывается через двоеточие после объявления:

```rsl
Var Имя: string;
```

Тип возвращаемого значения макропроцедуры или метода класса указывается после двоеточия, которое следует за списком формальных параметров (или непосредственно после имени, если параметров нет):

```rsl
Macro MyStrLen (str: string): integer
    Return strlen (str);
End;
```

**Ограничение:** переменные, являющиеся ссылками на файл или структуру, нельзя явно декларировать.

Для декларирования типов данных можно использовать имена классов:

```rsl
Var obj: MyClass;
```

**Примечание:** декларирование типов не приводит к увеличению производительности RSL-программы.
