## Продолжение примера скроллинга с отбором записей

```rsl
/* Выполняем замену стандартных процедур next, prev, rewind */
ReplaceMacro ("next", "MyNext");
ReplaceMacro ("prev", "MyPrev");
ReplaceMacro ("rewind", "MyRewind");
ReplaceMacro ("UserFill", "MyFill");

record dlg (blnc) dialog;    /* Основной диалог */
record rb (RBdlg) dialog;    /* Панель ввода интервала счетов */

const F4 = 318, F5 = 319;

macro MyMesProc (dl, cmd, id, key)
    var row;
    if (cmd == DLG_KEY)
        if (key == F4)
            /* Сделаем выбранную запись текущей в файле */
            row = FindRow (dl, id);
            if ((row != -1) and GetDirect (Scrff, Apos (row)))
                MsgBox ("Выбран счет " + Scrff.Balance);
            end;
            return CM_IGNORE
        elif (key == F5)
            /* Запрос нового интервала счетов */
            rb.fromB = fromB;
            rb.toB = toB;
            if (RunDialog (rb))
                fromB = rb.fromB;
                toB = rb.toB;
                /* Выводим на экран новый интервал счетов */
                rewind (Scrff);
                if (next (Scrff))
                    FillDown (dl, GetPos (Scrff))
                end;
            end;
        end;
    end;
    return ScrollMes (dl, cmd, id, key)
end;

SetScroll (ff,
    "Blnc", "Balance",
    "NameB", "Name_Part"
);

dlg.Comment = "Тест прокрутки";
dlg.CurDate = date;

if (RunDialog (dlg, "MyMesProc"))
    println (ff.Balance, " ", ff.Name_Part);
    println (dlg.Comment, " ", dlg.Curdate)
end;
```

## Способы работы с файлами

Для работы с файлами в Object RSL предусмотрены следующие конструкции:

| Конструкция/класс | Описание |
|-------------------|----------|
| `FILE` | Стандартная конструкция языка |
| `Tbfile` | Стандартный класс для работы с Btrieve-файлами |
| `TRecHandler` | Стандартный класс для работы со структурами записей |

**Примечание:** при работе с Btrieve-файлами предпочтительнее использовать классы `Tbfile` и `TRecHandler`. Конструкция `FILE` поддерживается для совместимости с ранее разработанными программами.

## Стандартный класс Tbfile

Стандартный класс `Tbfile` предназначен для работы с Btrieve-файлами и представляет собой объектную альтернативу стандартной конструкции языка `FILE`.

### Конструктор

```rsl
Tbfile ( TableName [, AttrStr] [, KeyNum] [, FileName] [, DicName] )
```

| Параметр | Описание |
|----------|----------|
| `TableName` | Имя файла в словаре базы данных |
| `AttrStr` | Строка атрибутов открытия файла (см. ниже) |
| `KeyNum` | Текущий индекс в Btrieve-файле. Если не указан — `0`. Для физической последовательности укажите `-1` |
| `FileName` | Имя файла на диске, если отличается от имени в словаре |
| `DicName` | Имя словаря базы данных, если отличается от используемого по умолчанию |

### Атрибуты открытия файла (AttrStr)

| Атрибут | Описание |
|---------|----------|
| `R` | Только для чтения (по умолчанию) |
| `W` | Для чтения и записи |
| `E` | Для чтения и записи в монопольном режиме |
| `A` | Ускоренный режим (только для временных файлов) |
| `C` | Открыть файл; если не существует — создать |
| `C+` | Создать файл; если существует — заместить |
| `B` | Файл с переменной частью записи, используемой как поле типа BLOB |

Если параметр `AttrStr` не задан, файл открывается только для чтения.

### Примеры создания Tbfile

```rsl
/* Файл account.dbt открывается только для чтения по нулевому индексу */
Ob = Tbfile ("Account");

/* Файл account.dbt открывается для записи по второму индексу */
Ob = Tbfile ("Account", "W", 2);
```

### Методы класса Tbfile

Все методы объекта класса `Tbfile` аналогичны процедурам для работы с Btrieve-файлами:

`next`, `prev`, `getpos`, `getdirect`, `insert`, `update`, `delete`, `getEQ`, `getGE`, `getLE`, `getGT`, `getLT`, `rewind`, `fldnumber`, `fldindex`, `fldname`, `fldoffset`, `nrecords`, `RecSize`, `VarSize`, `Clear`, `PackVarBuff`, `UnPackVarBuff`, `WriteBlob`, `ReadBlob`, `FileName`.

### Свойства класса Tbfile

| Свойство | Описание |
|----------|----------|
| `Rec` | Ссылка на объект типа `RECORD` для доступа к полям записи |
| `KeyNum` | Текущий индекс в Btrieve-файле |
| `TblName` | Имя Btrieve-файла |
| `DicName` | Имя словаря базы данных |

### Пример доступа к полям записи

```rsl
Ob.rec.Account = "711034";
Ob.rec.Rest = $12345.67;
```

### Свойство по умолчанию item

Объекты класса `Tbfile` имеют свойство по умолчанию `item`. Параметр этого свойства указывает имя поля или номер поля записи Btrieve-файла.

```rsl
Ob = Tbfile ("account");
Ob.item ("Sum") = $123.45;
Ob.item (1) = "Comment";

/* Так как item — свойство по умолчанию, можно записать: */
Ob. ("Sum") = $123.45;
Ob. (1) = "Comment";
```

## Стандартный класс TRecHandler

Стандартный класс `TRecHandler` представляет собой объектную альтернативу стандартной конструкции языка `RECORD`.

### Конструктор

```rsl
TRecHandler ( strName [, altDic [, file [, index [, offset [, isFixed ] ] ] ] ] )
```

| Параметр | Описание |
|----------|----------|
| `strName` | Имя структуры Btrieve-файла в словаре базы данных |
| `altDic` | Имя альтернативного словаря. Если `NULL` — используется словарь по умолчанию |
| `file` | Объект типа `Tbfile`. Если используется для работы с диалоговыми панелями — `TRUE` |
| `index`, `offset` | Смещение для наложения структуры на запись файла. Общее смещение вычисляется: `размер_структуры * index + offset` |
| `isFixed` | `TRUE` — смещение относительно начала записи; `FALSE` или не указан — относительно начала переменной части |

В отличие от класса `Tbfile`, класс `TRecHandler` не связан непосредственно с Btrieve-файлом, однако его объекты могут использоваться для хранения в памяти копии записи из Btrieve-файла.

### Пример использования TRecHandler

```rsl
/* Создаётся экземпляр класса Tbfile */
Ob = Tbfile ("account");

/* Выполняется обращение к методу next */
Ob.next;

/* Создаётся экземпляр класса TRecHandler */
Rec = TRecHandler ("account");

/* Создаётся копия записи файла account.dbt */
Copy (rec, ob);
```

### Наложение структуры на запись

Объекты `TRecHandler` могут использоваться совместно с объектами `Tbfile`. Можно "наложить" объект класса `TRecHandler` на постоянную или переменную часть записи Btrieve-файла. Наложение структуры предоставляет возможность считывать данные из записи файла, а также модифицировать его данные непосредственно в Btrieve-файле посредством обращения к полям структуры.

### Метод SetRecordAddr

```rsl
SetRecordAddr ( file, ind, offs, isFix )
```

| Параметр | Описание |
|----------|----------|
| `file` | Ссылка на объект `Tbfile` или его наследника |
| `ind`, `offs`, `isFix` | Аналогичны параметрам конструктора |

В результате использования этого метода все функции, работающие с конструкцией `FILE` (`next`, `prev`, `SetRecordAddr` и т.д.), могут принимать в качестве параметра ссылку на объект `Tbfile` или его наследника. Функции `fldname`, `fldindex` и др., принимающие ссылку на конструкцию `RECORD`, могут принимать ссылку на объект `TRecHandler` или его наследника.

```rsl
class (TBfile) MyTb
    InitTBfile ("client.dbt", "w", 0);
end;
ob = MyTb;

class (TRecHandler) MyRec
    InitTRecHandler ("client.dbt");
end;
obr = MyRec;

obr.SetRecordAddr (ob, 0, 0, true);
ob.next;
println (obr.rec.name_client);
```

## Использование конструкции FILE

Для выполнения действий с Btrieve-файлом необходимо определить его в RSL-программе с помощью конструкции `FILE`.

### Пример открытия файла только для чтения

```rsl
FILE aa (account);
```

Данная строка определяет идентификатор `aa` для Btrieve-файла с именем `account.dbt`, доступного только для чтения. Расширение файла `.dbt` принимается по умолчанию.

### Пример открытия файла для записи

```rsl
FILE aa (account) write;
```

### Поиск записей по ключу

Для Btrieve-файлов возможен поиск записей по ключу при помощи процедур:

| Процедура | Описание |
|-----------|----------|
| `getEQ` | Равно указанному значению |
| `getGT` | Строго больше указанного |
| `getGE` | Больше или равно указанному |
| `getLT` | Строго меньше указанного |
| `getLE` | Меньше или равно указанному |

Чтобы задать значение ключа для поиска, необходимо присвоить нужные значения тем полям записи файла, которые соответствуют сегментам текущего ключа, перед вызовом этих процедур.

Чтобы определить номер ключа, по которому открывается файл, необходимо при определении файла указать спецификатор `SORT`. По умолчанию используется ключ с номером `0`. Чтобы работать с файлом в физической последовательности, необходимо указать любое значение ключа, большее `999`.

### Пример выборки записей по ключу

```rsl
/* Отбор записей по ключу.
   Предполагается, что файл arhdoc.dbt имеет ключ с номером 0 по полю Date_Carry.
   Программа выбирает записи, попадающие в заданный диапазон дат. */

file from (arhdoc) sort 0;
var fromdate,      /* Минимальное значение диапазона */
    todate;        /* Максимальное значение диапазона */
var first = TRUE;

macro getarh (ff)
    macro Test    /* Проверка на превышение максимального значения */
        if (ff.Date_Carry > todate)
            first = TRUE;
            return FALSE;
        end;
        return TRUE;
    end;
    if (first)
        first = FALSE;
        /* Установка минимального значения для поиска по ключу первой записи */
        ff.Date_Carry = fromdate;
        if (getGE (ff)) return Test; end;
    else
        if (next (ff)) return Test; end;
    end;
    return FALSE;
end;

/* Установка минимального и максимального значения для отбора */
fromdate = date (1, 1, 1994);
todate = date (1, 4, 1994);

var ndoc = 0;
while (getarh (from))
    ndoc = ndoc + 1;
    message (from.Date_Carry, " ", ndoc);
end;
println (ndoc);

/* Задаём новый интервал */
fromdate = date (1, 6, 1994);
todate = date (1, 7, 1994);

/* Обрабатываем новую выборку */
while (getarh (from))
    message (from.Date_Carry, " ", ndoc);
end;
```

## Работа с Btrieve-файлами переменной длины

В RSL предусмотрено два способа организации доступа к Btrieve-файлам, имеющим переменную длину записи.

### Первый способ

При данном способе размер переменной части записи ограничивается пределом **64 Kb**.

Переменная часть записи считывается в буфер памяти вместе с постоянной частью. Доступ к данным переменной части осуществляется посредством наложения определённой пользователем структуры на переменную часть записи. Для этого используется процедура `SetRecordAddr`.

Процедура `SetRecordAddr` позволяет наложить пользовательскую структуру и на постоянную часть записи. Это даёт возможность иметь в одном файле записи с разной структурой, но для этого необходимо предусмотреть какой-либо способ идентификации типа записи.

Размер переменной части записи 64Kb соответствует её теоретическому пределу. Как правило, процедуры системы RS-Bank работают непосредственно с записями такого размера, но иногда размер переменной части записи необходимо уменьшить. С этой целью её "упаковывают". Для работы с "упакованными" записями в RSL предусмотрены две процедуры:

- `PackVarBuff`
- `UnPackVarBuff`

### Второй способ

При данном способе переменная часть записи рассматривается как специальное поле — **BLOB** (двоичный большой объект), которое не имеет ограничений на размер записываемой информации.

Для работы с полем типа BLOB необходимо создать Btrieve-файл версии 6.x с переменной частью записи. При описании идентификатора файла используется модификатор `BLOB`:

```rsl
FILE ff (Proba) blob
```

Обычные операции чтения и записи файла распространяются только на постоянную часть, а для доступа к полю BLOB применяются специальные процедуры:

- `WriteBlob`
- `ReadBlob`

## Обработка ошибок

Все процедуры, работающие с Btrieve-файлами, при возникновении критических ошибок аварийно завершают выполнение RSL-программы. В остальных случаях они возвращают статус завершения:

- `TRUE` — в случае успешного завершения;
- `FALSE` — в случае неуспешного завершения.

Если процедура завершилась неуспешно, для Btrieve-файлов можно использовать процедуру `status` для получения информации об ошибке:

```rsl
var ercode, ertext;
if (not next (ff))
    ercode = status (ertext);
    println ("Ошибка (", ercode, "): ", ertext)
end
```

### Процедура BtrError

Другой способ обработки ошибок Btrieve-операций предполагает определение в тексте программы процедуры с именем `BtrError`:

```rsl
macro BtrError (oper, kod, name)
```

Данная процедура вызывается неявно в случае возникновения ошибки, связанной с выполнением Btrieve-операций. Процедура вызывается для всех ошибок, за исключением ошибки "Конец файла" (`BEeof`).

| Параметр | Описание |
|----------|----------|
| `oper` | Код операции Btrieve |
| `kod` | Код ошибки |
| `name` | Имя Btrieve-файла, при обработке которого произошла ошибка |

Процедура должна возвращать значение типа `V_INTEGER`. От возвращаемого значения зависит дальнейшая работа системной процедуры, вызвавшей ошибку:

| Возвращаемое значение | Действие |
|-----------------------|----------|
| `0` | Процедура вернёт `FALSE` (действие по умолчанию) |
| `1` | Ошибка игнорируется, процедура вернёт `TRUE` |
| `2` | Процедура вернёт `FALSE`, на экран выводится сообщение об ошибке |
| `3` | Процедура вернёт `FALSE`, выполнение программы прерывается |

### Конструкция OnError

В RSL предусмотрена специальная конструкция `OnError` для обработки ошибок, которая позволяет не допустить аварийного завершения RSL-программы в случае возникновения ошибки времени выполнения.

```rsl
OnError ['(' ссылка_на_объект_класса_TrslError ')']
    <инструкции для обработки ошибок>
```

Если любая инструкция в теле макропроцедуры генерирует ошибку времени выполнения, аварийного завершения программы не происходит, а выполняются инструкции для обработки ошибок, указанные после ключевого слова `OnError`.

Если в обработчике ошибок нужна информация о случившейся ошибке, то в качестве параметра `OnError` необходимо указать имя переменной, которая после возникновения ошибки получит ссылку на специальный объект класса `TrslError`.

**Примечание:** переменная, которая в случае сбоя получает ссылку на объект класса `TrslError`, не должна декларироваться в теле макропроцедуры при помощи ключевого слова `var`.

```rsl
OnError (er)
    MsgBox (er.Message, "| Модуль:", er.Module, "| строка:", er.Line);
```

### Свойства объекта TrslError

| Свойство | Описание |
|----------|----------|
| `Code` | Код ошибки |
| `Message` | Строка, описывающая ошибку |
| `Module` | Название модуля RSL, вызвавшего ошибку |
| `Line` | Номер строки модуля, в которой произошла ошибка |

### Процедура RunError

Чтобы в обработчике принудительно сгенерировать некоторую ошибку для передачи её другому обработчику в цепочке вызова макропроцедур или обработчику RSL по умолчанию, необходимо вызвать стандартную процедуру `RunError` без параметров.

## Обработка транзакций

Обработка транзакций в RSL распределена между макросом, написанным пользователем на языке RSL, и вызывающим модулем, написанным на C или C++.

Для выполнения транзакции на языке RSL служат процедуры `ProcessTrn`, `ProcessConTrn` и `LoopInTrn`.

| Процедура | Описание |
|-----------|----------|
| `ProcessTrn` | Выполняет простую транзакцию |
| `ProcessConTrn` | Выполняет конкурентную транзакцию |

Если во время выполнения транзакции файл или запись оказываются захваченными другим пользователем, RSL автоматически выполняет серию повторов операции через определённые промежутки времени. Количество повторов и время задержки задаются в файле `rsreq.ini`:

| Переменная | Описание |
|------------|----------|
| `OPREPCOUNT` | Количество повторов операции |
| `OPSLEEPTIME` | Время задержки между повторами |

Действие процедур выполнения транзакции в случае захвата файлов или записей может быть изменено с помощью процедуры `LoopInTrn`. Если данная процедура вызвана перед процедурой выполнения транзакции с параметром, равным `TRUE`, то в случае блокировки файла или записи процедура выполнения транзакции прервет выполнение транзакции и выполнит её заново. Количество повторов и время задержки между повторами задаётся параметрами `REPCOUNT` и `SLEEPTIME`.

**Вложенные транзакции не поддерживаются.**

### Пример транзакции

```rsl
/* В данном примере в транзакции обрабатывается справочник клиентов.
   Если отсутствует хотя бы один клиент, выполняется откат транзакции */

file acc (account);
file cl (client) write;

if (ProcessTrn (NULL, "MyTrn", cl, acc))
    println ("Transaction OK !")
else
    println ("Transaction Aborted")
end;

macro UpdateClient (client)
    cl.Client = client;
    if (getEQ (cl))
        cl.szShortName = cl.Name_Client;
    else
        println ("Клиент ", client, " отсутствует в справочнике !");
        AbortTrn
    end
end;

macro MyTrn
    while (next (acc))
        UpdateClient (acc.Client)
    end
end
```

### Пример использования LoopInTrn

```rsl
ob = TBfile ("client.dbt", "w", 0);

macro TrnProc
    ob.next;
    /* ... */
    ob.update;
end;

LoopInTrn (true);
if (not ProcessTrn (0, "Test"))
    println ("Ошибка в транзакции ", status)
end;
```

## Работа с текстовыми файлами

Перед обработкой данных текстового файла необходимо определить его в RSL-программе с помощью определения `FILE`:

```rsl
FILE aa ("c:\\out\\data.rep") txt;
```

Данная строка определяет идентификатор `aa` для текстового файла с именем `c:\out\data.rep`, доступного только для чтения.

### Открытие текстового файла для записи

```rsl
FILE aa ("c:\\out\\data.rep") txt write;
```

В этом случае файл создаётся автоматически. Если файл уже существовал, то он будет перезаписан.

### Дозапись в текстовый файл

```rsl
FILE aa ("MyFile") txt append;
```

### Процедуры для работы с текстовыми файлами

| Процедура | Описание |
|-----------|----------|
| `open` | Открыть файл |
| `close` | Закрыть файл |
| `next` | Читать следующую строку |
| `insert` | Записать строку в файл |
| `rewind` | Перейти к началу файла |

Текстовый файл автоматически открывается при первой необходимости и закрывается при выходе из области видимости идентификатора файла. Поэтому явно вызывать процедуры `open` и `close` нет необходимости.

### Чтение текстового файла

Процедура `next` позволяет читать файл строка за строкой. Доступ к прочитанной информации осуществляется через фиксированное имя поля `str`.

```rsl
while (next (aa))
    println (aa.str)
end;
```

### Доступ к полям строки по индексу

Если строки текстового файла состоят из полей, разделённых символами-разделителями, то к ним можно обращаться по индексу. Первое поле имеет индекс `0`. Набор символов-разделителей можно задать при помощи процедуры `setdelim`. По умолчанию разделителями являются символы пробел и табуляция.

```rsl
f1 = aa(0);
f2 = aa(1);
f3 = aa(5);
```

### Запись в текстовый файл

Запись в файл осуществляется строка за строкой в конец файла при помощи процедуры `insert`. Данные для записи определяются при помощи поля с предопределённым именем `str` или поля с индексом `0`. Допускается передать в качестве второго параметра процедуры `insert` непосредственно строку для записи.

```rsl
aa.str = "Данные для записи";
insert (aa);

/* или */
aa(0) = "Данные для записи";
insert (aa);

/* или */
insert (aa, "Данные для записи");
```

### Переустановка текстового файла

Процедура `rewind` переустанавливает текущую позицию на начало текстового файла. Полезна, если необходимо повторно прочитать файл.

### Открытие файла с указанием имени позже

При описании файла можно опустить физическое имя файла (но не круглые скобки!). В этом случае необходимо явно вызвать процедуру `open` с указанием имени файла:

```rsl
File aa () txt;
open (aa, "data.txt");
```

### Максимальный размер строки

Максимальная длина буфера у конструкции `TXT` равна **1024 символа**. Если необходимо считывать более длинные строки, то нужно указать максимальный размер строки:

```rsl
file aa () txt 2048;
```

В этом примере задаётся максимальный размер строки, равный 2048 байт. Размер одного поля в строке, тем не менее, не должен превышать 1024 символа.

## Работа с файлами в формате DBF

При работе с файлами формата DBF применяются те же принципы, что и для Btrieve-файлов. Прежде всего, необходимо определить файл с помощью определения `FILE`:

```rsl
FILE aa ("c:\\out\\data.dbf") dbf;
```

Данная строка определяет идентификатор `aa` для DBF-файла, доступного только для чтения, с именем `c:\out\data.dbf`.

### Открытие DBF-файла для записи

```rsl
FILE aa ("c:\\out\\data.dbf") dbf write;
```

### Процедуры для работы с DBF-файлами

| Процедура | Описание |
|-----------|----------|
| `open`, `close` | Открытие и закрытие файла |
| `create` | Создание файла |
| `next` | Чтение следующей записи |
| `insert`, `update`, `delete` | Модификация данных |
| `rewind` | Переустановка файла |
| `getpos`, `getdirect` | Прямой доступ к записи |
| `fldnumber`, `fldindex`, `fldname` | Информация о полях |
| `clone` | Клонирование файла |

Индексы для DBF-файлов не поддерживаются. Файл читается всегда в физической последовательности.

Процедура `insert` добавляет новую запись всегда в конец файла. Процедура `getpos` возвращает номер записи в DBF-файле.

Если необходимо прочитать запись с конкретным номером, можно использовать процедуру `getdirect` без предварительного вызова `getpos`. В этом случае вторым параметром для `getdirect` необходимо передать номер интересующей записи в DBF-файле (первая запись имеет номер `1`).

### Пример чтения DBF-файла

```rsl
while (next (aa))
    println (aa.name, " ", aa.id)
end
```

DBF-файл автоматически открывается при первой необходимости и закрывается при выходе из области видимости идентификатора файла.

### Открытие DBF-файла с указанием имени позже

```rsl
File aa () dbf;
open (aa, "data.dbf");
```

## Управление файлами и каталогами

В языке RSL предусмотрены средства для манипулирования файлами и каталогами, расположенными на компьютере, в том числе и в сети двух- и трёхуровневой архитектуры. К ним относятся специальные процедуры управления файлами и каталогами, а также стандартный класс `TDirList`.

Для использования данной возможности при работе в трёхуровневой архитектуре на терминале должен быть установлен модуль `rsextt.d32`.

### Соглашения о путях

Если при использовании указанных средств управления файлами и каталогами в качестве имён файлов не используются абсолютные пути, то действуют следующие соглашения:

| Расположение | Поиск файла | Создание файла |
|--------------|-------------|----------------|
| Сервер | В каталогах текстовых файлов | В первом каталоге из списка каталогов текстовых файлов |
| Терминал | В каталогах, указанных в `rsextt.ini` в параметре `DNDIR` | В каталоге, указанном в `rsextt.ini` в параметре `UPDIR` |

Чтобы указать, что файл находится на терминале, перед именем файла необходимо указать символ `$`. Если прикладная программа запущена в сети двухуровневой архитектуры, то символ `$` игнорируется.

## Стандартный класс TDirList

Класс `TDirList` позволяет получать список файлов и каталогов по заданной маске на сервере или на терминале, а также позволяет удалять файлы по маске на сервере или терминале.

### Конструктор

```rsl
TDirList ([mask:string] [, attr:string])
```

Конструктор создаёт новый экземпляр объекта класса. Если вызван с параметрами, объект будет наполнен списком файлов, удовлетворяющих маске поиска.

| Параметр | Описание |
|----------|----------|
| `mask` | Строка с маской файлов для поиска |
| `attr` | Строка с атрибутами для отбора файлов в список. Может содержать символы `F` (файлы) и `D` (каталоги) |

Если задан только символ `F` — производится поиск только файлов. Если задан только символ `D` — только каталогов. Если заданы оба символа — поиск файлов и всех каталогов. Последний режим используется по умолчанию.

### Методы класса TDirList

| Метод | Описание |
|-------|----------|
| `List (mask, attr)` | Наполняет объект списком файлов и/или каталогов, удовлетворяющих заданной маске |
| `Remove (mask, attr)` | Наполняет объект списком файлов и/или каталогов и удаляет отобранные файлы |
| `Sort (sortBy, dirFirst)` | Выполняет сортировку списка файлов |

#### Параметр sortBy

| Значение | Порядок сортировки |
|----------|-------------------|
| `0` | По имени файла |
| `1` | По размеру |
| `2` | По дате модификации |

Если параметр `sortBy` не задан, выполняется сортировка по имени.

Если задан параметр `dirFirst` и он равен `TRUE`, то каталоги после сортировки будут находиться в списке перед всеми файлами.

### Свойства класса TDirList

| Свойство | Описание |
|----------|----------|
| `Count` | Количество элементов в списке |
| `Name(ind)` | Имя файла или каталога с индексом `ind` |
| `Size(ind)` | Размер файла с индексом `ind` |
| `FTime(ind)` | Время последней модификации файла |
| `FDate(ind)` | Дата последней модификации файла |
| `IsDir(ind)` | `TRUE`, если элемент с индексом `ind` является каталогом |
| `IsDel(ind)` | `TRUE`, если файл с индексом `ind` был успешно удалён методом `Remove` |

### Пример использования TDirList

```rsl
ob = TDirList ("$*.*", "f");
ob.Sort (0);

i = 0;
while (i < ob.Count)
    if (ob.isDir (i))
        dir = "<DIR>"
    else
        dir = ""
    end;
    if (ob.isDel (i))
        del = "Del"
    else
        del = ""
    end;
    /* ... */
    i = i + 1
end;
```
