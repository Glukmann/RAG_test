хранения и редактирования номеров лицевых счетов, а также денежные суммы, которые форматируются в
"финансовом" формате;
 "o" – при использовании с числовыми полями заполняет поле вывода слева нулями, а не пробелами.
Для управления выводом значений, длина которых превышает длину поля вывода, дополнительно
используются следующие символы форматирования:
 "t" – обрезать значение по ширине поля вывода; это действие по умолчанию выполняется при выводе
слишком длинных символьных строк;
 "d" – вывести в поле вывода символы "*"; так выводятся по умолчанию слишком длинные числа;
 "w" – вывести продолжение на следующей строке или строках; это действие выполняется только для
слишком длинных значений строкового типа. При этом формат строк продолжения дублируется, строки
повторяются до тех пор, пока самое длинное из значений не будет выведено полностью.
Пример:
print (a + b:10:5:c)
В данном примере для переменной b указаны следующие спецификаторы форматирования: ширина поля вывода,
равная 10 символам, вывод 5 знаков после запятой, печать значения переменной по центру поля вывода.
## Формирование отчетов с использованием шаблонов

При подготовке отчетов на языке Object RSL используются стандартные классы TRepForm (см. стр. 39) и
TPattFieldR (см. стр. 40).
### Стандартный класс TRepForm

Класс TRepForm предназначен для формирования отчетов на языке Object RSL с использованием шаблонов,
подготовленных в отдельных текстовых файлах. Для использования этого класса в макропрограмму
необходимо импортировать DLM-модуль prnfrm.d32 при помощи директивы Import:
Import prnfrm;
Конструктор класса выглядит следующим образом:
TRepForm ( fileName : string [ , useUnnamed : bool ] )
Конструктор загружает файл шаблона формы отчета из файла с именем fileName.
Файл шаблона представляет собой обыкновенный текстовый файл в который вставлены поля. По умолчанию
для этого файла используется расширение "tpl". Начиная с версии RSL 885, файл ищется во всех каталогах,
указанных для поиска макрофайлов. В более ранних версиях файл шаблона ищется в текущем каталоге.
В поля файла шаблона можно помещать необходимые значения при помощи класса TRepForm. Полем является
последовательность символов ‘#’. Поле может быть именованным. Имя поля задается непосредственно перед
полем внутри специальных символов.
Непосредственно после имени может быть задан один из следующих атрибутов форматирования поля:
 c – выравнивание по центру поля;
 l – выравнивание по левому краю поля;
 r – выравнивание по правому краю поля.
Пример:
<!--name:c-->########## /* поле name с центрированием значения */
Для удобства редактирования формы отчета с поименованными полями встроенный редактор систем RS-Bank
и RS-Balance имеет режим скрытия имен полей, который вызывается с помощью клавиши [F12] (см. стр. 111).
Если второй необязательный параметр задан и равен TRUE, то в качестве полей в панели могут быть
использованы неименованные поля. В противном случае неименованные поля игнорируются. По умолчание
этот параметр принимает значение FALSE.
