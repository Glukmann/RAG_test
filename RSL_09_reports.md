## Спецификаторы форматирования (продолжение)

| Спецификатор | Описание |
|--------------|----------|
| `m` | Для `V_MONEY` — вывод значения прописью; для `V_DATE` — вывод с названием месяца словом; для остальных типов игнорируется |
| `z` | Не выводит нулевые значения `V_INTEGER`, `V_MONEY`, `V_DOUBLE` и `V_UNDEF`. Игнорируется, если не определена ширина поля вывода |
| `f` | Форматирует строку по шаблону прикладной системы. Используется для полей типа SNR и денежных сумм в "финансовом" формате |
| `o` | При использовании с числовыми полями заполняет поле вывода слева нулями, а не пробелами |

### Обработка значений, превышающих длину поля

| Спецификатор | Описание |
|--------------|----------|
| `t` | Обрезать значение по ширине поля вывода (по умолчанию для слишком длинных строк) |
| `d` | Вывести в поле вывода символы `*` (по умолчанию для слишком длинных чисел) |
| `w` | Вывести продолжение на следующей строке или строках (только для строковых значений). Строки продолжения повторяются до полного вывода самого длинного значения |

## Формирование отчетов с использованием шаблонов

При подготовке отчётов на языке Object RSL используются стандартные классы `TRepForm` и `TPattFieldR`.

### Стандартный класс TRepForm

Класс `TRepForm` предназначен для формирования отчётов на языке Object RSL с использованием шаблонов, подготовленных в отдельных текстовых файлах.

Для использования класса необходимо импортировать DLM-модуль `prnfrm.d32`:

```rsl
Import prnfrm;
```

### Конструктор TRepForm

```rsl
TRepForm ( fileName: string [, useUnnamed: bool ] )
```

Конструктор загружает файл шаблона формы отчёта из файла с именем `fileName`.

Файл шаблона представляет собой обыкновенный текстовый файл, в который вставлены поля. По умолчанию для этого файла используется расширение `.tpl`. Начиная с версии RSL 885, файл ищется во всех каталогах, указанных для поиска макрофайлов. В более ранних версиях файл шаблона ищется в текущем каталоге.

### Поля шаблона

Полем является последовательность символов `#`. Поле может быть именованным. Имя поля задаётся непосредственно перед полем внутри специальных символов.

Непосредственно после имени может быть задан один из следующих атрибутов форматирования поля:

| Атрибут | Описание |
|---------|----------|
| `c` | Выравнивание по центру поля |
| `l` | Выравнивание по левому краю поля |
| `r` | Выравнивание по правому краю поля |

### Пример шаблона

```
<!--name:c-->########## /* поле name с центрированием значения */
```

Для удобства редактирования формы отчёта с поименованными полями встроенный редактор систем RS-Bank и RS-Balance имеет режим скрытия имён полей, который вызывается клавишей `[F12]`.

### Параметр useUnnamed

Если второй необязательный параметр задан и равен `TRUE`, то в качестве полей в панели могут быть использованы неименованные поля. В противном случае неименованные поля игнорируются. По умолчанию этот параметр принимает значение `FALSE`.
