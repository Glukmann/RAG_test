# UserCryptPlugin

Введение
В ИБС RS-Bank V.6 применение электронно-цифровой подписи (ЭЦП) основано на связке
двух программных инструментов: криптосервера и криптоплагинов.
Криптосервер (сервер криптозащиты, CPRSCOM) – RSCOM-сервер, обеспечивающий
управление и сопряжение криптоплагина 
с 
механизмом 
прикладного 
применения
криптографии (МППК), который используется в различных подсистемах ИБС RS-Bank V.6
release 20 для выполнения криптографических действий над электронными документами, в
том числе для формирования и проверки ЭЦП. Криптосервер представляет собой
исполняемый модуль, загружаемый в прикладной процесс пользователя и обеспечивающий
взаимодействие с криптоплагинами. Криптосервер может одновременно работать с
произвольным количеством криптоплагинов.
Криптоплагин (плагин, КПГ) – средство ИБС RS-Bank V.6, обеспечивающее сопряжение
специфического программного интерфейса криптосредства с универсальным программным
интерфейсом МППК. Криптоплагин представляет собой специального вида динамическую
библиотеку (файл с расширением *.dll), реализующую интерфейс для конкретного
криптосредства.
При первом обращении к криптосерверу выполняется его загрузка в адресное пространство
терминала или сервера приложений. После этого осуществляется анализ каталога, в
котором расположены криптоплагины. Путь к каталогу задается в качестве значения
спецпеременной CRYPTOPLUGINDIR секции [COMMON] в файле настроек криптосервера
rscpcom.ini, который располагается там же, где и криптосервер. Все найденные плагины
загружаются в память в виде списка, при этом выполняются следующие проверки:
· является ли динамическая библиотека криптоплагином;
· актуальности версии плагина;
· наличия ранее загруженного в память плагина с одноименным идентификатором.
Криптоплагины однозначно идентифицируются по строковому идентификатору (значение
константы PLUGIN_ID в файле usercrypt.h), а не по физическому имени файла. Вызов
методов, предназначенных для подписания документа, проверки подписи, шифрования,
дешифровки, выдачи списка абонентов криптосистемы
, сопровождается поиском в
памяти плагина с указанным в параметрах этих методов идентификатором. При отсутствии
указанного плагина в памяти выполняется повторный анализ каталога, содержащего
криптоплагины. В случае отсутствия плагина в каталоге или невозможности загрузки его в
память, на экран выдается соответствующее сообщение. Выгрузка криптосервера и всех
загруженных криптоплагинов выполняется перед завершением работы ИБС RS-Bank V.6.

Криптоплагины ИБС RS-Bank V.6 экспортируют свою базовую функциональность,
используемую при работе с ЭЦП, не через перечисленные выше экспортные функции, а
через объект класса, возвращаемый экспортной функцией _OpenPlugin. Криптосервер,
загружая в память очередной криптоплагин, с помощью вызова функции _OpenPlugin
получает указатель на статический объект, содержащийся в плагине. Таким образом, вся
функциональность, используемая при работе с ЭЦП, заложена в свойствах и методах этого
объекта.

Класс TCryptoPlugin
Абстрактный базовый класс TCryptoPlugin, используемый для создания пользовательского
криптоплагина, помимо методов
, которые переопределяются в производном классе,
содержит также следующие вспомогательные классы:
· TCryptoIdSequence – класс, используемый для работы со списком идентификаторов
криптоплагинов.
· TCheckSignaturesCallback – класс, содержащий callback-функции, используемые для
получения результатов работы функций проверки ЭЦП.
· TEnumerateAbonentsCallback – класс, содержащий callback-функцию AddAbonent,
предназначенную для передачи информации об абоненте системы криптографической
защиты информации.
В описаниях методов могут использоваться следующие типы данных:
· TPluginInt – четырехбайтовое целое число.
· TPluginBool – однобайтный 0 или 1.
· TPluginDouble – восьмибайтное число с плавающей точкой.
· TPluginLongDouble – десятибайтное число с плавающей точкой.
· TPluginDate – дата. Тип представляет собой структуру следующего вида:
struct TPluginDate
{
  unsigned char day;
  unsigned char mon;
  unsigned short int year;
  TPluginDate():day(0),mon(0),year(0) {}
}
· TPluginTime – время. Тип представляет собой структуру следующего вида:
struct TPluginTime
{
  unsigned char hundr;
  unsigned char sec;
  unsigned char min;
  unsigned char hour;
  TPluginTime():hundr(0),sec(0),hour(0) {min=0};
}

pluginAPI, содержащемся в каталоге RsPluginAPI

ERetCodes, обозначающий коды возврата некоторых методов. Список перечисления

· NotSupported – метод не реализован или не поддерживается указанным криптоплагином.

· Ok – при отсутствии ошибок.
· Cancel – при отмене действий пользователя, выполняемых в списке выбора.

Методы проверки ЭЦП
CryptoCheckSignatures()
Метод предназначен для проверки ЭЦП буфера памяти. После проверки подписи
необходимо заполнить структуру TCryptoCheckSignResult и передать ее в криптосервер,
используя метод AddSignResult объекта, указатель на который задан в качестве значения
параметра callback. В структуре TCryptoCheckSignResult необходимо заполнить следующие
поля:
· id – идентификатор абонента криптосистемы, который подписал сообщение.
· verified – результат проверки подписи. Возможные значения поля:
· True – подпись корректна;
· False – подпись не корректна.
· ResultCode – результат работы метода. Возможные значения поля:
· 0 – при успешном выполнении метода;
· код ошибки – в случае возникновения ошибки.
· name – имя/название подписавшего абонента.
· date – дата подписания.
· time – время подписания.
Криптоплагины ИБС RS-Bank V.6
· position – порядковый номер подписи.
· desc – текстовое описание результата проверки подписей.
· GenResult – дополнительное описание результата проверки подписи.
Прототип:
virtual TPluginBool PLUGINCALLTYPE CryptoCheckSignatures(const char *buf,
TPluginInt size, TCheckSignaturesCallback *callback, int CompINum = 0, bool

Параметры:

size – размер ЭЦП.
