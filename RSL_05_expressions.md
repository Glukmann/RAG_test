## Константы

### Целочисленные константы

Целочисленные константы типа `V_INTEGER` имеют диапазон допустимых значений от `-2'147'483'648` до `2'147'483'648`.

```rsl
2345, 1236 и т.п.
```

### Числа с плавающей точкой

Числа с плавающей точкой имеют тип `V_DOUBLE` и могут находиться в диапазоне от `1.7e-308` до `1.7e308`. При их записи используются точка и латинская буква `e`:

```rsl
4356.234, 345., .1234, 1231.2341e-23
```

### Денежные константы

При записи денежных сумм в начале ставят знак доллара `$`. Эти константы имеют тип `V_MONEY` и могут принимать максимальное значение `9'999'999'999'999.99`.

```rsl
$146       /* 14600 */
$765.23    /* 76523 */
```

В скобках дано внутреннее представление константы.

### Логические константы

Логические константы типа `V_BOOL` могут принимать только два значения: `TRUE` — истина, `FALSE` — ложь.

### Предопределённые символические константы

| Константа | Тип | Значение |
|-----------|-----|----------|
| `TRUE` | `V_BOOL` | Истина |
| `FALSE` | `V_BOOL` | Ложь |
| `NULL` | `V_UNDEF` | Неопределённое значение |

Предопределённые константы для работы с диалоговыми окнами описаны в разделе, посвящённом вводу/выводу.

## Синтаксис выражений

Выражения представляют собой последовательность операндов, разделённых знаками операций.

### Примеры выражений

```rsl
Name              /* выражение из одного операнда — простого имени */
MgFun(10)         /* выражение из одного операнда — сложного имени */
a + b             /* сумма двух переменных */
b * 10 + c        /* произведение и сумма переменных */
```

### Приоритет операций

В приведённой ниже таблице операции располагаются в порядке возрастания приоритета при вычислении выражений. Приоритет операций в каждой группе одинаков.

| Обозначение | Операция | Ассоциативность |
|-------------|----------|-----------------|
| `=` | Присваивание | Правая |
| `==` | Равно | Левая |
| `!=` | Не равно | Левая |
| `<` | Меньше | Левая |
| `>` | Больше | Левая |
| `<=` | Меньше или равно | Левая |
| `>=` | Больше или равно | Левая |
| `+` | Сложение | Левая |
| `-` | Вычитание | Левая |
| `OR` | Дизъюнкция | Левая |
| `*` | Умножение | Левая |
| `/` | Деление | Левая |
| `AND` | Конъюнкция | Левая |
| `-` | Унарный минус | Левая |
| `+` | Унарный плюс | Левая |
| `NOT` | Отрицание | Левая |
| `@` | Получение ссылки на процедуру | Левая |

Для явного задания порядка вычисления необходимо применять круглые скобки:

```rsl
a + B * c    /* эквивалентно */    a + (b * c)
```

### Особенности вычисления

- Все операции, кроме операции присваивания, имеют **левую ассоциативность**.
- Операция присваивания имеет **правую ассоциативность**: `a = b = 10` интерпретируется как `a = (b = 10)`.
- Последовательность вычисления операндов в выражении **не определена**.
- Исключение составляют операции `AND` и `OR`: для них гарантируется строгий порядок вычисления операндов — сначала вычисляется левый операнд, и если после этого результат операции уже известен, вычисление правого операнда не производится.

## Семантика операций

В зависимости от типов операндов операции определены различным образом. Если в выражении используются операнды разных типов, и для них не описаны правила выполнения операции, то производится повышение типа операнда с меньшим приоритетом до типа операнда с большим приоритетом.

### Приоритет типов

```
V_INTEGER -> V_MONEY -> V_DOUBLE -> V_BOOL
V_STRING -> V_BOOL
```

### Возможные явные преобразования типов

| Из типа | В типы |
|---------|--------|
| `V_INTEGER` | `V_DOUBLE`, `V_MONEY`, `V_BOOL`, `V_STRING` |
| `V_MONEY` | `V_DOUBLE`, `V_BOOL`, `V_STRING`, `V_MONEYL` |
| `V_DOUBLE` | `V_BOOL`, `V_INTEGER`, `V_MONEY`, `V_STRING`, `V_DOUBLEL` |
| `V_STRING` | `V_BOOL`, `V_INTEGER`, `V_DOUBLE` |
| `V_TIME` | `V_STRING` |
| `V_DATE` | `V_STRING` |
| `V_DTTM` | `V_STRING` |
| `V_GENOBJ` | `V_STRING` |
| `V_FILE` | `V_STRING` |

Такие преобразования осуществляются либо с помощью специальных встроенных процедур, либо в процессе присвоения значений переменных полям в файлах и структурах (`FILE`, `RECORD`). Преобразование также происходит при присвоении значения переменной с явно декларированным типом данных.

```rsl
var a : string;
a = 10;    /* a будет содержать строковое значение "10" */
```

### Операции с логическими типами

Логические операции `AND`, `OR` и `NOT` определены только для операндов типа `V_BOOL`.

### Арифметические операции

Все арифметические операции определены для числовых типов данных (`V_INTEGER`, `V_DOUBLE`, `V_MONEY`). Тип результата бинарных операций будет таким же, как у операнда с большим приоритетом, кроме особых случаев с операндами типа `V_MONEY`.

### Конкатенация строк

Операция `+` определена для операндов типа `V_STRING` как конкатенация (соединение строк):

```rsl
aa = "Stroka1" + "Stroka2";    /* aa имеет тип V_STRING и содержит "Stroka1Stroka2" */
```

### Операции с датами

Операции `+` и `-` определены для операндов типа `V_DATE` и `V_INTEGER`. Результат операции имеет тип `V_DATE`:

```rsl
Сегодня = date;
Завтра = Сегодня + 1;
Вчера = Сегодня - 1;
```

Операция `-` определена для двух операндов типа `V_DATE`. Результат имеет тип `V_INTEGER` и представляет собой количество дней между двумя датами:

```rsl
n = Завтра - Вчера;    /* n получает тип V_INTEGER и значение 2 */
```

### Операции со временем

Операции `+` и `-` определены для операндов типа `V_TIME`. Результат операции имеет тип `V_TIME`:

```rsl
n = time(10, 0) + time(0, 15);    /* n получает тип V_TIME и значение 10:15 */
```

### Операции с денежными значениями

| Операция | Операнды | Результат |
|----------|----------|-----------|
| `+`, `-` | `V_MONEY` | `V_MONEY` |
| `*`, `/` | `V_MONEY` | `V_DOUBLE` |
| `*` | `V_MONEY` и `V_INTEGER` | `V_MONEY` |
| `*` | `V_MONEY` и `V_DOUBLE` | `V_DOUBLE` |
| `/` | `V_MONEY` и `V_INTEGER` | `V_MONEY` |
| `/` | `V_MONEY` и `V_DOUBLE` | `V_DOUBLE` |

```rsl
Дебет = $100;
Кредит = $33;
Оборот = Дебет - Кредит;    /* Оборот получит тип V_MONEY и значение $67 */
Результат = Дебет / Кредит; /* Результат получит тип V_DOUBLE и значение 3.3333(3) */
```

### Целочисленное деление

Результатом операции `/` между операндами типа `V_INTEGER` будет значение типа `V_INTEGER`, равное целой части от деления:

```rsl
a = 5;
b = 3;
c = a / b;    /* c получает тип V_INTEGER и значение 1 */
```

### Операции сравнения

Операции сравнения определены для всех типов данных. Для данных типа `V_BOOL` определены операции равно `==` и не равно `!=`. Результатом операций сравнения является значение типа `V_BOOL` — ложь или истина.

Для переменных типов `V_FREF`, `V_SREF`, `V_AREF`, `V_TREF`, `V_DREF` и `V_MEMADDR` определены операции сравнения с `NULL`.

Для переменных типа `V_MEMADDR` предусмотрена операция сравнения друг с другом.
