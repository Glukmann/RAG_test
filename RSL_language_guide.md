## Введение

Настоящее Руководство содержит описание языка RSL, являющегося неотъемлемой частью систем, разработанных программистами компании R-Style Softlab.

Руководство состоит из трех частей:
1. **Язык интерпретатора RSL** — язык, при помощи которого пользователь имеет возможность создавать собственные программы.
2. **Интегрированная среда разработки программ на RSL** — средство, при работе с которым пользователю предоставляется возможность работать с Btrieve-файлами в интерактивном режиме, а также создавать и редактировать диалоговые окна.
3. **Отладчик макропрограмм** — средство, позволяющее эффективно отлаживать RSL-программы, написанные пользователем.

RSL — современный язык, разработанный с учетом нынешних требований к программному обеспечению. В компании R-Style Softlab ведется постоянная работа по его развитию и совершенствованию.

Язык интерпретатора RSL — это язык высокого уровня. Он достаточно прост в изучении и применении и одинаково хорош как для квалифицированного, так и для неквалифицированного пользователей системы, так как имеет широкий спектр функциональных возможностей. В отличие от других подобных ему языков, язык интерпретатора интегрирован с системами, разработанными программистами компании R-Style Softlab. Он вызывается непосредственно из меню системы, обеспечивает доступ к базе данных, использует одинаковые с ней форматы данных, работает с экранными формами.

## Часть 1. Язык интерпретатора RSL

### 1.1. Элементы языка

#### 1.1.1. Служебные слова

Служебное, или зарезервированное, слово — это имя, с которым в языке жестко сопоставлены определенные смысловые значения, и которое не может быть использовано ни для каких других целей. Ниже приведен список служебных слов языка RSL (они записываются в любом регистре).

Основные служебные слова:
- `and` — логическое И
- `array` — определение массива
- `class` — определение класса
- `const` — определение константы
- `elif` — ветвление "иначе если"
- `end` — конец блока
- `false` — логическое значение "ложь"
- `file` — определение файла
- `if` — условная инструкция
- `import` — импорт модуля
- `local` — локальная переменная
- `macro` — определение процедуры
- `not` — логическое НЕ
- `NULL` — пустое значение
- `onerror` — обработка ошибок
- `or` — логическое ИЛИ
- `private` — приватный доступ
- `record` — определение записи
- `return` — возврат из процедуры
- `this` — ссылка на текущий объект
- `true` — логическое значение "истина"
- `var` — определение переменной
- `while` — цикл
- `with` — конструкция WITH

Ключевые слова для определений FILE и RECORD:
- `btr` — Btrieve файл
- `dbf` — DBF файл
- `dialog` — диалоговое окно
- `key` — ключ
- `mem` — память
- `sort` — сортировка
- `txt` — текстовый файл
- `write` — запись

#### 1.1.2. Имена (идентификаторы)

Под именем (идентификатором) в программе понимается не являющаяся служебным словом последовательность букв (латинских и русских) и цифр, начинающаяся с буквы; символ "_" (подчеркивание) рассматривается как буква. Длина идентификатора в RSL не должна превышать 80 символов. Прописные и строчные буквы не рассматриваются как различные символы, то есть регистр игнорируется. Имена используются для идентификации объектов RSL.

Кроме обычных идентификаторов, в RSL применяются имена специальных переменных, которые могут содержать любые символы, включая специальные, без ограничений. Такие имена должны быть заключены в фигурные скобки `{}`, которые также входят в имя переменной.

Специальные переменные являются глобальными и используются обычно для передачи значений из вызывающих модулей, написанных на языках C или C++, в программу на языке RSL.

Для обозначения полей в структурах и файлах, описанных в словаре базы данных, используются составные имена. Составные имена состоят из имени, идентифицирующего файл или структуру, после которого следует точка и название поля либо индекс поля в круглых скобках.

Примеры имен:
```rsl
Счет              // простое имя
Сумма010          // простое имя
{Debet account 52.5}  // имя специальной переменной
Клиенты.Name      // составное имя: поле Name в структуре Клиенты
Клиенты(10)       // составное имя: десятое поле в структуре Клиенты
```

Если имя переменной совпадает с именем переменной среды, оно инициализируется ее значением.

Пример:
```rsl
PATH  // переменная будет проинициализирована значением системной переменной PATH
```

Присвоение значений этим переменным не изменяет содержимого системных переменных.

#### 1.1.3. Область видимости имен

Глобальными именами считаются те, которые объявлены вне любой процедуры RSL. Имена специальных переменных, независимо от места их появления, считаются глобальными. Для размещения глобальных объектов используется специальная область памяти, через которую осуществляется доступ к их значениям из любого места программы RSL или из вызывающего внешнего модуля.

Локальными именами считаются те, которые объявлены в любой процедуре RSL. Локальные объекты создаются заново при каждом вызове процедуры, в которой были объявлены их имена, и доступны только в этой процедуре, а также во всех вложенных в нее процедурах RSL.

Следует отметить, что определенные явно имена локальных объектов могут совпадать с именами глобальных объектов. В этом случае имя локального объекта перекрывает имя глобального, и доступ к глобальному объекту будет блокирован.

Каждая RSL-программа образует глобальную область видимости. Каждая процедура, содержащаяся в программе, образует свою, вложенную область видимости. Из вложенной области видимости доступны имена этой области и всех вышележащих. В каждой области видимости можно "перекрывать" имена из вышележащих областей.

При входе в область видимости переменные конструируются, при выходе разрушаются. Таким образом, время жизни переменной ограничено областью ее видимости.

#### 1.1.4. Комментарии

Любой фрагмент текста, заключенный в скобки вида `/* */`, является комментарием. Язык RSL разрешает наличие вложенных комментариев.

Пример:
```rsl
/*********************************************************
Комментарий на языке RSL
/*
Вложенный комментарий
*/
**********************************************************/
```

Данный пример содержит два комментария, один из которых вложен в другой.

#### 1.1.5. Объекты языка

Объект RSL представляет собой совокупность информации, с которой оперирует язык. Объектами языка RSL являются:

- **Переменная** — объект, содержащий значение одного из типов данных, который может изменять величину и тип хранимого значения. Это происходит при присвоении переменной нового значения операцией присваивания (при этом старое значение теряется).
- **Константа** — объект, содержащий значение одного из типов данных, которое не может быть изменено.
- **Массив** — объект, содержащий упорядоченную совокупность элементов одного типа.
- **Класс** — объект, содержащий описание структуры данных и методов работы с ними.
- **Экземпляр класса (объект)** — объект, созданный на основе описания класса.
- **Процедура** — объект, содержащий программный код, который может быть вызван по имени.

#### 1.1.6. Типы данных

Язык RSL поддерживает следующие типы данных:

| Тип | Описание | Пример |
|-----|----------|--------|
| `INTEGER` | Целое число | `42`, `-17` |
| `REAL` | Вещественное число | `3.14`, `-0.5` |
| `STRING` | Строка символов | `"Hello"`, `'RSL'` |
| `BOOLEAN` | Логическое значение | `true`, `false` |
| `DATE` | Дата | `DATE('01.01.2024')` |
| `TIME` | Время | `TIME('12:30:00')` |
| `DATETIME` | Дата и время | `DATETIME('01.01.2024 12:30:00')` |
| `CURRENCY` | Денежный тип | `CURRENCY(100.50)` |
| `ARRAY` | Массив | `ARRAY[1..10] OF INTEGER` |
| `FILE` | Файл | Определение через FILE |
| `RECORD` | Запись | Определение через RECORD |
| `CLASS` | Класс | Определение через CLASS |
| `OBJECT` | Объект | Экземпляр класса |
| `VARIANT` | Универсальный тип | Может содержать любое значение |

Тип переменной определяется автоматически при присвоении значения. Язык RSL является динамически типизированным в части переменных, но поддерживает статическую типизацию для определений CLASS, FILE, RECORD.

#### 1.1.7. Константы

Константы в RSL определяются с помощью ключевого слова `CONST`. Константа может быть любого простого типа.

Пример:
```rsl
CONST
  MAX_COUNT = 100
  PI = 3.14159
  COMPANY_NAME = "R-Style Softlab"
  FLAG = true
END
```

Константы, определенные таким образом, являются глобальными для всей программы.

#### 1.1.8. Выражения

Выражение в RSL — это конструкция, состоящая из операндов (констант, переменных, вызовов функций), соединенных знаками операций. Выражения вычисляются слева направо с учетом приоритета операций.

Приоритет операций (от высшего к низшему):
1. `()` — скобки
2. `.` — доступ к полю/свойству
3. `[]` — индексация массива
4. `+`, `-` (унарные) — знак числа
5. `*`, `/`, `%` — умножение, деление, остаток
6. `+`, `-` — сложение, вычитание
7. `=`, `!=`, `<`, `>`, `<=`, `>=` — сравнение
8. `not` — логическое отрицание
9. `and` — логическое И
10. `or` — логическое ИЛИ

Примеры выражений:
```rsl
a + b * c
(x > 0) and (y < 100)
not flag or (count = MAX_COUNT)
Клиенты.Сумма * Курс.Значение
```

### 1.2. Структура программы

RSL-программа представляет собой совокупность модулей, каждый из которых хранится в отдельном файле. Модуль может содержать:
- определения констант (`CONST`)
- определения переменных (`VAR`)
- определения классов (`CLASS`)
- определения процедур (`MACRO`)
- определения файлов (`FILE`)
- определения записей (`RECORD`)
- исполняемый код

Структура модуля:
```rsl
// Заголовок модуля (опционально)
MODULE ИмяМодуля

// Импорт других модулей
IMPORT ДругойМодуль

// Определения констант
CONST
  // константы
END

// Определения переменных
VAR
  // переменные
END

// Определения классов, файлов, записей
CLASS ИмяКласса
  // поля и методы
END

FILE ИмяФайла
  // описание структуры файла
END

// Определения процедур
MACRO ИмяПроцедуры(параметры)
  // тело процедуры
END

// Главная программа (исполняемый код)
// инструкции
```

#### Загрузка и кэширование модулей

Исполняющая система RSL загружает модули по мере необходимости. При первом обращении к модулю он загружается в память и кэшируется. При повторном обращении используется закэшированная версия, если только исходный файл не был изменен.

Модули ищутся в следующем порядке:
1. Текущий каталог
2. Каталоги, указанные в переменной окружения `RSL_PATH`
3. Стандартный каталог библиотек RSL

### 1.3. Конструкции языка RSL

#### 1.3.1. Пустая инструкция

Пустая инструкция не выполняет никаких действий и записывается как `;`.

#### 1.3.2. Инструкция "выражение"

Вычисление выражения, результат которого не используется, может выполняться как отдельная инструкция.

Пример:
```rsl
x + y;  // вычисление суммы, результат отбрасывается
```

#### 1.3.3. Условная инструкция IF

Условная инструкция позволяет выполнять различные действия в зависимости от значения логического выражения.

Синтаксис:
```rsl
IF условие THEN
  // инструкции, выполняемые если условие истинно
ELIF другое_условие THEN
  // инструкции, выполняемые если другое_условие истинно
ELSE
  // инструкции, выполняемые если все условия ложны
END
```

Части `ELIF` и `ELSE` являются необязательными. Вложенные условные инструкции допускаются.

Пример:
```rsl
IF x > 0 THEN
  PRINT("Положительное число")
ELIF x < 0 THEN
  PRINT("Отрицательное число")
ELSE
  PRINT("Ноль")
END
```

#### 1.3.4. Инструкция цикла WHILE

Цикл `WHILE` выполняет тело цикла пока условие истинно.

Синтаксис:
```rsl
WHILE условие DO
  // тело цикла
END
```

Пример:
```rsl
i = 1
WHILE i <= 10 DO
  PRINT("i = " + i)
  i = i + 1
END
```

#### 1.3.5. Инструкция возврата RETURN

Инструкция `RETURN` завершает выполнение текущей процедуры и возвращает управление вызывающему коду. Опционально может возвращать значение.

Синтаксис:
```rsl
RETURN          // возврат без значения
RETURN значение // возврат с значением
```

Пример:
```rsl
MACRO Sum(a, b)
  RETURN a + b
END
```

#### 1.3.6. Инструкция вывода PRINT

Инструкция `PRINT` выводит значения выражений на экран или в файл.

Синтаксис:
```rsl
PRINT(выражение1, выражение2, ...)
PRINT("форматная_строка", выражение1, выражение2, ...)
```

Форматная строка может содержать спецификаторы форматирования:
- `%s` — строка
- `%d` — целое число
- `%f` — вещественное число
- `%c` — символ
- `%t` — дата/время
- `%m` — денежный формат

Пример:
```rsl
PRINT("Сумма: %m", Сумма)
PRINT("Дата: %t, Клиент: %s", Дата, Клиенты.Name)
```

#### 1.3.7. Определение переменных VAR

Переменные объявляются с помощью ключевого слова `VAR`. Тип переменной определяется автоматически при первом присвоении.

Синтаксис:
```rsl
VAR
  имя1, имя2, ...
END
```

Пример:
```rsl
VAR
  Счетчик
  Сумма
  ИмяКлиента
  МассивЗначений
END
```

#### 1.3.8. Определение классов и объектов CLASS

Класс в RSL определяет структуру данных и методы работы с ними. Классы являются основой объектно-ориентированного программирования в RSL.

Синтаксис:
```rsl
CLASS ИмяКласса [РодительскийКласс]
  // Поля (переменные класса)
  VAR
    Поле1
    Поле2
  END

  // Методы (процедуры класса)
  MACRO Метод1(параметры)
    // тело метода
  END

  MACRO Метод2(параметры)
    // тело метода
  END
END
```

Пример:
```rsl
CLASS TКлиент
  VAR
    Имя
    Адрес
    Телефон
    Баланс
  END

  MACRO Инициализировать(имя, адрес, телефон)
    Имя = имя
    Адрес = адрес
    Телефон = телефон
    Баланс = 0
  END

  MACRO Пополнить(сумма)
    Баланс = Баланс + сумма
  END

  MACRO ПолучитьБаланс()
    RETURN Баланс
  END
END
```

Создание экземпляра класса:
```rsl
VAR
  Клиент
END

Клиент = NEW TКлиент
Клиент.Инициализировать("Иванов", "Москва", "123-45-67")
Клиент.Пополнить(1000)
PRINT("Баланс: %m", Клиент.ПолучитьБаланс())
```

#### 1.3.9. Определение символических констант CONST

Константы определяются в блоке `CONST`. Они доступны во всей программе.

Пример:
```rsl
CONST
  MAX_RECORDS = 1000
  VERSION = "2.1.0"
  DEBUG_MODE = false
  COMPANY = "R-Style Softlab"
END
```

#### 1.3.10. Определение процедуры MACRO

Процедура (макро) — это именованный блок кода, который может принимать параметры и возвращать значение.

Синтаксис:
```rsl
MACRO ИмяПроцедуры([параметры])
  // тело процедуры
  [RETURN значение]
END
```

Пример:
```rsl
MACRO РассчитатьНДС(сумма, ставка)
  VAR
    НДС
  END
  НДС = сумма * ставка / 100
  RETURN НДС
END

// Вызов процедуры
VAR
  Результат
  Сумма
END
Сумма = 10000
Результат = РассчитатьНДС(Сумма, 20)
PRINT("НДС: %m", Результат)
```

#### 1.3.11. Передача параметров

Параметры в RSL передаются по значению (для простых типов) или по ссылке (для объектов и массивов). Чтобы передать простой тип по ссылке, используется ключевое слово `VAR` перед параметром в определении процедуры.

Пример передачи по ссылке:
```rsl
MACRO Увеличить(VAR число, шаг)
  число = число + шаг
END

VAR
  Значение
END
Значение = 10
Увеличить(Значение, 5)
PRINT("Значение: %d", Значение)  // Выведет: 15
```

#### 1.3.12. Определение массивов

Массивы в RSL могут быть определены двумя способами: с помощью конструкции `ARRAY` или как стандартный класс `TArray`.

Способ 1: Конструкция ARRAY
```rsl
ARRAY ИмяМассива[нижняя_граница..верхняя_граница] OF тип
```

Пример:
```rsl
ARRAY Числа[1..100] OF INTEGER
ARRAY Имена[0..50] OF STRING

Числа[1] = 10
Числа[2] = 20
Имена[0] = "Первый"
```

Способ 2: Класс TArray

#### 1.3.13. Определения FILE и RECORD

FILE определяет структуру файла базы данных, RECORD определяет структуру записи.

Синтаксис FILE:
```rsl

  ...

```

Типы файлов:
- `btr` — Btrieve файл (по умолчанию)
- `dbf` — DBF файл
- `txt` — текстовый файл
- `mem` — файл в памяти
- `dialog` — диалоговое окно

Пример:
```rsl
FILE Клиенты btr

  KEY Имя
END
```

```rsl
RECORD ИмяЗаписи

  ИмяПоля2 : Тип2
  ...
END
```

Пример:

### 1.7. Работа с файлами

#### 1.7.1. Использование стандартного класса Tbfile

Класс `Tbfile` предоставляет низкоуровневый доступ к файлам Btrieve.

Методы и свойства:
- `Open(имя_файла)` — открытие файла
- `Close()` — закрытие файла
- `Create(структура)` — создание файла
- `Read(ключ)` — чтение записи по ключу
- `Write(запись)` — запись записи
- `Delete(ключ)` — удаление записи
- `Update(запись)` — обновление записи
- `First()` — первая запись
- `Last()` — последняя запись
- `Next()` — следующая запись
- `Prev()` — предыдущая запись
- `EOF` — признак конца файла
- `BOF` — признак начала файла
- `RecordCount` — количество записей

Пример:
```rsl
VAR
  Файл
  Запись
END
Файл = NEW Tbfile
Файл.Open("clients.btr")

Запись = Файл.First()
WHILE NOT Файл.EOF DO
  PRINT("Клиент: %s", Запись.Имя)
  Запись = Файл.Next()
END

Файл.Close()
```

#### 1.7.2. Использование стандартного класса TRecHandler

Класс `TRecHandler` предоставляет высокоуровневый интерфейс для работы с записями.

Методы:
- `GetRecord(файл, ключ)` — получение записи
- `PutRecord(файл, запись)` — сохранение записи
- `DeleteRecord(файл, ключ)` — удаление записи
- `FindRecord(файл, условие)` — поиск записи

#### 1.7.3. Использование конструкции FILE

При работе с файлами, определенными через `FILE`, можно использовать более удобный синтаксис:

```rsl
FILE Клиенты btr
  Код : INTEGER
  Имя : STRING[50]
  Баланс : CURRENCY
  KEY Код Уникальный
END

// Открытие файла
Клиенты.Open()

// Добавление записи
VAR
  НовыйКлиент
END
НовыйКлиент = Клиенты.NewRecord()
НовыйКлиент.Код = 100
НовыйКлиент.Имя = "Иванов Иван"
НовыйКлиент.Баланс = CURRENCY(1000.50)
Клиенты.Append(НовыйКлиент)

// Поиск записи
VAR
  Найденный
END
Найденный = Клиенты.Find(100)  // по ключу Код
IF Найденный != NULL THEN
  PRINT("Найден: %s", Найденный.Имя)
END

// Перебор всех записей
VAR
  Текущая
END
Текущая = Клиенты.First()
WHILE Текущая != NULL DO
  PRINT("%d: %s", Текущая.Код, Текущая.Имя)
  Текущая = Клиенты.Next()
END

// Закрытие файла
Клиенты.Close()
```

#### 1.7.4. Работа с Btrieve-файлами переменной длины

Btrieve поддерживает записи переменной длины. В RSL это реализуется двумя способами.

**Первый способ:** Использование полей переменной длины в определении FILE.

```rsl
FILE Документы btr
  Номер : INTEGER
  Содержимое : STRING[0]  // длина 0 = переменная длина
  KEY Номер
END
```

**Второй способ:** Использование отдельных сегментов данных.

```rsl
FILE Документы btr
  Номер : INTEGER
  РазмерДанных : INTEGER
  KEY Номер
END

// Данные хранятся отдельно и загружаются по мере необходимости
```

#### 1.7.5. Обработка ошибок

При работе с файлами рекомендуется использовать обработку ошибок:

```rsl
ONERROR

  IF ERROR.CODE = 2 THEN
    PRINT("Файл не найден, создаем новый...")

  END

```

#### 1.7.6. Обработка транзакций

RSL поддерживает транзакции для обеспечения целостности данных:

```rsl
// Начало транзакции
BEGIN TRANSACTION

Клиенты.Open()
Счета.Open()

// Выполнение операций
VAR
  Клиент
  Счет
END
Клиент = Клиенты.Find(КодКлиента)
Счет = Счета.Find(КодСчета)

Клиент.Баланс = Клиент.Баланс - Сумма
Счет.Баланс = Счет.Баланс + Сумма

Клиенты.Update(Клиент)
Счета.Update(Счет)

// Фиксация транзакции
COMMIT

// Или откат при ошибке
// ROLLBACK
```

```rsl
// Рекурсивный расчет факториала
```

### 1.10. Средство разработки расширений (DLM SDK)

#### 1.10.1. Создание и использование DLM-модулей

DLM (Dynamic Link Module) — механизм расширения возможностей RSL путем подключения модулей, написанных на C/C++.

Создание DLM-модуля:
1. Создать проект в DLM SDK
2. Реализовать экспортируемые функции
3. Скомпилировать в DLL/SO
4. Подключить к RSL

Использование DLM в RSL:
```rsl
IMPORT "myextension.dlm"

VAR
  Результат
END

Результат = MyExtensionFunction(параметры)
```

#### 1.10.2. Передача параметров и возврат значений

DLM-модули поддерживают передачу параметров следующих типов:
- INTEGER
- REAL
- STRING
- BOOLEAN
- DATE
- TIME
- DATETIME
- CURRENCY
- ARRAY
- OBJECT

Пример C++ кода для DLM:
```cpp
// myextension.cpp
#include "dlm.h"

EXPORT int MyExtensionFunction(int param1, double param2) {
    // обработка
    return param1 * (int)param2;
}
```

### 1.11. Сводка синтаксиса RSL

#### Полная грамматика языка

```
Программа ::= { Модуль }

Модуль ::= [ MODULE Имя ] { Определение } { Инструкция }

Определение ::= ОпределениеConst | ОпределениеVar | ОпределениеClass 
                | ОпределениеMacro | ОпределениеFile | ОпределениеRecord

ОпределениеConst ::= CONST { Имя = Значение } END

ОпределениеVar ::= VAR { Имя } END

ОпределениеClass ::= CLASS Имя [ : ИмяРодителя ] { Поле | Метод } END

Поле ::= VAR { Имя } END

Метод ::= MACRO Имя [ ( Параметры ) ] { Инструкция } END

ОпределениеMacro ::= MACRO Имя [ ( Параметры ) ] { Инструкция } END

ОпределениеFile ::= FILE Имя [ ТипФайла ] { ПолеФайла | KEY } END

ПолеФайла ::= Имя : Тип [ Размер ]

KEY ::= KEY Имя [ Уникальный ]

ОпределениеRecord ::= RECORD Имя { ПолеФайла } END

Инструкция ::= ПустаяИнструкция | ИнструкцияВыражение | ИнструкцияIf 
             | ИнструкцияWhile | ИнструкцияReturn | ИнструкцияPrint
             | ИнструкцияWith | ИнструкцияOnError

ПустаяИнструкция ::= ;

ИнструкцияВыражение ::= Выражение ;

ИнструкцияIf ::= IF Выражение THEN { Инструкция } 
                 [ ELIF Выражение THEN { Инструкция } ] 
                 [ ELSE { Инструкция } ] END

ИнструкцияWhile ::= WHILE Выражение DO { Инструкция } END

ИнструкцияReturn ::= RETURN [ Выражение ]

ИнструкцияPrint ::= PRINT ( [ ФорматнаяСтрока , ] { Выражение } ) ;

ИнструкцияWith ::= WITH Выражение DO { Инструкция } END

ИнструкцияOnError ::= ONERROR { Инструкция } DO { Инструкция } END

Выражение ::= ПростоеВыражение [ ОператорСравнения ПростоеВыражение ]

ПростоеВыражение ::= [ + | - ] Терм { ( + | - | or ) Терм }

Терм ::= Фактор { ( * | / | % | and ) Фактор }

Фактор ::= ( Выражение ) | Литерал | Имя | ВызовПроцедуры | NEW Имя

ВызовПроцедуры ::= Имя ( [ Выражение { , Выражение } ] )

Литерал ::= Число | Строка | TRUE | FALSE | NULL
```

## Часть 3. Отладчик макропрограмм

### 3.1. Запуск отладчика

Отладчик запускается через IDE: **Отладка → Старт** или клавиша `F9`.

Также возможен запуск из командной строки:
```bash
rsldebug имя_макрофайла.rsl
```

### 3.2. Основное окно отладчика

Основное окно содержит:
- **Панель исходного кода** — отображение текущего макрофайла с подсветкой текущей строки
- **Панель переменных** — значения локальных и глобальных переменных
- **Панель выражений** — результаты вычисления выражений
- **Панель стека** — стек вызовов процедур
- **Панель вывода** — результаты операций PRINT

### 3.3. Отладка макропрограмм

#### 3.3.1. Точки останова

Установка/снятие точки останова: `F9` или клик на поле слева от номера строки.

Точки останова позволяют приостановить выполнение программы в определенном месте для анализа состояния.

#### 3.3.2. Список точек останова

Просмотр всех точек останова: **Отладка → Точки останова** или `Ctrl+Shift+B`.

#### 3.3.3. Список импортируемых модулей

Просмотр импортированных модулей: **Вид → Модули**.

### 3.4. Окно проверки

#### 3.4.1. Стек вызовов процедур

Просмотр стека: **Вид → Стек вызовов** или `Ctrl+Alt+S`.

Стек отображает последовательность вызовов процедур от текущей точки выполнения до начальной.

#### 3.4.2. Окно переменных

Просмотр значений переменных: **Вид → Переменные** или `Ctrl+Alt+V`.

Отображаются:
- Локальные переменные текущей процедуры
- Глобальные переменные
- Параметры процедур
- Специальные переменные

#### 3.4.3. Окно выражений

Вычисление произвольных выражений: **Вид → Выражения** или `Ctrl+Alt+E`.

#### 3.4.4. Ввод выражения

Ввод выражения для вычисления: `Ctrl+Shift+E`.

#### 3.4.5. Редактирование выражения

Изменение ранее введенного выражения: двойной клик на выражении в окне.

### 3.5. Настройка отладчика

#### 3.5.1. Параметры отладки

Настройка через: **Сервис → Настройки отладчика**.

Доступные параметры:
- Автоматическое обновление переменных
- Показывать встроенные процедуры
- Останавливаться при ошибке
- Логировать операции в файл
