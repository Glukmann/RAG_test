## Импорт модулей (IMPORT)

Директива `IMPORT` позволяет включить в текст макромодуля информацию из других файлов.

```rsl
IMPORT blnc, common;
```

**Важно:** Директива `IMPORT` должна находиться **вне** определения макропроцедур.

### Поддерживаемые типы модулей

| Тип модуля | Расширение | Примечание |
|------------|------------|------------|
| RSL-модули | `.MAC` | Имя указывается без расширения |
| DLM-модули | `.D32`, `.D16`, `.DPM16`, `.DLM` | Созданы с помощью DLM SDK |
| Встроенные модули | — | Написаны на C/C++ |

```rsl
IMPORT BankInter, Баланс;
```

### Порядок поиска файлов

1. Среди стандартных модулей на C/C++
2. Среди DLM-модулей
3. Среди макромодулей в Btrieve-файле `btrmac.ddf`
4. Среди текстовых файлов `.MAC` (сначала ищется `.RSM` — прекомпилированный код, затем `.MAC`)

## Загрузка и кэширование модулей (ПЭИ)

В процессе выполнения RSL-программы исполняющая система создаёт в памяти **поименованные экземпляры интерпретатора (ПЭИ)**.

- ПЭИ с именем `default` создаётся всегда
- ПЭИ состоит из **постоянной** части (кэшируется) и **отгружаемой** части (выгружается после выполнения)
- В постоянной части сохраняются системные модули

### Автозагрузка модулей

Чтобы пользовательские модули попали в постоянную часть всех ПЭИ, включите их в `autoinc.mac`:

```rsl
/* Пример содержимого модуля autoinc.mac: */
IMPORT a /* В результате модуль а будет включен в */
         /* постоянную часть всех существующих ПЭИ */
```

Для конкретного ПЭИ используйте файл с именем ПЭИ, например `default.mac`.

## Конструкции языка RSL

RSL предоставляет программисту широкие возможности по управлению данными. В качестве конструкций языка используются:

- **Выполняемые инструкции**
- **Определение объектов и процедур**

### Соглашения в описании синтаксиса

| Обозначение | Значение |
|-------------|----------|
| `[ ... ]` | Необязательные элементы |
| `{ ... }` | Элементы, которые могут повторяться или отсутствовать |

---

## Пустая инструкция

Пустая инструкция состоит только из разделителя — точки с запятой. Используется там, где по правилам языка могла бы находиться инструкция, а по логике программы ничего выполнять не надо:

```rsl
if ( aa != 10 ); /* Пустая инструкция */
else
    bb = aa
end
```

---

## Инструкция "выражение"

Любое выражение RSL. Вычисленное значение отбрасывается.

```rsl
a;
a + b;
a + (b = 12);
a = 10;
d + f2 (12, c = b = 20);
a = b = c = d = 10;
```

## Условная инструкция IF

### Синтаксис

```rsl
IF '(' условное_выражение ')'
    список_инструкций
{ ELIF '(' условное_выражение ')'
    список_инструкций }
[ ELSE
    список_инструкций ]
END
```

Количество конструкций `ELIF` не ограничено.

### Пример

```rsl
if ( a < 10 )
    a = 10
elif ( a < 20 )
    a = 20
elif ( (a < 100) and (a > 50) )
    a = 0
else
    a = b;
    b = 0
end
```

**Логика работы:**
1. Первым анализируется условие после `IF`. Если истинно — выполняется соответствующий блок.
2. В противном случае последовательно анализируются условия после `ELIF` (сверху вниз).
3. Если какое-либо условие истинно — выполняется его блок, остальные условия не анализируются.
4. Если все условия ложны и присутствует `ELSE` — выполняется блок `ELSE`.

## Инструкция цикла WHILE

Для организации циклов в RSL используется инструкция `WHILE`.

### Синтаксис

```rsl
WHILE '(' условное_выражение ')'
    список_инструкций
END
```

Список инструкций выполняется до тех пор, пока условное выражение остаётся истинным. Если условие ложно до входа в цикл — список инструкций не выполняется ни разу.

### Пример

```rsl
a = 0;
while ( a < 10 )
    a = a + 1;
    println (a)
end
```

## Инструкция возврата RETURN

Инструкция `RETURN` применяется для выхода из процедуры или завершения всей RSL-программы. Во втором случае она должна быть указана вне любой процедуры.

### Синтаксис

```rsl
RETURN [выражение];
```

Выражение, заданное после слова `RETURN`, определяет возвращаемое процедурой значение. Если в инструкции завершения RSL-программы в качестве выражения задать текст (символьная константа или переменная типа `V_STRING`), он будет выведен на стандартное устройство вывода.

## Инструкция вывода

Инструкция вывода позволяет определить в тексте программы форму для вывода в стандартный выходной поток. Выводимая форма заключается в квадратные скобки.

### Базовый синтаксис

```rsl
'[' управляющая_строка ']' [ '(' фактические_параметры ')' ]
```

### Пример формы вывода

```rsl
[ Номер счета ##########
  Остаток    ########## ] ( Account, Summa )
```

В результате в выходной поток будет выведен номер счёта из переменной `Account` и остаток из переменной `Summa`.

### Формат управляющей строки

Управляющая строка представляет собой последовательность печатных и управляющих символов.

| Символ | Значение |
|--------|----------|
| `#` | Поле для вывода значения из списка параметров. Количество символов `#` задаёт ширину поля |
| Остальные символы | Печатные символы, выводятся без изменений |

### Пример ширины поля

```rsl
[ ########## ] (123.45) /* Вывод поля шириной 10 символов */
```

### Позиция табуляции (одиночный `#`)

Одиночный управляющий символ `#` задаёт не ширину поля, равную 1, а **позицию табуляции** — точку, относительно которой происходит выравнивание:

| Спецификатор | Выравнивание |
|--------------|--------------|
| `l` | По левому краю |
| `r` | По правому краю |
| `c` | По центру |

```rsl
a = "Пример";
[ |
  #
  #
  # ] ( a:l, a:r, a:c )
```

Вывод:
```
Пример
Пример
Пример
```

## Определение переменных VAR

Имена переменных RSL по умолчанию считаются объявленными в момент их первого появления в тексте программы. Однако для избежания ошибок из-за неточного написания имён рекомендуется использовать явное объявление через `VAR`.

### Синтаксис

```rsl
[local | private] VAR идентификатор [ : имя_типа ] [ '=' выражение ]
    { ',' идентификатор [ : имя_типа ] [ '=' выражение ] }
```

**Важно:** После первого обнаружения `VAR` в тексте программы, любая необъявленная явно переменная в текущем RSL-модуле приведёт к сообщению об ошибке.

### Пример

```rsl
var
    Остаток40 : money,
    Дата = date;
```

## Определение классов и объектов

В языке RSL имеется возможность создавать классы и объекты. Классы и объекты используются для поддержки программирования визуальной среды: отчёты, кнопки, поля редактирования, переключатели — всё это можно представить в качестве объектов со своими свойствами и методами.

### Синтаксис определения класса

```rsl
[local | private] CLASS [ '(' идентификатор ')' ] идентификатор [ '(' список_формальных_параметров ')' ]
    <свойства класса>;
    <методы класса>;
End
```

- **Идентификатор класса** — имя конструктора класса, используется при создании экземпляра
- **Список формальных параметров** — параметры инициализатора объекта класса
- **Все локальные переменные** внутри класса — это свойства класса
- **Все локальные макропроцедуры** — методы класса
- **Любой код внутри класса**, не включённый в методы — код конструктора

### Пример: класс "Персона"

```rsl
Class Персона (p1, p2)
    Var Имя, Фамилия;
    
    Macro Отчет
        Println ("Имя:", Имя);
        Println ("Фамилия:", Фамилия);
    End;
    
    Имя = p1;
    Фамилия = p2;
End;
```

### Создание объекта (экземпляра класса)

```rsl
Obj = Персона ("Иван", "Петров");
```

В данной инструкции объект одновременно создаётся и инициализируется. Ссылка на объект присваивается переменной `Obj`.

### Автоматическое создание объектов (версия 852+)

Начиная с версии интерпретатора 852, добавлена возможность автоматического создания объектов при первом обращении к декларированной переменной класса:

```rsl
Class TestClass
    Var prop = 10;
End;

Var Ob:TestClass;        /* Декларируем переменную типа TestClass */
Println (Ob.prop);       /* Объект автоматически создаётся */
```

**Жизненный цикл объекта:** Созданный объект существует до тех пор, пока на него есть хотя бы одна ссылка. Ссылки хранятся в переменных RSL. При очистке переменной происходит удаление ссылки на объект.

### Обращение к свойствам и методам класса

```rsl
Obj.Имя;      /* Доступ к свойству по имени */
Obj.Отчет;    /* Вызов метода */
```

### Доступ к свойствам по индексу

```rsl
Class Test
    Var prop1, prop2;
End;

Ob = Test;
A = ob.prop1;    /* Доступ к свойству по имени */
B = ob (0);      /* Доступ к свойству по индексу (prop1) */
```

### Свойство по умолчанию

Для совместимости со стандартом автоматизации добавлена поддержка свойства по умолчанию. Свойством по умолчанию считается свойство, имя которого в выражении можно не указывать:

```rsl
/* Объект ob содержит свойство по умолчанию item */
ob. (10) = 100;      /* Эквивалентно: ob.item (10) = 100; */
```

### Методы с параметрами

Если метод имеет параметры, они указываются в круглых скобках:

```rsl
A = ob.met (3);
```

Классы также могут иметь свойства с параметрами (создаются через DLM SDK):

```rsl
A = ob.prop (3);
```

### Деструктор

Классы могут содержать деструктор, определяемый пользователем. Для его использования необходимо указать метод с предопределённым именем `Destructor`:

```rsl
Class Test
    macro Destructor
        println ("Destructor called");
    end;
End;
```

Деструктор вызывается автоматически при "разрушении" объекта. Если деструктор не определён, ресурсы освобождаются автоматически.

### Наследование классов

Object RSL позволяет наследовать классы от других классов. Базовый класс указывается в круглых скобках после ключевого слова `CLASS`. Для инициализации базового класса необходимо вызвать предопределённый метод, название которого образуется путём добавления к имени класса приставки `Init`.

**Важно:** Множественное наследование не поддерживается.

### Пример наследования

Унаследуем класс "Сотрудник" от класса "Персона", добавив свойство "Должность":

```rsl
Class ( Персона ) Сотрудник ( п_Имя, п_Фамилия, п_Должность )
    Var Должность;
    
    Macro Отчет
        Отчет;                           /* Вызов метода родительского класса */
        Println ("Должность:", Должность);
    End;
    
    InitПерсона (п_Имя, п_Фамилия);
    Должность = п_Должность;
End;
```

### Скрытый параметр this

В каждый метод класса передаётся скрытый параметр `this`, представляющий собой ссылку на объект, для которого вызывается метод или свойство.

### Виртуальные методы

Все методы классов являются виртуальными. Добавление в дочерний класс метода с именем, уже используемым в базовом классе, вызовет замену метода родительского класса методом дочернего.

### Наследование от внешних классов

RSL-классы можно наследовать от классов, определённых в DLM-модулях, а также от встроенных классов (`TBFile`, `TDirList` и т.п.).

**Особенность:** Невозможно обращаться напрямую к методам родительского класса после их переопределения в дочернем классе. Для этих целей служит стандартное свойство `_extObj`, возвращающее ссылку на внешний объект:

```rsl
class (TBFile) TBfileEx (name, mode, key)
    InitTBFile (name, mode, key);
    
    macro GetDirect (pos)
        return _extObj.GetDirect (pos);
    onerror
        println (status);
        if (status == 43)
            return false;
        end;
        RunError;
    end;
End;

ob = TBfileEx ("client.dbt", "w", 0);
ob.next;
pos = ob.getPos;
println (ob.GetDirect (pos));
```

## Определение символических констант CONST

Все символические константы должны быть объявлены явно при помощи определения `CONST`. Попытка изменить значение любой из объявленных констант приведёт к ошибке при компиляции.

### Синтаксис

```rsl
[local | private] CONST идентификатор [ : имя_типа ] '=' выражение
    { ',' идентификатор [ : имя_типа ] '=' выражение }
```

### Пример

```rsl
const
    Количество = 35,
    ФИО = "Иванов Иван Иванович";
```

## Определение процедуры MACRO

Определение `MACRO` используется для описания процедуры RSL.

### Синтаксис

```rsl
[local | private] MACRO идентификатор [ '(' список_формальных_параметров ')' ] [ : имя_типа ]
    список_определений_и_инструкций
END
```

- **Идентификатор процедуры** — имя, используемое для вызова процедуры
- **Список формальных параметров** — разделённые запятыми идентификаторы переменных

### Формат списка параметров

```rsl
идентификатор [ : имя_типа ] { , идентификатор [ : имя_типа ] }
```

Список определений и инструкций может содержать любые инструкции и определения языка RSL, в том числе и другие определения `MACRO`. Таким образом, структура процедуры повторяет структуру RSL-программы.

### Пример процедуры

```rsl
/* Пример определения процедуры, печатающей содержимое файла: */
MACRO Содержимое (ff)
    WHILE (next (ff))
        [ # |############## | # ]
        (ff.Balance, ff.Name_Part, ff.Part)
    END;
END;
```

## Процедуры языка RSL

Процедуры являются неотъемлемой частью любой программы, написанной на RSL. Они являются фрагментом программы со своим именем, к которому можно обратиться для выполнения необходимых действий.

- Допускается использовать стандартные процедуры, входящие в состав языка RSL
- Для косвенного вызова процедуры служит стандартная процедура `execmacro`
- Процедуры RSL могут присутствовать в выражениях, при этом вместо процедуры в выражение будет подставлено возвращаемое значение
- Если процедура явно не возвращает значение — возвращаемое значение будет иметь тип `V_UNDEF`

---

## Передача параметров в процедуры

### Объявление формальных параметров

```rsl
macro My (a, b, c, d)
```

Здесь `a`, `b`, `c`, `d` — формальные параметры, создаваемые и инициализируемые значениями фактических аргументов при входе в процедуру. После завершения процедуры эти переменные прекращают своё существование.

**Нумерация параметров начинается с нуля.**

### Передача по значению и по ссылке

Передача параметров производится **по значению**, однако параметры типа `ARRAY`, `RECORD` и `FILE`, а также все объекты RSL-классов передаются **по ссылке**.

### Модификация параметров (передача по ссылке)

Для изменения значений фактических параметров в вызывающей процедуре необходимо:

1. При указании типа параметров перед декларацией типа должен находиться символ `@`
2. Фактические параметры при вызове процедуры должны быть переданы по ссылке (с `@`)

```rsl
/* Пример модификации параметров: */
macro Test (p1:@variant, p2:@integer)
    p1 = 10;
    p2 = 20;
end;

var v1, v2;
Test (@v1, @v2);
/* В результате переменная v1 будет равна 10, а v2 равна 20. */
```

### Процедура setparm

Альтернативный способ изменения фактических параметров — процедура `setparm`:

```rsl
setparm (0, 10+aa)        /* Замена первого параметра на значение 10+aa */
setparm (1, Local (bb))   /* Замена второго параметра на значение Local (bb) */
```

### Пример процедуры с setparm

```rsl
macro Demo (number, text)
    setparm (0, 10);
    setparm (1, "Привет")
end;

/* Теперь после вызова: */
var id, name;
Demo (id, name)
/* переменная id получит значение 10, а name — значение "Привет" */
```

### Процедура getparm

Если количество фактических параметров меньше количества формальных — вместо недостающих передаются неопределённые значения типа `V_UNDEF`. Если фактических больше — доступ к избыточным параметрам получается через `getparm`.

```rsl
getparm (0, parm1)   /* Присвоить parm1 значение первого параметра (№ 0) */
getparm (1, parm2)   /* Присвоить parm2 значение второго параметра (№ 1) */
```

### Процедура с произвольным количеством параметров

```rsl
macro Calc
    var sum = 0, i, val;
    while (getparm (i, val))
        sum = sum + val;
        i = i + 1;
    end;
    return sum
end

/* Вызовы: */
sum = Calc (1, 2, 3, 4, 5);
sum = Calc (10, 20);
```

## Определение массивов

В языке RSL можно объявлять только **одномерные массивы**. Для этого предусмотрены:

1. Конструкция `ARRAY` (устаревшая, поддерживается для совместимости)

## Определение массивов с помощью конструкции ARRAY

### Синтаксис

```rsl
[local | private] ARRAY идентификатор { ',' идентификатор }
```

- Идентификатор массива — имя объекта, через которое можно ссылаться на массив
- В выражении идентификатор массива преобразуется к переменной типа `V_ARRAY` (ссылка на массив)
- Элементы массива могут хранить значения любых типов, включая `V_ARRAY`
- Нумерация элементов начинается с **0**
- Размер массива изменяется **динамически**

### Пример

```rsl
/* Определение пустых массивов */
array MyVar, First, Second;

/* Присвоение значений элементам массива First */
First (0) = "Имя";
First (1) = Second;

/* Присвоение значений элементам массива Second */
Second (0) = 1;
Second (1) = 23;
Second (2) = 567;

/* Определение размера массива MyVar */
asize (MyVar, 10);

/* Присвоение значения элементу массива MyVar */
MyVar (9) = First;

[##############] (MyVar (9)(1)(2));
/* В стандартный выходной поток будет выведено: 567 */
/* Конструкция MyVar (9)(1)(2) эквивалентна MyVar (9, 1, 2) */
```

### Определение размера массива

```rsl
cc = asize (aa);        /* Получить текущий размер массива aa */
asize (MyVar, 10);     /* Изменить размер массива (старое содержимое удаляется) */
```

**Рекомендация:** Используйте `asize` с двумя параметрами, чтобы уменьшить количество перераспределений памяти.

## Копирование массивов

Если в левой и правой части инструкции присваивания стоят массивы или переменные типа `V_ARRAY`, то происходит **копирование массивов**:

```rsl
dd = aa;    /* dd — переменная типа V_ARRAY, ссылается на массив aa */
bb = dd;    /* Создаётся копия массива dd */
```

### Пример работы с размером массива

```rsl
asize (aa, 10);     /* Установить для массива aa новый размер 10 */
i = 0;              /* Цикл присвоения элементам новых значений */
while (i < 10)
    aa (i) = i;
    i = i + 1;
end;
```

## Стандартный класс TArray

Стандартный класс `TArray` языка RSL используется для реализации **динамического массива**. Имеется возможность замены конструкции `ARRAY` на новую конструкцию `TArray`:

```rsl
Array myArray;          /* Прежняя конструкция */
myArray = TArray;       /* Замена прежней конструкции на новую */
```

### Конструктор TArray

```rsl
TArray [ ( n1 [, n2] ) ]
```

Параметры конструктора:

| Параметр | Описание | Значение по умолчанию |
|----------|----------|----------------------|
| `n1` | Количество элементов на первой странице | 50 |
| `n2` | Количество элементов на последующих страницах | 10 |

```rsl
/* Опишем массив, в котором первая страница содержит 100 элементов, а последующие — по 5: */
MyArray = TArray (100, 5);
```

В качестве параметра можно указать ссылку на уже существующий массив — в этом случае создаётся новый массив, содержащий копию элементов.

### Доступ к элементам TArray

```rsl
FirstArray = TArray;
FirstArray (0) = 100;           /* Присвоение значения */
SecondArray = TArray (FirstArray);  /* Создаётся копия массива FirstArray */
```

### Свойства класса TArray

| Свойство | Описание |
|----------|----------|
| `Size` | Текущий размер массива. Присваивание нового значения изменяет размер |
| `Value(N)` | Значение N-го элемента массива |
| `CvtToSafeArray` | Если `TRUE`, объект автоматически конвертируется в `SAFEARRAY` при передаче в ActiveX |

```rsl
Println ("Текущий размер массива:", FirstArray.Size);
```

### Разрешение неоднозначности (квадратные скобки)

Для разрешения неоднозначности между индексом и свойством с параметрами можно использовать квадратные скобки:

```rsl
Ob = TArray;
Ob [5] = 100;       /* Эквивалентно Ob (5) = 100, но без неоднозначности */
```

### Пример: обращение к элементам массива тремя способами

```rsl
class TTT             /* Определяем класс ТТТ */
    var prop = TArray;
    prop (5) = 200;
    macro Me
        return prop;
    end;
end;

ob = TTT;             /* Создаём экземпляр класса TTT */

/* Распечатаем значение пятого элемента массива тремя способами: */
println (ob.prop() (5));
println (ob.prop.Value (5));
println (ob.prop [5]);
```

## Определения FILE и RECORD

Конструкции `FILE` и `RECORD` используются для объявления объектов типа `FILE`, `DBFFILE`, `TXTFILE` и `RECORD`.

### Различия между FILE и RECORD

| Конструкция | Тип объекта по умолчанию |
|-------------|-------------------------|
| `FILE` | `FILE` (Btrieve-файл) |
| `RECORD` | `RECORD` (структура) |

### Синтаксис

```rsl
[local | private] FILE идентификатор (имя_объекта [, имя_словаря]) [список_параметров_объекта]
[local | private] RECORD идентификатор (имя_объекта [, имя_библиотеки_диалогов]) [список_параметров_объекта]
```

### Спецификаторы параметров

| Спецификатор | Описание |
|--------------|----------|
| `SORT <number>` | Номер ключевого индекса файла (только для `FILE`) |
| `KEY <number>` | Эквивалент `SORT` |
| `WRITE` | Режим открытия на запись |
| `APPEND` | Режим дозаписи для текстового файла |
| `MEM` | Спецификатор структуры (умолчание для `RECORD`) |
| `TXT` | Спецификатор текстового файла |
| `DBF` | Спецификатор файла формата DBF |
| `DIALOG` | Структура диалогового окна |
| `BTR` | Объект типа `FILE` (умолчание для `FILE`) |

### Эквивалентности

```rsl
file id (name)           равносильно   record id (name) btr;
record id (name)         равносильно   file id (name) mem;
```

### Примеры определений

#### 1. Btrieve-файл на запись

```rsl
File Счета (account) write;
/* или */
Record Счета (account) btr write;
```

#### 2. Btrieve-файл из словаря с указанием пути

```rsl
File Счета (account, "mydir\\mydic.def") write;
```

#### 3. Файл формата DBF

```rsl
File demo ("demo.dbf") dbf;
/* или */
Record demo ("demo.dbf") dbf;
```

#### 4. Текстовый файл

```rsl
File tfile ("demo.txt") txt;
/* или */
Record tfile ("demo.txt") txt;
```

#### 5. Текстовый файл в режиме дозаписи

```rsl
File ff ("demo") txt append;
/* или */
Record tfile ("demo") txt append;
```

#### 6. Структура файла (RECORD)

```rsl
File rec (account) mem;
/* или */
Record rec (account);
```

#### 7. Структура диалогового окна

```rsl
File dlg (demo) mem dialog;
/* или */
Record dlg (demo) dialog;
```

#### 8. Диалоговое окно из библиотеки с указанием пути

```rsl
Record dlg (demo, "mydir\\mylib.lbr") dialog;
```

### Доступ к полям объекта

Доступ к полям осуществляется по именам полей (из словаря или библиотеки диалогов) или по индексу поля (нумерация с нуля):

```rsl
Номер = aa.Account;         /* По имени поля */
Имя = Клиенты.Name;         /* По имени поля */
Номер = aa (0);              /* По индексу */
Имя = Клиенты (10);          /* По индексу */
```

## Поддержка технологии ActiveX в RSL

В Object RSL предусмотрена поддержка **ActiveX-объектов** (объектов автоматизации). Пользователи могут создавать ActiveX-объекты и обращаться к их свойствам и методам из RSL-программы.

### Требования

Необходимо импортировать модуль `rslx`:

```rsl
import rslx;
```

### Конструктор ActiveX

```rsl
ACTIVEX (имя_объекта [, имя_удаленного_компьютера [, ссылка_на_объект]])
```

| Параметр | Описание |
|----------|----------|
| `имя_объекта` | Идентификатор класса COM-объекта (progID) |
| `имя_удаленного_компьютера` | Имя компьютера для создания удалённого объекта (необязательно) |
| `ссылка_на_объект` | Если `TRUE` — возвращает ссылку на существующий объект, не создаёт новый |

### Примеры использования ActiveX

#### Пример 1: Запись строки в MS Word

```rsl
ob = ActiveX ("Word.basic");
ob.FileNew;
ob.Insert ("Hello, World!");
ob.AppShow;
```

#### Пример 2: Работа с Excel

```rsl
import rslx;
ob = ActiveX ("Excel.Application", NULL, TRUE);
ob.Visible = TRUE;
ob.Workbooks.add;
MsgBox ("Нажмите любую клавишу для выхода из Excel");
ob.Quit
```

### Альтернативный способ создания через GenObject

```rsl
ob = GenObject ("ActiveX\\Word.basic")
```

## Поддержка стандартных коллекций

Object RSL позволяет использовать стандартные коллекции объектов автоматизации. Для создания коллекций и доступа к их содержимому существует специальный стандарт.

### Объект-перечислитель (CreateEnum)

Для создания объекта-перечислителя используется предопределённый метод `CreateEnum`, автоматически добавляемый к каждому ActiveX-объекту:

```rsl
En = ob.Workbooks.CreateEnum;
While (En.Next)
    MsgBox (En.Item.Name)
End;
```

### Методы объекта-перечислителя

| Метод | Описание |
|-------|----------|
| `Next` | Переход к следующему элементу. Возвращает `FALSE` когда коллекция просмотрена до конца |
| `Reset` | Сброс перечислителя в начальное состояние |
| `Item` | Свойство, содержащее ссылку на текущий элемент |

### Пример: коллекция TArray

```rsl
Ar = TArray;
Ar (0) = "Element 1";
Ar (1) = "Element 2";
En = Ar.CreateEnum;
While (En.Next)
    Println (En.Item)
End;
```

## Обращение к свойствам объектов с параметрами

Объекты автоматизации могут содержать свойства с параметрами, к которым можно обращаться из Object RSL.

### Пример: двумерный массив XArray

```rsl
ob = ActiveX ("XArray.XArray");
ob.ReDim (0, 10, 0, 20);     /* Две размерности: 0-10 и 0-20 */
i = 0;
while (i < 10)
    ob.Value (i, 0) = "String " + i;
    i = i + 1
end;
```

### Разрешение неоднозначности для TArray

При появлении свойств с параметрами возникает неоднозначность с индексами массива. Для разрешения используйте:

```rsl
/* Раньше: */
Ob.prop (0) = "Строка1";     /* Теперь интерпретируется как параметр свойства prop */

/* Теперь — для доступа к элементам массива: */
Ob.prop () (0) = "Строка1";  /* Дополнительные скобки () для доступа к массиву */
Ob.prop [0] = "Строка1";     /* Квадратные скобки для индекса */
Ob.prop.Value (0) = "Строка"; /* Через свойство Value */
```

Все объекты классов Object RSL являются объектами автоматизации и могут быть доступны из других языков программирования.

```rsl
import rslx;

vb.TestRSL (rsl);
```

## Обработка событий (TRslEvSource / TRslEvHandler)

Объекты автоматизации могут иметь не только свойства и методы, но и **события** — способы уведомления клиента об изменениях. Object RSL позволяет обрабатывать эти события.

### Класс TRslEvHandler

Стандартный класс `TRslEvHandler` предназначен для обработки событий от ActiveX-объектов. Способы использования:

1. Унаследовать класс-обработчик событий от `TRslEvHandler`
2. Создать экземпляр `TRslEvHandler` и передать объект-обработчик событий в конструктор
3. Создать экземпляр без объекта-обработчика — для использования глобальных макропроцедур

**Глобальный экземпляр:** RSL автоматически создаёт объект `RslEvHandler`.

### Конструктор TRslEvSource

```rsl
TRslEvSource ( [ handlerObj ] [, doAddRef ] )
```

| Параметр | Описание |
|----------|----------|
| `handlerObj` | Объект-обработчик событий. Если `NULL` или не задан — используются глобальные макропроцедуры |
| `doAddRef` | Если `TRUE` (по умолчанию), инкрементирует счётчик ссылок на `handlerObj` |

### Свойство EvSource

Обеспечивает доступ к внутренней коллекции объектов-источников событий.

#### Способ 1: С использованием библиотеки типов

```rsl
EvSource ( pref [, intfName ] ) = ob
```

| Параметр | Описание |
|----------|----------|
| `pref` | Уникальная строка для формирования имени метода обработки: `pref_<имя_события>` |
| `intfName` | Имя интерфейса событий из библиотеки типов (необязательно) |
| `ob` | ActiveX-объект — источник событий |

#### Способ 2: Без библиотеки типов (более эффективный)

```rsl
EvSource ( pref, GUID, numProc ) = ob
```

| Параметр | Описание |
|----------|----------|
| `pref` | Строка с именем объекта-источника во внутренней коллекции |
| `GUID` | Уникальный идентификатор интерфейса событий, например `"{00024413-0000-0000-C000-000000000046}"` |
| `numProc` | Количество процедур обработчиков, которые будут добавлены через `SetHandler` |

**Для прекращения обработки событий** присвойте свойству `EvSource` с соответствующим `pref` значение `NULL`.

### Свойство TypeLib

```rsl
TypeLib = fileName
```

Имя файла, содержащего библиотеку типов. Необходимо только если RSL не может получить информацию автоматически.

### Метод SetHandler

Устанавливает связь между событием и методом обработки:

```rsl
SetHandler ( pref, proc, id )
```

| Параметр | Описание |
|----------|----------|
| `pref` | Имя объекта-источника во внутренней коллекции |
| `proc` | Имя метода обработчика события |
| `id` | Целочисленный идентификатор события |

### Метод RemHandler

Разрывает связь события и процедуры обработчика:

```rsl
RemHandler ( pref, id )
```

### Метод raise

Применяется когда источниками событий являются объекты Object RSL. Для этого создаётся класс-наследник `TRslEvHandler`. Событие происходит в результате вызова `raise`:

```rsl
raise ( имя_события [, параметры... ] )
```

### Пример: класс-источник событий

```rsl
import rslx;

class (TRslEvHandler) Source
    macro Test
        raise ("onEvent1", "Это параметр события");
    end;
end;

/* Объект-обработчик */
class (TRslEvHandler) Handler (ob)
    EvSource ("OB") = ob;
    
    macro OB_onEvent1 (par)
        MsgBox ("Событие OnEvent1 с параметром: ", par)
    end;
end;

ob = Source;
ev = Handler (ob);
ob.Test
```

### Примеры обработки событий Excel

#### Пример 1: Глобальный объект RslEvHandler

```rsl
import rslx;
ob = ActiveX ("Excel.Application");
ob.Visible = true;
RslEvHandler.EvSource ("Excel", "Application") = ob;
ob.Workbooks.add;

macro Excel_NewWorkBook (wb)
    println ("New Book: ", wb.Name)
End;

MsgBox ("OK");
RslEvHandler.EvSource ("Excel") = NULL;
ob.Quit;
```

#### Пример 2: Специальный класс-обработчик и экземпляр TRslEvHandler

```rsl
import rslx;

class THandler
    macro Excel_NewWorkBook (wb)
        println ("New Book: ", wb.Name)
    End;
end;

handler = TRslEvHandler (THandler);
ob = ActiveX ("Excel.Application");
ob.Visible = true;
handler.EvSource ("Excel", "Application") = ob;
ob.Workbooks.add;
MsgBox ("OK");
handler.EvSource ("Excel") = NULL;
ob.Quit;
```

#### Пример 3: Экземпляр TRslEvHandler как свойство класса (без циклических ссылок)

```rsl
import rslx;

class THandler
    evObj = TRslEvHandler (this, false);   /* false — без инкремента счётчика ссылок */
    
    macro Excel_NewWorkBook (wb)
        println ("New Book: ", wb.Name)
    End;
end;

handler = THandler;
ob = ActiveX ("Excel.Application");
ob.Visible = true;
handler.evObj.EvSource ("Excel", "Application") = ob;
ob.Workbooks.add;
MsgBox ("OK");
handler.evObj.EvSource ("Excel") = NULL;
ob.Quit;
```

#### Пример 4: Класс-обработчик, унаследованный от TRslEvHandler

```rsl
import rslx;

class (TRslEvHandler) THandler
    macro Excel_NewWorkBook (wb)
        println ("New Book: ", wb.Name)
    End;
end;

handler = THandler;
ob = ActiveX ("Excel.Application");
ob.Visible = true;
handler.EvSource ("Excel", "Application") = ob;
ob.Workbooks.add;
MsgBox ("OK");
handler.EvSource ("Excel") = NULL;
ob.Quit;
```

#### Пример 5: Без библиотеки типов (вторая форма EvSource)

```rsl
import rslx;

class (TRslEvHandler) THandler
    macro Excel_NewWorkBook (wb)
        println ("New Book: ", wb.Name)
    End;
end;

handler = THandler;
ob = ActiveX ("Excel.Application");
ob.Visible = true;
handler.EvSource ("Excel", "{00024413-0000-0000-C000-000000000046}", 1) = ob;
handler.SetHandler ("Excel", "Excel_NewWorkBook", 1565);
ob.Workbooks.add;
MsgBox ("OK");
handler.EvSource ("Excel") = NULL;
ob.Quit;
```

## Передача параметров в методы ActiveX

### Модификаторы параметров

| Модификатор | Описание |
|-------------|----------|
| `:i` | Передача параметра по ссылке |
| `:iv` | Передача как ссылки на вариант (VARIANT). Метод может менять значение и тип |
| `:v` | Передача по значению (для Visual RSL, где по умолчанию передача по ссылке) |

### Пример передачи по ссылке

```vb
' Visual Basic:
Sub BasicProc (ByRef par1 As String, ByRef par2 As Variant)
    Par1 = "New string";
    Par2 = 10;
End Sub
```

```rsl
/* Object RSL: */
Val1 = "Old string 1";
Val2 = "Old string 2";
Ob.BasicProc (Val1:i, Val2:iv)
```

### Передача TArray в ActiveX (SAFEARRAY)

По умолчанию `TArray` передаётся как объект со свойствами `Size` и `Value`. Для автоматической конвертации в `SAFEARRAY`:

```rsl
Ar = TArray;
Ar.Value (0) = "value for array";
Ar.CvtToSafeArray = true;
/* При передаче в ActiveX массив Ar автоматически преобразуется в SAFEARRAY */
```

## Обработка ошибок времени выполнения (OnError)

В Object RSL можно осуществить "перехват" любой ошибки времени выполнения, не допустив аварийного завершения программы.

### Синтаксис обработчика ошибок

```rsl
MACRO Test
    <инструкции RSL>
OnError
    <инструкции для обработки ошибок>
End;
```

Если инструкция генерирует ошибку — управление передаётся на обработчик после `OnError`, аварийного завершения не происходит.

### Обработчик с информацией об ошибке

```rsl
MACRO Test
    <инструкции RSL>
OnError (er)
    MsgBox (er.Message, " | Модуль: ", er.Module, " | строка: ", er.Line)
End;
```

Переменная `er` получает ссылку на объект класса `TrslError`.

### Свойства объекта TrslError

| Свойство | Описание |
|----------|----------|
| `Code` | Код ошибки |
| `Message` | Строка с описанием ошибки |
| `Module` | Название модуля RSL, вызвавшего ошибку |
| `Line` | Строка модуля, в которой произошла ошибка |
| `AxCode` | Код ошибки ActiveX-объекта |
| `AxMes` | Строка с информацией об ошибке ActiveX-объекта |

### Пример с ActiveX

```rsl
macro Demo
    ob = ActiveX ("Test.TestClass");
onError (er)
    println (er.Message);
    if (er.AxCode)
        println (er.AxMes);
    end
end /* Demo */
```

### Повторная генерация ошибки (RunError)

Чтобы сгенерировать ту же ошибку для передачи другому обработчику в цепочке вызовов:

```rsl
RunError;   /* Без параметров — повторно генерирует текущую ошибку */
```

### Обработчик ошибок в макромодуле

```rsl
Import rslx;

Macro Test
    /* ... */
End;

OnError (er)
    MsgBox (er.Message)
/* End может отсутствовать */
```

## Автоматическое создание объектов Object RSL

Объекты классов создаются автоматически при первом обращении к декларированной переменной типа класса:

```rsl
Class TestClass
    Var prop = 10;
End;

Var Ob:TestClass;        /* Декларируем переменную типа TestClass */
Println (Ob.prop);       /* Объект автоматически создаётся */
```
