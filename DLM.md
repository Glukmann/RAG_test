# DLM

Введение
При разработке ИБС RS-Bank V.6 учитывается технология работы и требования кредитных
организаций. В связи с тем, что банки могут использовать различные автоматизированные
системы, разработчики RS-Bank V.6 по мере необходимости обеспечивают совместную
работу этих систем с ИБС RS-Bank V.6.
Для взаимодействия различных клиентских приложений с ИБС RS-Bank V.6 специалистами
компании R-Style Softlab созданы специальные DLM-интерфейсы, позволяющие выполнять
экспорт и импорт данных, а также использовать функциональность ИБС RS-Bank V.6.

Инициализация DLM
Инициализация DLM осуществляется в макросе, при выполнении которого будет
производиться работа с данными ИБС RS-Bank V.6. В частности, для импорта документов
АС "Клиент-Сбербанк" в дистрибутиве поставляется макрос clb_dlm.mac. Получение
интерфейсов включает следующие этапы:
· Импорт DLM интерфейса для чтения (iabs) и/или для записи (iabsdw) с помощью
директивы import. Для этого перед непосредственной инициализацией DLM в макросе
должна присутствовать строка:
import iabs, iabsdw;
· Непосредственная 
инициализация

с

чтения данных, и ABSDirectWriteGetInterface([Oper:Integer,] [Password:String]), если
инициализируется модуль для записи данных.
Параметры:
Функции ABSGetInterface и ABSDirectWriteGetInterface содержат следующие
параметры:
Oper – номер операциониста.
Password – пароль.

## Класс: `IABSError`

```rsl
. С помощью методов этого объекта может быть получено подробное текстовое описание возникшей ошибки и причины, по которой она произошла. Следует отметить, что в некоторых кредитных организациях практикуется совместное использование DLM и стандартных RSL-модулей. При этом возникает ситуация блокировки таблиц базы данных, что приводит к невозможности доступа к необходимой информации через интерфейс. Во избежание такой проблемы следует инициализацию DLM и работу с ними выполнять в различных макросах . Особенности инициализации DLM при работе с RSL-модулями При выполнении макроса, в котором совместно используются DLM и стандартные RSL- модули, осуществляется открытие файлов с данными, необходимыми для работы как одних, так и других модулей. Однако открытие файлов для RSL-модулей выполняется при их импорте, а открытие файлов для работы с интерфейсом выполняется при инициализации соответствующего DLM . Так как в большинстве случаев импорт RSL-модулей выполняется раньше, чем инициализация DLM, то может возникнуть следующая ситуация: при попытке открытия с доступом на чтение файлов, необходимых для работы с интерфейсом iabs, оказывается, что эти файлы уже открыты с доступом на запись для работы RSL-модулей. Во избежание подобного рода проблем инициализацию DLM и работу с ними настоятельно рекомендуется организовывать в разных макрофайлах. Рассмотрим структуру макрофайлов подробнее: · Макрос инициализации, который будет выполняться при работе с интерфейсами к ИБС RS-Bank V.6. В частности, при импорте документов АС "Клиент-Сбербанк" этим макросом будет макрофайл с именем clb_dlm.mac. В макросе инициализации осуществляется: · Импорт DLM . · Инициализация DLM . На этом этапе открываются с доступом на чтение файлы, требуемые для работы с интерфейсом iabs, и с доступом на запись файлы, требуемые для работы с интерфейсом iabsdw. В случае успешной инициализации DLM будут сформированы объекты, реализующие эти интерфейсы. Такие объекты создаются временно и при дальнейшей работе не используются. Внимание! Временные объекты, реализующие интерфейсы, должны быть доступными для работы только в пределах макроса инициализации, поэтому при объявлении переменных этих объектов необходимо использовать модификатор private. Более подробное описание модификатора содержится в главе "Структура программы" Руководства программиста "Проблемно-ориентированный язык RSL" (Books\Tools\BnRSL.pdf). · Описание функции, которая является так называемой "оберткой" для макрофункции, непосредственно работающей с интерфейсом.
```
