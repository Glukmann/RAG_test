## Работа со словарём

### Создание словаря

В этом режиме пользователю предоставляется возможность создать новый словарь для своих приложений. Перед созданием система запрашивает имя нового словаря. Пользователю необходимо ввести любое имя с расширением `.def`.

### Закрытие словаря

При выборе этого режима происходит закрытие открытого словаря. При этом имя словаря, находившееся в правом верхнем углу экрана, исчезает.

После того как словарь был закрыт, пользователь не сможет работать ни в одном из режимов системы.

### Тестирование словаря

При выборе пользователем данного режима производится проверка соответствия структур, описанных в словаре, структурам физических файлов на диске.

В результате формируется протокол проверки, в который заносятся сообщения системы об обнаруженных несоответствиях.

Необходимо отметить, что эти сообщения не влияют на работу программного комплекса, поскольку вызваны несоответствием размера поля в описании Btrieve-файла с размером ключа Btrieve-файла. Более подробная информация о построении ключей Btrieve-файла содержится в документации, которая входит в комплект поставки Pervasive SQL.

#### Сообщения об ошибках тестирования

| Сообщение | Описание |
|-----------|----------|
| **Файл не найден** | Структура файла определена в словаре, а файл с такой структурой и именем на диске не существует |
| **Несоответствие размеров буферов (N1 – N2)** | Размер записи в байтах в словаре (`N1`) и в файле (`N2`) |
| **Отсутствует сегмент (N1 – N2)** | Номер ключа (`N1`) и номер сегмента (`N2`). Ключи и сегменты нумеруются, начиная с нуля |
| **Неверный размер или позиция сегмента N1-N2 (N3,N4-N5,N6)** | Номер ключа (`N1`), номер сегмента (`N2`), позиция (`N3`) и размер (`N4`) в словаре, позиция (`N5`) и размер (`N6`) в файле |
| **Тип сегмента не соответствует типу поля (N1 – N2)** | Номер ключа (`N1`) и номер сегмента (`N2`) |

## Конвертирование в DDF

### Конвертирование словаря в DDF

Система позволяет конвертировать словарь в формат стандартного словаря фирмы Novell. Это позволяет использовать подготовленную базу данных пользователя в приложениях, разработанных для программ третьих фирм, например, Microsoft Access.

В процессе конвертирования в рабочем каталоге создаётся пустой словарь, состоящий из трёх файлов:

- `file.ddf`
- `index.ddf`
- `field.ddf`

В этот словарь заносится вся информация о базе данных из открытого DEF-словаря.

### Дозапись в DDF

При выборе данного режима в словарь формата DDF, находящийся в рабочем каталоге, дописываются недостающие данные о файлах открытого DEF-словаря.

В результате дозаписи формируется протокол, в который система заносит сообщения о возникших ошибках.

## Директории

С каждым словарём связан файл конфигурации. Файл конфигурации создаётся в том же каталоге, что и словарь. Его имя состоит из имени файла словаря, к которому добавляется символ `$`, и расширения `.cfp`.

В файле конфигурации хранится вся информация, которую пользователь вводит в режиме **«Директории»**.

В полях, определяющих директории файлов, пользователь вводит список директорий, разделённых символом `;`. При обращении к файлам в интерактивном режиме или из макропрограммы система выполняет поиск в установленных директориях.

При работе с Btrieve-файлами в интерактивном режиме поиск файлов выполняется только в первом каталоге из этого списка. Если обращение к файлам производится из макропрограммы на языке RSL, то поиск файлов выполняется во всех каталогах из списка.

### Описание полей настройки директорий

| Поле | Описание | Значение по умолчанию |
|------|----------|----------------------|
| **ДИРЕКТОРИИ DBT-ФАЙЛОВ** | Список директорий с Btrieve-файлами | `..\dbfile` |
| **ДИРЕКТОРИИ DBF-ФАЙЛОВ** | Список директорий DBF-файлов | — |
| **ДИРЕКТОРИИ TXT-ФАЙЛОВ** | Список директорий с текстовыми файлами | `..\txtfile` |
| **ДИРЕКТОРИИ МАКРОФАЙЛОВ** | Список директорий с файлами макропрограмм | — |
| **РАБОЧИЙ ДИРЕКТОРИЙ** | Директория для временных файлов | Совпадает с директорией текстовых файлов |
| **ИМЯ ФАЙЛА С ДИАЛОГОВЫМИ ПАНЕЛЯМИ** | Имя файла диалоговых панелей | — |
| **ИМЯ ФАЙЛА СПИСКА ПРОЦЕДУР** | Файл `..\dbfile\extrep.dbt`, создаётся автоматически | `..\dbfile\extrep.dbt` |
| **ВНЕШНИЙ ТЕКСТОВЫЙ РЕДАКТОР** | Имя внешнего редактора для работы с макропроцедурами | — |

### Функциональные клавиши

| Клавиша | Действие |
|---------|----------|
| `[F9]` | Выход из панели с сохранением изменений в файле конфигурации |
| `[Esc]` | Выход из панели, изменения сохраняются только в течение сеанса работы |

## Файлы

В этом разделе подробно описаны процедуры работы с Btrieve-файлами. При помощи этих процедур пользователь может создавать и просматривать структуры файлов, описанные в открытом словаре.

### Структуры

В данном режиме пользователю предоставлена возможность создавать новые, изменять и удалять уже существующие структуры файлов в словаре.

На экран выводится список структур Btrieve-файлов, определённых в текущем словаре. Ввод новой структуры осуществляется по клавише `[F9]` в пустую строку списка. В списке структур Btrieve-файлов пользователь задаёт структуры записи файла и ключа (клавиши `[F5]` и `[F6]`). Удаление записи осуществляется по клавише `[F8]`.

Следует отметить, что при изменении или удалении структуры в словаре соответствующий Btrieve-файл на диске не изменяется.

#### Поля списка структур файлов

| Поле | Описание |
|------|----------|
| **ИМЯ ФАЙЛА** | Имя Btrieve-файла |
| **ВЛАДЕЛЕЦ** | Имя владельца файла (пароль) |
| **ФФ** | Атрибуты установки пароля файла. Заполняются только при установке имени владельца |
| **СТР.** | Размер страницы Btrieve-файла. Должен быть кратен 512 и не более 4096 |
| **ФЛАГИ** | Флаги Btrieve-файла |
| **МДПЧ** | Используется совокупно с полем **В** при установке флага `Variable length` |
| **В** | Отметка, определяющая, будет ли переменная часть записи использоваться как BLOB |
| **ПРИМЕЧАНИЕ** | Произвольная текстовая информация о файле |

#### Атрибуты поля ФФ

| Атрибут | Описание |
|---------|----------|
| `Encryption` | Содержимое файла будет зашифровано |
| `Read-Only mode` | Доступ только для чтения осуществляется без указания пароля. Если атрибут не указан, пароль требуется для любого доступа |

#### Флаги Btrieve-файла

| Флаг | Описание |
|------|----------|
| `Variable length` | Btrieve-файл будет иметь записи переменной длины |
| `Truncate blanks` | Подавление концевых пробелов |
| `Preallocate` | Не поддерживается в данной версии языка RSL |
| `Compress data` | Сжатие данных в Btrieve-файле |
| `Key-only file` | Btrieve-файл будет создан как файл только ключей. Устанавливается только при одном ключе и маленькой структуре. Без необходимости не следует создавать такие файлы, так как на их использование накладываются ограничения |
| `10% free space` | Резервирует процент страницы для расширения записей переменной длины |
| `20% free space` | Резервирует процент страницы для расширения записей переменной длины |
| `30% free space` | Резервирует процент страницы для расширения записей переменной длины |

Все флаги являются стандартными и подробно описаны в документации к Btrieve.

#### Поле МДПЧ

Поле **МДПЧ** используется в совокупности с полем **В** и заполняется в том случае, если в поле **ФЛАГИ** установлен флаг `Variable length`.

| Отметка в поле В | Значение поля МДПЧ |
|------------------|--------------------|
| Не установлена | Максимальный размер в байтах переменной части записи файла |
| Установлена | Переменная часть записи файла рассматривается как объект типа BLOB; задаётся размер буфера, в который считываются данные из поля BLOB |

#### Функциональные клавиши режима «Структуры»

| Клавиша | Действие |
|---------|----------|
| `[F3]` | Использование встроенного справочника флагов и атрибутов файла |
| `[F4]` | Поиск записи по заданному значению в любом из полей |
| `[F5]` | Список полей файла |
| `[F6]` | Список ключей файла |
| `[F8]` | Удаление структуры Btrieve-файла из списка |
| `[F9]` | Ввод новой структуры в пустую строку списка |
| `[Space]` | Установка и отмена отметки в поле В |
| `[Esc]` | Выход из режима |

### Список полей файла

Вход по клавише `[F5]` из «Списка файлов».

После нажатия клавиши `[F5]` на экран выводится панель, содержащая структуру файла. В ней представлено подробное описание всех полей файла. Список отсортирован по возрастанию номеров полей.

В этом списке пользователю предоставлена возможность изменять структуру файла путём ввода новых, корректировки и удаления уже существующих записей. Ввод нового поля структуры осуществляется в пустую строку списка после нажатия клавиши `[F9]`.

В данном списке пользователю предоставляется возможность по клавише `[F7]` сформировать отчёт, в котором будет содержаться информация о полях и ключах файла.

#### Поля списка полей файла

| Поле | Описание |
|------|----------|
| **НОМ** | Порядковый номер поля. Нумерация определяет порядок сортировки полей в списке |
| **ИМЯ ПОЛЯ** | Имя поля файла |
| **ТИП** | Тип значения, которое будет храниться в поле файла |
| **ДЛИНА** | Длина поля в байтах. Указывается только для полей типа `STRING` и `SNR` |
| **ПРИМЕЧАНИЕ** | Произвольная текстовая информация о поле структуры файла |

#### Типы полей

| Тип | Описание |
|-----|----------|
| `INT` | Целочисленное значение (2 байта) |
| `LONG` | Целочисленное значение (4 байта) |
| `DOUBLE` | Вещественное значение (8 байт) |
| `DOUBLEM` | Денежная величина (8 байт) |
| `STRING` | Символьная строка, заканчивающаяся нуль-символом |
| `SNR` | Символьная строка, состоящая из цифр и выравненная вправо |
| `DATE` | Значение даты |
| `TIME` | Значение времени |
| `CHR` | Одиночный символ |
| `UCHR` | Беззнаковое целое число (1 байт) |
| `DOUBLER` | Величина, эквивалентная `DOUBLEM`, в которой тысячи отделяются апострофами при выводе на экран |

#### Функциональные клавиши режима «Список полей файла»

| Клавиша | Действие |
|---------|----------|
| `[F3]` | Использование встроенного справочника типов полей в поле ТИП |
| `[F4]` | Поиск поля структуры по заданному значению в любом из полей списка |
| `[F7]` | Формирование отчёта о структуре и ключах файла |
| `[F8]` | Удаление поля структуры после подтверждения пользователя |
| `[F9]` | Ввод нового поля структуры в пустую строку списка |
| `[Esc]` | Выход из режима |

### Отчёт о структуре и ключах файла

Вход по клавише `[F7]` из списка полей файла.

После нажатия клавиши `[F7]` на экран выводится сформированный отчёт, содержащий полную информацию о структуре файла и его ключах.

Отчёт содержит две таблицы:

- перечень полей файла, аналогичный содержимому списка полей файлов;
- перечень ключей, аналогичный содержимому списка ключей.

Работа с отчётом осуществляется с помощью встроенного текстового редактора.

### Список ключей файла

Вход по клавише `[F6]` из «Списка файлов».

После нажатия клавиши `[F6]` на экран выводится панель, содержащая список ключей файла. Каждый сегмент ключа представлен в списке отдельной записью. Список отсортирован по возрастанию номеров сегментов.

В этом списке пользователю предоставлена возможность изменять структуру ключа файла путём ввода новых, а также корректировки и удаления уже существующих сегментов. Ввод нового сегмента осуществляется в пустую строку списка после нажатия клавиши `[F9]`.

#### Поля списка ключей файла

| Поле | Описание |
|------|----------|
| **НОМ** | Порядковый номер сегмента ключа |
| **ИМЯ ПОЛЯ** | Имя поля структуры, соответствующее данному сегменту ключа. Заполняется только при помощи списка полей файла по клавише `[F3]` |
| **ФЛАГИ** | Набор флагов ключей Btrieve |
| **ТИП** | Один из расширенных типов ключей Btrieve |
| **ДЛИНА** | Длина сегмента ключа, равная длине соответствующего поля. Заполняется автоматически |
| **ПРИМЕЧАНИЕ** | Произвольная текстовая информация о сегменте |

#### Флаги ключей Btrieve

| Флаг | Описание |
|------|----------|
| `Duplicates` | Значение ключа может дублироваться. Если этот флаг указан для одного сегмента ключа, то он должен быть указан и для всех остальных сегментов данного ключа |
| `Modifiable` | Значение ключа может быть модифицировано |
| `Extended type` | Сегмент ключа принадлежит к расширенным типам ключей Btrieve. Этот флаг следует указывать для всех сегментов |
| `Segment` | Следующий сегмент принадлежит этому же ключу |
| `Alternate order` | Для полей типа `STRING` или `SNR` символы верхнего и нижнего регистра считаются одинаковыми при сортировке |

#### Соответствие типов полей и ключей Btrieve

| Тип поля | Тип ключа Btrieve |
|----------|-------------------|

| `DOUBLE` | `FLOAT` |

| `STRING` | `ZSTRING` |

| `CHR` | `STRING` |

| `TIME` | `TIME` |

#### Функциональные клавиши режима «Список ключей файла»

| Клавиша | Действие |
|---------|----------|
| `[F3]` | Использование встроенных списков в полях ИМЯ ПОЛЯ, ФЛАГИ и ТИП |

| `[F8]` | Удаление сегмента ключа после подтверждения пользователя |

| `[Esc]` | Выход из режима |

### Создание Btrieve-файла

В данном режиме выполняется создание на диске Btrieve-файла, структура которого совпадает со структурой, описанной в словаре.

При входе в режим на экран выводится список структур файлов, описанных в открытом словаре. После нажатия клавиши `[Enter]` система создаёт в установленном директории Btrieve-файл с таким же именем и с такой же структурой, как у выбранного файла. Если такой файл в директории уже существует, система предлагает создать пустой файл взамен существующего.

**Внимание!** При создании нового файла взамен существующего все данные будут уничтожены!

В случае успешного создания файла система выдаёт соответствующее сообщение. При возникновении ошибки система выводит на экран сообщение с кодом ошибки Btrieve.

#### Функциональные клавиши режима «Создание»

| Клавиша | Действие |
|---------|----------|
| `[F4]` | Поиск структуры файла по заданному значению в любом поле списка файлов |
| `[Enter]` | Создание Btrieve-файла со структурой и переход к режиму просмотра |
| `[Esc]` | Выход из режима |

### Установка владельца

В данном режиме осуществляется установка имени владельца файла в соответствии со значением, определённым в словаре.

При входе в режим на экран выводится список файлов, аналогичный описанному в режиме «Создание». После выбора необходимого имени файла система устанавливает пароль в соответствии с атрибутами, заданными пользователем в словаре:

- устанавливается пароль, аналогичный имени владельца, введённому в соответствующем поле списка структур;
- определяется режим доступа к данным в соответствии со значением, введённым пользователем в поле **ФФ** списка структур.

После успешной установки имени владельца файла система выдаёт соответствующее сообщение. При возникновении ошибки система выводит сообщение с кодом ошибки Btrieve.

### Снятие владельца

В данном режиме осуществляется отмена установленного пароля файла.

При входе в режим на экран выводится список файлов, аналогичный описанному в режиме «Создание». После выбора необходимого имени файла система отменяет действия пароля.

В случае успешного выполнения этой процедуры система выдаёт соответствующее сообщение. При возникновении ошибки система выводит на экран сообщение с кодом ошибки Btrieve.

### Заимствование структур

В данном режиме пользователю предоставлена возможность копировать структуры файлов в словаре. Можно заимствовать структуры как в текущем словаре, так и из разных словарей.

При входе в режим на экран выводится панель, в которой необходимо в соответствующих полях указать имя файла, структура которого будет заимствована, и имя файла, заимствующего структуру. Кроме этого, в соответствующие поля нужно ввести имена словарей, в которых эти файлы определены.

Ввод значений в поля **ФАЙЛ** можно осуществлять по нажатию клавиши `[F3]`. При этом на экран выводится список структур словаря. При нажатии клавиши `[F3]` в поле **СЛОВАРЬ** на экране появляется диалоговое окно выбора имени файла, с помощью которого пользователь может ввести имя словаря.

#### Функциональные клавиши режима «Заимствование структур»

| Клавиша | Действие |
|---------|----------|
| `[F3]` | Использование списка структур файлов, а также выбор имени файла словаря в соответствующих полях |
| `[F5]` | Выполнение копирования структуры файла |
| `[Esc]` | Выход из режима |

## Данные

В данном разделе дано описание режимов, в которых осуществляется просмотр и редактирование Btrieve-файлов при помощи шаблона, описанного в словаре.

### Просмотр

При выборе данного режима на экран выводится список файлов, описанных в словаре. В нём пользователю необходимо выбрать нужный файл для просмотра.

После нажатия клавиши `[Enter]` на экран выводится панель скроллинга, столбцы которой представляют собой поля файла. В качестве заголовков столбцов скроллинга используются имена полей файла, определённые в словаре.

Если общая ширина полей файла превышает ширину экрана, то на экран выводится только часть полей. Переход к остальным полям можно осуществить при помощи клавиш перемещения курсора.

#### Функциональные клавиши режима «Просмотр»

| Клавиша | Действие |
|---------|----------|
| `[F4]` | Поиск записи по заданному значению в любом из полей |
| `[F5]` | Просмотр содержимого всех полей записи |
| `[F6]` | Выбор ключа для изменения сортировки файла |
| `[Alt+F4]` | Поиск записи по заданному ключевому значению в любом из полей |
| `[Esc]` | Выход из режима |

### Содержимое полей записи

Вход по клавише `[F5]` из режима просмотра файла.

В данном режиме предоставляется возможность просматривать поля только одной записи, отмеченной курсором. При этом на экран выводится панель, содержащая перечень полей записи и их значений. Наличие каждого поля файла в данной панели устанавливается пользователем в режиме корректировки изображения файла.

#### Функциональные клавиши

| Клавиша | Действие |
|---------|----------|
| `[Esc]` | Выход из режима |

### Выбор ключа

Вход по клавише `[F6]` из режима просмотра файла.

Панель просмотра файла выводит на экран записи в физической последовательности, однако последовательность вывода записей на экран можно изменить. Для этого при помощи клавиши `[F6]` нужно задать номер ключа, по которому будут отсортированы записи в файле.

При этом на экране появится панель, содержащая список номеров ключей файла. Пользователю необходимо выбрать требуемый номер ключа, в результате записи файла будут отсортированы по указанному ключу.

В этот список также входит строка **«Физ. п.»**, при выборе которой записи файла будут выведены на экран в физической последовательности.

### Поиск записи по ключевому значению

Вход по клавише `[Alt+F4]` из режима просмотра или редактирования файла.

С помощью данного режима осуществляется оперативный поиск записи в файле по значению ключа. Так как в процессе поиска применяются средства Btrieve, данный режим незаменим в тех случаях, когда требуется быстро найти какую-либо запись в файле большого размера, например, `Arhdoc`, `Account`, `Restdate` и т.п.

При выборе режима поиска на экран выводится окно, в котором отображается первое значение ключа, с помощью которого по клавише `[F6]` отсортированы записи в файле.

В этом окне необходимо указать все значения ключевых полей записи, которую нужно найти, и в поле без названия с помощью списка, вызываемого по клавише `[F3]`, выбрать одно из условий поиска:

- искать равное;
- искать большее;
- искать большее либо равное;
- искать меньшее;
- искать меньшее либо равное.

После нажатия клавиши `[F2]` курсор будет установлен на первую запись, удовлетворяющую заданным условиям.

#### Функциональные клавиши режима «Поиск по ключевому значению»

| Клавиша | Действие |
|---------|----------|
| `[F2]` | Выполнение поиска записи по заданному ключевому значению |
| `[F3]` | Использование списка в поле для установки условий поиска |
| `[Esc]` | Выход из режима |

### Редактирование

Режим позволяет в интерактивном режиме редактировать файлы, описанные в словаре.

При выборе данного пункта меню на экран выводится панель выбора файла, аналогичная приведённой в режиме «Создание». После нажатия клавиши `[Enter]` пользователь переходит в режим редактирования файла. Панель редактирования файла аналогична панели для просмотра.

#### Функциональные клавиши режима «Редактирование»

| Клавиша | Действие |
|---------|----------|
| `[F4]` | Поиск записи по заданному значению в любом из полей |
| `[F5]` | Просмотр содержимого всех полей записи |
| `[F6]` | Выбор ключа для изменения сортировки файла |
| `[F8]` | Удаление записи из файла |
| `[F9]` | Ввод новой записи в пустую строку панели |
| `[Esc]` | Выход из режима |

### Изображение

В данном режиме пользователю предоставляются следующие возможности:

- выбор полей, которые будут отображаться на экране в режимах просмотра и редактирования файла;
- установка ширины выводимых полей файла на экране;
- установка порядка полей файла при экспорте/импорте.

При выборе данного пункта меню на экран выводится панель выбора файла, аналогичная приведённой в режиме «Создание». После нажатия клавиши `[Enter]` пользователь переходит в режим корректировки изображения полей записи.

В панели выводится список полей выбранного файла. Для каждого поля высвечиваются текущие значения, определяющие его изображение.

#### Поля корректировки изображения

| Поле | Описание |
|------|----------|
| **ИМЯ ПОЛЯ** | Информация о поле структуры, определённом в словаре. Заполняется автоматически, недоступно для корректировки |
| **ТИП** | Информация о типе поля структуры. Заполняется автоматически, недоступно для корректировки |
| **ДЛИНА ПОЛЯ_В ФАЙЛЕ** | Длина поля, указанная в словаре |
| **ДЛИНА ПОЛЯ_В ФОРМЕ** | Длина поля на экране в режимах просмотра и редактирования файлов. По умолчанию равна длине поля в файле. Для типа `STRING` не должна превышать значения, меньшего на единицу, чем длина строки в файле. Для типа `DATE` не изменяется |
| **ТОЧКА** | Количество знаков после десятичной точки в поле вывода значения. Для данных типа `DOUBLE` и `MONEY` устанавливается равным `2`. Для строковых данных значение должно быть равно длине строки в файле |
| **ПРИСУТСТВИЕ_СКРОЛ.** | Отметка о присутствии данного поля структуры на экране в режимах просмотра и редактирования файла. Установка и отмена пометки — клавиша `[Space]` |
| **ПРИСУТСТВИЕ_ПАНЕЛЬ** | Отметка о присутствии поля структуры в панели просмотра записи |
| **ПРИСУТСТВИЕ_ЭКСП.** | Расположение поля структуры в текстовом файле экспорта. По умолчанию равно `1`. Если установлено `0`, поле не участвует в экспорте |
| **ПРИСУТСТВИЕ_ИМП.** | Расположение поля структуры в текстовом файле импорта. По умолчанию равно `1`. Если установлено `0`, поле не участвует в импорте |

По умолчанию в полях `ПРИСУТСТВИЕ_ЭКСП.` и `ПРИСУТСТВИЕ_ИМП.` система устанавливает значение, равное `1`. При этом в текстовом файле экспорта/импорта поля Btrieve-файла будут располагаться в том же порядке, как в структуре.

Пользователь также может устанавливать в этих полях любые целочисленные значения. При этом поля записи Btrieve-файла будут расположены в порядке, установленном пользователем.

#### Функциональные клавиши режима «Изображение»

| Клавиша | Действие |
|---------|----------|
| `[F4]` | Поиск записи по заданному значению в любом из полей |
| `[Space]` | Установка или отмена отметки в полях присутствия |
| `[Esc]` | Выход из режима |

### Экспорт

В данном режиме осуществляется выгрузка данных Btrieve-файла в текстовый файл. Пользователю предоставлена возможность экспортировать любой Btrieve-файл, структура которого описана в открытом словаре.

Файл выгружается позаписно. Каждое поле записи отделяется символом-разделителем, установленным пользователем. Поля записей выгружаются в порядке, установленном в режиме «Изображение».

При входе в режим на экран выводится панель, в которой пользователю необходимо ввести параметры экспорта.

#### Поля панели экспорта

| Поле | Описание |
|------|----------|
| **ОПИСАНИЕ СТРУКТУРЫ** | Имя структуры Btrieve-файла, описанной в словаре. Ввод по `[F3]` из списка структур |
| **BTRIEVE-ФАЙЛ** | Имя Btrieve-файла, данные которого будут экспортироваться |
| **ФАЙЛ ЭКСПОРТА** | Имя текстового файла выгрузки |
| **РАЗДЕЛИТЕЛЬ ПОЛЕЙ** | Символ, отделяющий поля записи в текстовом файле экспорта. По умолчанию — `,` |
| **НОМЕР КЛЮЧА** | Порядок сортировки записей в Btrieve-файле |

#### Функциональные клавиши режима «Экспорт»

| Клавиша | Действие |
|---------|----------|
| `[F3]` | Использование списка структур файлов, а также выбор имени текстового и Btrieve-файла |
| `[F5]` | Выполнение экспорта файла |
| `[Esc]` | Выход из режима |

### Импорт

В данном режиме осуществляется загрузка данных текстового файла в Btrieve-файл.

После входа в режим на экран выводится панель установки параметров импорта, аналогичная панели экспорта.

Порядок заполнения полей, а также использование функциональных клавиш в ней аналогичны описанным в режиме «Экспорт».

### Конвертирование

В данном режиме пользователю предоставляется возможность копировать данные из одного Btrieve-файла в другой.

При помощи процедуры конвертирования можно копировать Btrieve-файлы разной структуры. Это позволяет изменять структуру файла, не теряя при этом данных. Для этого используется следующая технология:

1. Пользователь создаёт временный файл с новой структурой.
2. Далее выполняется конвертирование файла со старой структурой во временный.
3. После этого пользователю необходимо переименовать временный файл и его структуру в словаре, вернув ему прежнее имя.

При входе в режим на экран выводится панель ввода параметров конвертирования, аналогичная панели заимствования структур. Ввод значений и использование функциональных клавиш осуществляются так же, как и при заимствовании структур файлов.

В результате выбранный файл макрозадания при помощи стандартного встроенного текстового редактора выводится на экран.

### Работа со встроенным текстовым редактором

Управление редактированием текстов осуществляется как при помощи функциональных клавиш, так и с помощью встроенного меню, вызываемого по клавише `[F10]`.
