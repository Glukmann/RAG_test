# DBExp

Введение
Руководство администратора содержит описание работы с программой DB Explorer.
Программа DB Explorer предназначена для работы со структурами таблиц в базе данных,
которая заключается в осуществлении следующих действий:
· создание и изменение словаря;
· добавление и редактирование структур таблиц в словаре;
· создание таблиц в соответствии с их структурами;
· проверка таблиц на соответствие структурам;
· создание и удаление индексов таблиц и последовательностей индексов.
Чтобы приступить к работе с программой следует выполнить ряд действий:
· Установить из дистрибутива утилиту DB Explorer, используя режим "Дополнительные
компоненты" программы установки ИБС RS-Bank V.6. Описание программы приведено в
Руководстве "Установка ИБС RS-Bank V.6 release 20" (файл Books\Install.pdf). По
умолчанию утилита будет установлена в каталог EXT\DBEXP.EXT.
Внимание!
Для корректной работы утилиты не рекомендуется модули из этого
каталога размещать в каталоге OBJ установленной системы RS-Bank
V.6.
· Запустить из каталога файл DbExp.exe.

Свойства таблицы
На Рис. 3

следующие атрибуты:
· ID – номер структуры в словаре. Значение поля устанавливается автоматически после
создания структуры и недоступно для корректировки.
· Имя таблицы – имя структуры. Поле обязательно для заполнения при создании
структуры и может состоять из латинских букв, цифр и знака подчёркивания. После
сохранения структуры поле недоступно для корректировки. Имя структуры может быть в
формате *.dbt, *.tmp, *.rec, либо вообще без расширения. Это зависит от целей создания
структуры: например, для создания физической таблицы в Oracle по этой структуре в
формате d<name>_dbt, имя структуры задается в формате *.dbt. Максимальная длина
имени составляет 29 символов.
· Комментарий – характеристика структуры. Поле может содержать буквы кириллицы.
Сервис
· Первичный ключ – номер индекса, являющегося первичным ключом. Нумерация
индексов начинается с 0. При отсутствии первичного ключа в поле указывается значение
"-1".
· Длина переменной части – максимальный размер буфера, который выделяется в
программе для таблиц с записями переменной длины. В СУБД Oracle такие записи
представлены полями типа Blob.
· Временная – признак структуры временной Oracle-таблицы. По такой структуре
физически будет создана Global Temporary Table (GTT). Отметка, определяющая действие
признака, устанавливается автоматически по клавише [Space] или щелчку левой кнопки
мыши.
· Структура – признак, определяющий, что данная структура представляет лишь набор
полей для представления в RSL, но не связана ни с какой таблицей базы данных.
Элементы словаря с этим признаком игнорируются при массовом создании таблиц.
Отметка, определяющая действие признака, устанавливается автоматически по клавише
[Space] или щелчку левой кнопки мыши.

Индексы
Закладка "Индексы" (см. Рис. 5
) содержит перечень индексов и их сегментов.
Строки, содержащие информацию об индексах, выделены желтым цветом. Каждый индекс
состоит из одного или нескольких сегментов. Порядок расположения сегментов в индексе
можно изменять с помощью кнопок 
, расположенных в левой нижней части окна. При
этом запись о сегменте индекса будет перемещена на одну позицию вверх или вниз.
Добавление или удаление записи об индексе выполняется по кнопкам "Добавить индекс"
или "Удалить индекс" соответственно. Добавление или удаление записи о сегменте
выполняется по кнопкам "Добавить сегмент" или "Удалить сегмент" соответственно.
Кнопки расположены в нижней части окна. 
Примечание.
При добавлении сегмента следует предварительно выделить строку с
индексом, кликнув на серую ячейку слева от ячейки с именем индекса. Для
Сервис
удаления сегмента или индекса следует таким же образом выделить
соответствующие строки.
Для индексов и их сегментов можно указать следующие атрибуты:
· Имя – имя индекса/сегмента. Имя индекса генерируется автоматически, при его создании:
к имени таблицы в базе данных добавляется суффикс "_IDX0(1, 2, 3..)". Имя сегмента
индекса совпадает с именем столбца таблицы и выбирается из раскрывающегося списка
столбцов.
· Тип – тип данных сегмента индекса. Значение атрибута устанавливается автоматически
после задания имени сегмента в соответствии с типом данных столбца таблицы.
Недоступно для корректировки.
· DUP – признак дублирования индекса. Отметка, которая определяет действие признака,
устанавливается в строке, соответствующей записи об индексе, по щелчку левой кнопки
мыши или нажатию клавиши [Space]. Если отметка установлена пользователем, то в
строки, соответствующие записям о сегментах индекса, она переносится автоматически и
не может быть снята вручную.
· NULL – способ построения индекса с NULL-сегментами. Если значение атрибута не
задано, то в индекс попадают записи с любыми значениями сегментов. Установка
значения атрибута выполняется с помощью раскрывающегося списка, который содержит
следующие записи:
· ALL – в индекс не попадают записи с полностью NULL-сегментами.
· ANY – в индекс не попадают записи хотя бы с одним NULL-сегментом.
Если значение атрибута задано, то в строки, соответствующие записям о сегментах
индекса, оно переносится автоматически и не может быть изменено вручную.
· Исключить из NULL – признак того, что сегмент не учитывается при построении индекса
с NULL-сегментами. Отметка, определяющая действие признака, устанавливается в
строке, соответствующей записи о сегменте индекса, по щелчку левой кнопки мыши или
нажатию клавиши [Space].
· Autoinc – признак автоинкрементного индекса. Отметка, определяющая действие
признака, устанавливается в строке, соответствующей записи об индексе, по клавише
[Space] или щелчку левой кнопки мыши. Отметка может быть установлена для индекса,
состоящего из одного сегмента целого типа. В таблице не может быть больше одного
автоинкрементного индекса. Если отметка установлена пользователем, то в строку,
соответствующую записи о сегменте индекса, она переносится автоматически и не может
быть снята вручную.
· Desc order – признак обратного порядка сортировки сегмента индекса. Отметка,
определяющая действие признака, устанавливается в строке, соответствующей записи о
сегменте индекса, по щелчку левой кнопки мыши или нажатию клавиши [Space].
· Исключить из индекса – признак исключения сегмента из индекса. Отметка,
определяющая действие признака, устанавливается в строке, соответствующей записи о
сегменте индекса, по щелчку левой кнопки мыши или нажатию клавиши [Space].
· LOCAL – признак создания локального индекса. Отметка, определяющая действие
признака, устанавливается в строке, соответствующей записи об индексе, по щелчку
левой кнопки мыши или нажатию клавиши [Space]. Признак используется при создании
индексов на секционированных таблицах.
