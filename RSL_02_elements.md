Данная глава содержит описание языка интерпретатора RSL. Изучение этой главы предполагает знание языков программирования высокого уровня.

Наличие хорошо структурированного языка, тесно связанного с базой данных, значительно расширяет возможности системы. Язык может быть использован для создания силами работников банка специфических отчётов, отсутствующих в системе, или других процедур, реализующих дополнительную функциональность.

Язык интерпретатора RSL — это язык высокого уровня. Он достаточно прост в изучении и применении и одинаково хорош как для квалифицированного, так и для неквалифицированного пользователей системы, так как имеет широкий спектр функциональных возможностей. В отличие от других подобных ему языков, язык интерпретатора интегрирован с системами, разработанными программистами компании R-Style Softlab. Он вызывается непосредственно из меню системы, обеспечивает доступ к базе данных, использует одинаковые с ней форматы данных, работает с экранными формами.

## Служебные слова

Служебное, или зарезервированное, слово — это имя, с которым в языке жёстко сопоставлены определённые смысловые значения, и которое не может быть использовано ни для каких других целей.

### Основные служебные слова RSL

Служебные слова записываются в любом регистре.

```rsl
and      if        record
array    import    return
const    macro     this
class    not       true
elif     NULL      var
end      onerror   with
false    or        while
file     local     private
```

### Служебные слова для FILE и RECORD

```rsl
btr   key   txt
dbf   mem   sort
dialog     write
```

Значение каждого служебного слова поясняется при описании соответствующих конструкций языка.

## Имена (идентификаторы)

Под именем (идентификатором) в программе понимается не являющаяся служебным словом последовательность букв (латинских и русских) и цифр, начинающаяся с буквы. Символ `_` (подчёркивание) рассматривается как буква.

- Длина идентификатора в RSL не должна превышать **80 символов**.
- Прописные и строчные буквы не различаются, то есть регистр игнорируется.
- Имена используются для идентификации объектов RSL.

### Специальные переменные

Кроме обычных идентификаторов, в RSL применяются имена специальных переменных, которые могут содержать любые символы, включая специальные, без ограничений. Такие имена должны быть заключены в фигурные скобки `{}`, которые также входят в имя переменной.

Специальные переменные являются глобальными и используются обычно для передачи значений из вызывающих модулей, написанных на C или C++, в программу на языке RSL.

### Составные имена

Для обозначения полей в структурах и файлах, описанных в словаре базы данных, используются составные имена. Составные имена состоят из имени, идентифицирующего файл или структуру, после которого следует точка и название поля либо индекс поля в круглых скобках.

### Примеры имён

```rsl
Счет                       /* простое имя */
Сумма010                   /* простое имя */
{Debet account 52.5}       /* имя специальной переменной */
Клиенты.Name               /* составное имя: поле Name в структуре Клиенты */
Клиенты(10)                /* составное имя: десятое поле в структуре Клиенты */
```

### Системные переменные среды

Если имя переменной совпадает с именем переменной среды, оно инициализируется её значением.

```rsl
PATH /* переменная будет проинициализирована значением системной переменной PATH,
        содержащей список каталогов для поиска программ */
```

Присвоение значений этим переменным не изменяет содержимого системных переменных.

## Область видимости имен

| Область | Описание |
|---------|----------|
| **Глобальная** | Имена, объявленные вне любой процедуры RSL. Специальные переменные всегда глобальны |
| **Локальная** | Имена, объявленные внутри процедуры RSL. Доступны только внутри этой процедуры и всех вложенных |

- Из вложенной области видимости доступны имена этой области и всех вышележащих.
- В каждой области видимости можно "перекрывать" имена из вышележащих областей.
- При входе в область видимости переменные конструируются, при выходе разрушаются.
- Время жизни переменной ограничено областью её видимости.

## Комментарии

Любой фрагмент текста, заключённый в скобки вида `/* */`, является комментарием. Язык RSL разрешает наличие вложенных комментариев.

```rsl
/*********************************************************
Комментарий на языке RSL
/*
Вложенный комментарий
*/
**********************************************************/
```

Данный пример содержит два комментария, один из которых вложен в другой.

## Объекты языка RSL

Объект RSL представляет собой совокупность информации, с которой оперирует язык.

### Виды объектов

| Объект | Описание |
|--------|----------|
| **Переменная** | Объект, содержащий значение одного из типов данных. Может изменять величину и тип хранимого значения при присваивании |
| **FILE** | Хранит информацию о Btrieve-файле (структуру файла и текущие значения полей) |
| **RECORD** | Хранит информацию о структуре файла или диалоговой панели |
| **DBFFILE** | Хранит информацию о файле формата DBF |
| **TXTFILE** | Хранит информацию о текстовом файле |
| **ARRAY** | Вектор переменных, который можно индексировать |
| **CLASS** | Совокупность свойств и методов; служит для создания объектов |

### Пример переменной

```rsl
aa = 10;           /* aa хранит значение V_INTEGER */
aa = "Привет";     /* aa хранит значение V_STRING */
```
